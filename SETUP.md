# Установка и запуск

**[English version →](SETUP_EN.md)**

## Требования

- Python 3.10+
- GPU с CUDA (работает и на CPU, медленнее)
- GGUF-модель в папке `models/`

---

## Установка

```bash
git clone https://github.com/Volk-Arch/Baddle
cd baddle
python setup.py        # определит CUDA, установит llama-cpp-python + скачает llama-server
pip install flask      # для веб-UI
```

После установки положи GGUF-модель в папку `models/` (создаётся автоматически):

```
models/Qwen3-8B-Q4_K_M.gguf
```

### Какие модели подходят

Baddle работает с моделями в формате **GGUF** — это формат llama.cpp для запуска
на CPU/GPU без PyTorch. Модели можно скачать с [Hugging Face](https://huggingface.co/models?library=gguf).

**Квантование** — модели бывают разных размеров (квантований):

| Квантование | Размер (для 8B) | Качество | Когда использовать |
|---|---|---|---|
| **Q4_K_M** | ~5 GB | хорошее | основной выбор, баланс скорости и качества |
| **Q5_K_M** | ~5.5 GB | лучше | если хватает VRAM |
| **Q8_0** | ~8.5 GB | почти без потерь | для исследования качества квантования |
| **F16** | ~16 GB | оригинал | нужно много VRAM, максимальное качество |

**Как выбрать:**
- Для GPU с 8 GB VRAM — Q4_K_M моделей до 8B параметров
- Для GPU с 12 GB VRAM — Q5_K_M/Q8_0 моделей до 8B, или Q4_K_M до 14B
- Для GPU с 24 GB VRAM — Q8_0 до 14B, или Q4_K_M до 32B

**Контекст** (`--ctx`) влияет на потребление памяти. По умолчанию 4096 токенов.
Увеличение до 8192+ требует дополнительной VRAM.

`setup.py` автоматически:
- Определит версию CUDA
- Установит `llama-cpp-python` (с GPU если есть CUDA, иначе CPU)
- Скачает нативный `llama-server` в папку `llama-server/` (для параллельного режима)

> **Установка занимает 5–15 минут** — это нормально.
> `llama-cpp-python` компилируется из C++ исходников.
> При наличии CUDA дополнительно собираются GPU-ядра.

## Запуск

```bash
python ui.py             # веб-интерфейс на localhost:7860
python ui.py --server    # веб-интерфейс + параллельный сервер (рекомендуется)
```

Если в `models/` одна модель — загрузится автоматически. Если несколько — появится выбор.

```bash
python ui.py -m Qwen3-8B-Q4_K_M.gguf   # выбрать модель явно
python ui.py --ctx 8192                 # увеличить контекст (по умолчанию 4096)
python ui.py --gpu-layers 20            # частичная выгрузка на GPU (по умолчанию все слои)
python ui.py --no-gpu                   # только CPU
python ui.py --port 8080                # другой порт
```

> Эти параметры задаются **только при запуске** — они определяют как модель
> загружается в память и не могут быть изменены на лету. Для смены нужно
> перезапустить `ui.py`.

| Параметр | По умолчанию | Что делает |
|---|---|---|
| `-m` / `--model` | авто (единственная в `models/`) | какую GGUF-модель загрузить |
| `--ctx` | 4096 | размер контекста в токенах, больше = больше VRAM |
| `--gpu-layers` | -1 (все) | сколько слоёв модели выгрузить на GPU; -1 = все |
| `--no-gpu` | выкл | принудительно CPU, игнорировать GPU |
| `--port` | 7860 | порт веб-интерфейса |
| `--server` | выкл | использовать llama-server для параллельной генерации |

## Параметры генерации

Параметры настраиваются прямо в интерфейсе, для каждого режима свои:

### Температура (temp)

Управляет «случайностью» выбора следующего токена.

| Значение | Эффект |
|---|---|
| **0.0** | всегда выбирается самый вероятный токен (greedy) |
| **0.3–0.7** | баланс предсказуемости и разнообразия |
| **1.0** | стандартное распределение вероятностей |
| **1.2–2.0** | маловероятные токены становятся равноценными — креативный, но нестабильный текст |

Доступна в: step (поле `temp`), parallel/compare (поля `temp A`/`temp B`), chat (поле `temp`).
В step и chat можно менять на лету — влияет на следующий токен.

### Top-K

Перед выбором токена модель оставляет только top-K самых вероятных кандидатов,
остальные отбрасываются. Меньше K — предсказуемее, больше — разнообразнее.

| Значение | Эффект |
|---|---|
| **1** | эквивалент temp=0 (всегда один вариант) |
| **10–40** | стандартный диапазон |
| **100** | почти не фильтрует |

Доступен в: step (поле `top_k`), parallel/compare (поля `top_k A`/`top_k B`).

### Seed

Фиксирует случайность. С одинаковым seed и параметрами результат идентичен —
удобно для A/B тестирования: меняешь один параметр и видишь именно его эффект.
Значение `-1` — случайный seed каждый раз.

Доступен в: parallel, compare.

### Max tokens (chat)

Лимит токенов на один ответ. Если ответ обрезался — кнопка **Continue** догенерирует.
По умолчанию 200.

### Где что настраивается

| Параметр | Step | Parallel | Compare | Chat |
|---|---|---|---|---|
| temp | в панели | поля A/B | поля A/B | в панели |
| top_k | в панели | поля A/B | поля A/B | — |
| seed | — | есть | есть | — |
| max tokens | Auto N | поле N | поле N | поле max |

## Server mode

`--server` включает настоящий параллелизм — два промпта обрабатываются одновременно.

```bash
# Автозапуск (сервер запустится и остановится сам):
python ui.py --server

# Подключение к уже запущенному серверу:
python ui.py --server http://localhost:8080
```

`--server` без URL найдёт `llama-server` в папке `llama-server/`, запустит его
и остановит при выходе.

| | In-process (по умолчанию) | Server mode (`--server`) |
|---|---|---|
| **Как работает** | Один запрос, потом второй (последовательно) | Оба запроса на GPU параллельно |
| **Скорость** | ~2x от одного запроса | ~1x от одного запроса |
| **Step mode** | Есть (доступ к logits) | Нет |

**Для step mode нужен in-process** (прямой доступ к logits).
**Для быстрого parallel/compare — `--server`.**

#### Откуда берётся llama-server

`python setup.py` скачивает бинарник автоматически с
[llama.cpp releases](https://github.com/ggml-org/llama.cpp/releases).

Если setup не скачал (не Windows, нет интернета) — вручную:

1. Зайди на https://github.com/ggml-org/llama.cpp/releases
2. Скачай `llama-bXXXX-bin-win-cuda-XX.X-x64.zip` (версия CUDA как у тебя)
3. Скачай `cudart-llama-bin-win-cuda-XX.X-x64.zip` (CUDA runtime DLL)
4. Распакуй оба в `llama-server/` внутри проекта

> Узнать свою версию CUDA: `nvidia-smi` → строка `CUDA Version: XX.X`

## Кнопки и управление

### Step mode

| Кнопка | Действие |
|---|---|
| **Init** | загрузить промпт в модель |
| **Next Token** | сгенерировать один токен |
| **Auto** + N | автоматически N токенов без остановки |
| **Stop** | остановить Auto-генерацию |
| **Edit** | включить редактирование текста (inject, обрезка, правки) |
| **Sync** (Ctrl+Enter) | применить изменения — модель подхватит отредактированный текст |
| **Reset** | откатиться к исходному промпту |

Панель **Next token probs** показывает top-10 вероятных следующих токенов с гистограммой.
Параметры **temp** и **top_k** можно менять на лету — влияют на следующий токен.

### Parallel / Compare

| Кнопка | Действие |
|---|---|
| **Generate** | запустить генерацию обоих потоков |
| **Stop** | остановить генерацию |
| **→ Step** | после генерации — переключиться в step mode с выбранным потоком (только in-process) |

В compare-режиме отображается badge с номером шага расхождения.

### Chat

| Кнопка | Действие |
|---|---|
| **Send** | отправить сообщение |
| **Stop** | остановить генерацию ответа |
| **Continue** | догенерировать ответ, если он обрезался по лимиту токенов |
| **Reset** | очистить историю и сбросить состояние модели |

Параметры: **temp** (температура), **max** (лимит токенов на ответ).

### Общее для всех режимов

- **Heatmap уверенности** — токены окрашиваются по энтропии: зелёный (уверена) → жёлтый → красный (гадает)
- **Счётчик токенов** — в шапке, показывает использованные / доступные (от n_ctx), краснеет при >90%
- **Роли** — общий выпадающий список из `roles.json` для всех режимов. В step/parallel/compare — prefix, в chat — system message

## Структура проекта

```
baddle/
├── main.py            # движок: модель, сэмплинг, batch-генерация
├── ui.py              # веб-интерфейс (Flask + SSE), точка входа
├── server_backend.py  # HTTP-клиент для llama-server
├── setup.py           # установщик: llama-cpp-python + llama-server
├── models/            # GGUF-модели
└── llama-server/      # нативный бинарник (скачивается setup.py)
```
