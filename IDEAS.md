# Baddle — Ideas

---

## Класс применений: управляемое автодополнение

Большинство LLM-продуктов — это «чат»: промпт → ответ. Baddle работает иначе —
это **автодополнение с полным контролем**, ближе по духу к GitHub Copilot или
автодополнению в IDE, но для произвольного текста и с возможностью вмешательства.

Разница с чатом:
- **Чат:** человек задаёт вопрос, ИИ отвечает целиком. Контроль — только через промпт.
- **Baddle:** человек начинает фразу, ИИ продолжает токен за токеном. Человек может
  остановить, поправить, сменить направление — и ИИ подхватывает с новой точки.

Это открывает класс задач, где чат бесполезен:
- **Соавторство** — не «напиши мне текст», а совместное написание, где человек
  и модель чередуются
- **Отладка промптов** — видеть где именно модель сворачивает не туда,
  а не просто получать плохой результат
- **Исследование модели** — как параметры влияют на генерацию,
  где модель уверена, а где гадает

## Сценарии

### A/B тестирование параметров

Compare + seed. Фиксируешь seed, меняешь один параметр — видишь чистый эффект
без случайного шума. Где ответы совпадают — модель уверена. Где расходятся —
зона, где настройки действительно влияют на результат.

### Точки бифуркации

Step + top-10 показывает где модель колеблется. Когда топ-3 токена имеют
близкие вероятности — минимальный inject максимально меняет дальнейший текст.
Для писателей, сценаристов, гейм-дизайнеров.

### Отладка галлюцинаций

Модель врёт — в step-режиме видно на каком именно токене вероятность
фактического ответа упала. Можно вручную вернуть генерацию.
Для разработчиков RAG-систем.

### Исследование квантования

Одна модель в разных квантованиях (Q4, Q8) — compare показывает
где именно теряется качество. Какие темы, какие конструкции страдают первыми.

### Обучение prompt engineering

Видно не просто «модель ответила плохо», а на каком шаге и почему.
Heatmap уверенности показывает — модель угадывала или знала.
Полезно для курсов и воркшопов по промпт-инженерии.

### Карта уверенности модели

Heatmap во всех режимах позволяет визуально оценить, насколько модель
«уверена» в каждом токене. Паттерн: факты — зелёные (низкая энтропия),
стилистика и связки — жёлтые, выдумка и галлюцинации — красные.
Можно использовать как инструмент верификации: если ответ модели весь
зелёный — скорее всего это воспроизведение обучающих данных; если красный —
модель импровизирует.

### Сравнение моделей

Parallel с двумя разными GGUF на одном промпте. Одна модель —
Q4_K_M, другая — Q8_0 (или вообще другая архитектура). Видно не абстрактный
бенчмарк, а конкретно: вот этот токен одна модель выбрала уверенно,
а другая колебалась. Пока не реализовано — требует загрузки двух моделей
одновременно.

### Стресс-тест промптов

Compare с крайними параметрами (temp=0 vs temp=1.5) показывает
робастность промпта. Если оба потока дают похожий результат — промпт
сильный, модель уверена в ответе. Если расходятся сразу — промпт
оставляет слишком много свободы, результат зависит от случайности.
Инструмент для оценки качества промптов перед продакшеном.
