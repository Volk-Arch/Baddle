# Baddle — Ideas

---

## Класс применений: управляемое автодополнение

Большинство LLM-продуктов — это «чат»: промпт → ответ. Baddle работает иначе —
это **автодополнение с полным контролем**, ближе по духу к GitHub Copilot или
автодополнению в IDE, но для произвольного текста и с возможностью вмешательства.

Разница с чатом:
- **Чат:** человек задаёт вопрос, ИИ отвечает целиком. Контроль — только через промпт.
- **Baddle:** человек начинает фразу, ИИ продолжает токен за токеном. Человек может
  остановить, поправить, сменить направление — и ИИ подхватывает с новой точки.

Это открывает класс задач, где чат бесполезен:
- **Соавторство** — не «напиши мне текст», а совместное написание, где человек
  и модель чередуются
- **Отладка промптов** — видеть где именно модель сворачивает не туда,
  а не просто получать плохой результат
- **Исследование модели** — как параметры влияют на генерацию,
  где модель уверена, а где гадает

## Сценарии

### A/B тестирование параметров

Compare + seed. Фиксируешь seed, меняешь один параметр — видишь чистый эффект
без случайного шума. Где ответы совпадают — модель уверена. Где расходятся —
зона, где настройки действительно влияют на результат.

### Точки бифуркации

Step + `top` показывает где модель колеблется. Когда топ-3 токена имеют
близкие вероятности — минимальный inject максимально меняет дальнейший текст.
Для писателей, сценаристов, гейм-дизайнеров.

### Отладка галлюцинаций

Модель врёт — в step-режиме видно на каком именно токене вероятность
фактического ответа упала. Можно вручную вернуть генерацию.
Для разработчиков RAG-систем.

## Фичи

| Фича | Что даёт | Сложность |
|------|----------|-----------|
| `cut N` | Обрезать генерацию до токена N, продолжить с другим inject | Низкая |
| Heatmap уверенности | Цвет токена по энтропии — зелёный (уверена), красный (гадает) | Низкая |
| Diff-режим | Подсветка расхождений между двумя потоками в compare | Низкая |
| State branching | Сохранить KV cache → ветка А → откат → ветка Б без пересчёта | Средняя |
| Гибридный режим | parallel → выбрал лучший поток → переключился в step | Средняя |
| Streaming diff моделей | Два квантования одной модели side-by-side, видно где Q4 теряет | Средняя |
| Batch benchmark | N промптов × M конфигов → CSV со скоростью и метриками | Средняя |
