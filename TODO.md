### Камень 7: Storage layer
*Переход от "граф в памяти" к "граф как файловая БД с lazy load". Своя графовая БД на файлах.*

**Фаза 0b — JSONL файловое хранение (следующий шаг):**
- [ ] `nodes.jsonl` — одна строка на узел, append-only. Streaming read, частичная загрузка
- [ ] `edges.jsonl` — одна строка на связь. Отдельно similarity, manual, directed
- [ ] `meta.json` — topic, settings, позиции узлов
- [ ] Папка `graphs/` — каждый граф в своей подпапке
- [ ] Endpoints `/graph/save-file`, `/graph/load-file` — серверный save/load
- [ ] Lazy load: рендер графа по edges + тексты для лейблов, полный узел по клику
- [ ] Теги/слои вместо папок — узел может быть в нескольких слоях (Знания, Идеи, Действия)
- [ ] Git-friendly: diff читаем, каждая строка = один узел
- [ ] Obsidian-совместимо: конвертация node.jsonl ↔ .md

**Фаза 1 — SQLite (когда файлов станет >1000):**
- [ ] Миграция из файлов в SQLite (один файл БД, zero dependencies)
- [ ] sqlite-vss для векторного поиска (O(log n) вместо O(n²) cosine)
- [ ] Активный view — 50-200 узлов на экране, остальное подгружается по запросу
- [ ] Инкрементальный similarity — при добавлении узла считать только его связи

**Фаза 2 — DuckDB (аналитический движок, когда дойдём до Марков/Байес):**
- [ ] DuckDB читает тот же SQLite для тяжёлых запросов
- [ ] Random Walk, стационарное распределение — native колоночные операции
- [ ] Поиск паттернов ("12 раз за месяц пропустил обед") — аналитические агрегации
- [ ] Матрица transition_prob — колоночное хранение, 10-100x быстрее SQLite

**Масштаб:** SQLite нормально до 10000+ узлов для CRUD. DuckDB подключается для аналитики на 1000+ узлах.

### Самостоятельные фичи (не зависят от камней)

**Визуализация и навигация:**
- [ ] **3D граф** — режим визуализации для масштаба (100+ узлов), three.js/WebGL. 2D остаётся основным
- [ ] **2D проекция** — текущий SVG, основной режим
- [ ] **Навигация стрелками** — перемещение по графу клавиатурой. Выбор пути: ручной или автоматический (по наиболее вероятным сценариям Марков-цепи)
- [ ] **Constraint-based layout** — d3-force/dagre/ELK для размещения без пересечений
- [ ] **Flow/Free toggle** — двойной переключатель
- [ ] **Grid/card layout** — узлы как карточки по колонкам depth
- [ ] **Mind map layout** — авторасширение секторов
- [ ] **Режимы визуализации** — "Уверенность+новизна" (зелёный/красный/фиолетовый)

**Структура и поиск:**
- [ ] **Папки графа** — организация графов в иерархию (Знания, Идеи, Действия...) с подпапками. Основа для слоёв
- [ ] **Поиск по графу** — семантический поиск по узлам + вопросы по графу ("какие идеи связаны с X?", "что я думал про Y?")
- [ ] **Оптимизация структуры** — связи отдельно от текстов, индексы для быстрого поиска. Важно для масштабирования до 1000+ узлов
- [ ] **Хранение документов** — PDF/URL/заметка в узле. Граф = карта знаний с источниками
- [ ] **Проверка связанности** — 2-3 узла → "связаны ли, как, чего не хватает?" → узел-мост

**Генерация:**
- [ ] **Дуэль моделей** — один промпт → два разных LLM через API → сравнение в Studio, подсветка расхождений
- [ ] **Параллельные запросы** — Think 6 = 6 параллельных вызовов к серверу/API. 3-6x ускорение
- [ ] **Пересчёт ветки** — выбрать цепочку, ввести новые вводные, заменить
- [ ] **Энтропия в графе** — автонормализация по min/max текущего графа
- [ ] **Извлечение графа из текста** — вставляешь статью → модель строит граф сущностей

**Экспорт/импорт:**
- [ ] **Экспорт** — PNG/SVG/GraphML/DOT/markdown
- [ ] **Импорт/экспорт Obsidian** — md-файлы, wiki-links, кластеры = папки
- [ ] **Импорт Markdown** — парсинг заголовков/ссылок → граф

**Перспектива:**
- [ ] **Режим "Анти-паралич"** — 3 случайных узла → таймер 90 сек → микро-продолжение → ✓/✗. Дофаминовая петля
- [ ] **Помощь с параличом выбора** — визуализация вариантов + Марков + Байес
- [ ] **Симуляция диалога** — два агента с разными ролями, коллапс ищет консенсус
- [ ] **World model для агентов** — вход = сенсоры, выход = действия. Та же архитектура
