## Planned features

- **Next token в parallel/compare** — кнопка пошаговой генерации (по одному токену)
  вместо только полного Auto. Позволит наблюдать генерацию в реальном времени
  и останавливаться в интересных местах.

- **Варианты продолжения в step mode** — генерация нескольких вариантов (3–5)
  следующих N токенов, каждый в отдельном окошке. Клик по варианту — применяет его.
  По сути visual branching: видишь куда модель может пойти и выбираешь путь.

- **Diff-режим** — подсветка расхождений между двумя потоками в compare.
  Видно где именно тексты разошлись и какие фрагменты совпадают.

- **State branching** — сохранить KV cache → ветка А → откат → ветка Б
  без пересчёта промпта. Дерево генераций.

- **Гибридный режим** — parallel → выбрал лучший поток → переключился в step.
  Комбинация обзора и точного контроля.

- **Streaming diff моделей** — два квантования одной модели side-by-side,
  видно где Q4 теряет качество относительно Q8.

- **Batch benchmark** — N промптов × M конфигов → CSV со скоростью и метриками.
