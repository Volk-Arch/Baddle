# TODO & Ideas — Graph Thinking

## Концепция

Человек мыслит не линейно. Он набрасывает точки (факты, гипотезы, ассоциации),
проверяет можно ли их соединить, и если получается — углубляется. Если нет —
добирает новых точек или перестраивает связи.

Ни одна текущая архитектура это не воспроизводит:
- Авторегрессия — строго слева направо
- Диффузия — уточняет равномерно, без избирательности
- Tree of Thoughts — дерево, не граф (ветвится, но не соединяется обратно)

Цикл: генерация мыслей → граф связей → кластеризация → коллапс → повтор.

## Критичное

- [ ] Настоящие эмбеддинги вместо Jaccard — `embedding=True` при загрузке, cosine similarity. Jaccard по токенам не ловит семантическое сходство
- [ ] Стриминг мыслей — сейчас UI зависает пока генерируются все N мыслей. SSE для поочерёдного отображения
- [ ] Повторные/похожие мысли — фильтр по similarity к уже существующим

## Визуализация

- [ ] Force-directed layout — связанные узлы притягиваются, несвязанные отталкиваются. Вместо статичного круга
- [ ] Drag & drop узлов — перетаскивание для ручной группировки
- [ ] Анимация коллапса — узлы визуально сходятся в один
- [ ] Цвет узлов по энтропии — высокая энтропия = красный узел

## Интерактивность

- [x] Ручное добавление мыслей — поле ввода + кнопка Add
- [x] Ручное удаление мыслей — крестик у каждой мысли в списке
- [x] Выбор кластера для коллапса — отдельная цветная кнопка для каждого кластера
- [x] Ручное пересоединение узлов — клик на узел → клик на второй → toggle ребра. Ручные рёбра (пунктир) сохраняются при пересчёте
- [ ] Редактирование мыслей — клик → изменить текст → пересчитать связи
- [ ] Undo — откат последнего действия

## Алгоритмы

- [x] Многоуровневый коллапс — результат коллапса становится новым узлом, граф пересчитывается
- [ ] Управляемая генерация — "сгенерируй мысль, связанную с этим узлом" (направленный добор точек)
- [ ] Оценка связности — метрика "готовности к коллапсу" для всего графа
- [ ] Лучшая кластеризация — spectral clustering или DBSCAN вместо connected components

## UX

## Benchmark mode

- [ ] Автоматический прогон: список тем → генерация N мыслей по каждой → подсчёт метрик
- [ ] Метрики: количество кластеров, средняя уникальность мыслей, diversity score (доля несвязанных узлов), скорость насыщения (на какой мысли модель начинает повторяться)
- [ ] Сравнение моделей: одни и те же темы, разные GGUF — таблица с результатами
- [ ] Сравнение квантований: Q4 vs Q8 vs F16 — как квантование влияет на diversity
- [ ] Экспорт отчёта: JSON + markdown с графиками

- **A/B тестирование промптов** — compare mode, фиксированный seed
- **Точки бифуркации** — где temp/top_k меняют выбор модели
