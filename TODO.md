## Фичи

### 7 режимов работы (один движок, разные ритмы)

Один tick(), один граф, одна логика — три координаты определяют ритм:
- **Цель**: дискретная (стоп) / непрерывная (вектор)
- **Данные**: пассивные / активные / смешанные
- **Выход**: внутреннее обновление / внешний артефакт

| # | Режим | Цель | Автономность | Стоп-условие | Статус |
|---|-------|------|-------------|-------------|--------|
| 1 | **Исследование** | Понять тему, найти связи | 70% | confidence > 0.85 | ✅ Работает |
| 2 | **Вектор** | Держать состояние/привычку | 80% | Бесконечный heartbeat | ⬜ |
| 3 | **Создание** | Собрать артефакт (эссе/план/код) | 50% | structure_complete + ✅ | ⬜ |
| 4 | **Выбор** | Принять решение | 60% | max_delta < threshold | ⬜ |
| 5 | **Диагностика** | Найти корень проблемы | 65% | root_cause.conf > 0.8 | ⬜ |
| 6 | **Обучение** | Освоить навык | 55% | mastery > 0.85 на ключевых | ⬜ |
| 7 | **Рефлексия** | Увидеть паттерны | 75% | pattern.conf > 0.75 | ⬜ |

**Режим 1: Исследование** — текущий цикл generate→merge→elaborate→doubt→META. Работает, сходится за 28-43 шага.

**Режим 2: Вектор** — фоновый мониторинг состояния. Heartbeat раз в N часов. Обновление confidence привычки, рост β при пропусках. Бесконечный цикл.

**Режим 3: Создание** — конвергентный: каркас→детали→шлифовка. Узлы сворачиваются в артефакт, связи укрепляются (α+=2). Нужно ручное подтверждение.

**Режим 4: Выбор** — параллельные ветви для каждого варианта. Рёбра с expected_utility. Collapse в решение с confidence. Сценарии "что если".

**Режим 5: Диагностика** — обратный поиск от следствия к причине. Отсечение ложных веток (β рост). Корень = fact с высоким α.

**Режим 6: Обучение** — skill_node с mastery_level, prerequisite рёбра, decay для забытого. Адаптивные задачи, проверка усвоения.

**Режим 7: Рефлексия** — мета-узлы pattern, поиск повторений→группировка→абстрагирование→инсайт. Зеркальный цикл.

**Автоопределение режима** — по промпту/намерению, без ручного переключения. Или селектор в UI.

### UI: 1 граф = 1 режим

Выбираешь режим → input-поле заменяется формой под режим:

| Режим | Поля ввода |
|-------|-----------|
| Исследование | `[Тема/вопрос]` |
| Создание | `[Тема]` + `[Формат: эссе/план/код]` + `[Требования]` |
| Выбор | `[Вариант A]` `[Вариант B]` `[Вариант C]` + `[Критерии]` |
| Диагностика | `[Проблема/симптом]` + `[Контекст]` |
| Обучение | `[Навык]` + `[Текущий уровень]` |
| Рефлексия | `[Период/фокус]` + `[Действия/решения]` |
| Вектор | `[Целевое состояние]` + `[Метрика]` + `[Интервал]` |

Каждый режим создаёт свой тип goal-ноды с нужными полями. Tick читает тип goal и подбирает ритм цикла.

### Источники данных

Три источника для generate/elaborate/doubt — выбираются per-режим или per-граф:

| Источник | Что делает | Когда |
|----------|-----------|-------|
| **Нейронка (local/API)** | Генерирует из весов модели | Творческие задачи, синтез, рефлексия |
| **Интернет (поиск/RAG)** | Ищет факты, статьи, данные | Исследование, диагностика, обучение |
| **Гибрид** | Нейронка генерит гипотезу → интернет проверяет фактами | Исследование с фактчекингом |

- Для режима **Исследование** — интернет критичен: нейронка придумывает, поиск подтверждает/опровергает
- Для **Создание/Рефлексия** — достаточно нейронки
- Для **Диагностика/Обучение** — гибрид: нейронка строит дерево причин, интернет даёт референсы

### Выбор модели per-задача

Каждый этап цикла может использовать свою модель:

| Этап | Модель | Зачем |
|------|--------|-------|
| Generate | Локальная 8B | Быстро, дёшево, массово |
| Elaborate | Локальная 8B | Углубление не требует мощности |
| Doubt (SmartDC) | API (GPT-4o/Claude) | Качественный антитезис требует сильной модели |
| Collapse (синтез) | API или локальная | Зависит от требований к качеству |
| Final essay | API | Длинная связная генерация |

Настраивается в UI: per-этап выбор local/API/auto.

### Transition model — переходы между типами нод

Каждый тип ноды имеет вероятности перехода к следующему типу. Tick может использовать это для автовыбора действия:

```
goal ──► question ──► hypothesis ──► evidence ──► fact
  │         │              │              │         │
  │         ▼              ▼              ▼         ▼
  └───── thought ◄─────────┘              └────► action ◄─┘
              │                                 │
              └──────────── feedback ───────────┘
```

| Переход | Вероятность | Логика |
|---------|------------|--------|
| question → hypothesis | 0.6 | Естественный ответ на вопрос |
| question → thought | 0.3 | Рефлексия вместо ответа |
| hypothesis → evidence | 0.7 | Валидация требует данных |
| hypothesis → action | 0.2 | Проверка через практику |
| evidence → fact | 0.8 | Если вес достаточен |
| fact → action | 0.6 | Опора на знание → шаг |
| action → hypothesis | 0.4 | Обратная связь → новые допущения |
| thought → question | 0.5 | Мысль рождает уточнение |

### Flow patterns — паттерны потока per-режим

| Паттерн | Режимы | Как работает |
|---------|--------|-------------|
| **Дивергентный** | Исследование | Расширяет ветки, ищет пробелы |
| **Конвергентный** | Создание | От каркаса к деталям, сужение |
| **Циклический** | Обучение, Вектор | Практика→проверка→закрепление→повтор |
| **Параллельный** | Выбор | Одновременный прогон сценариев |
| **Обратный** | Диагностика | От следствия к причине |
| **Зеркальный** | Рефлексия | Поиск повторений в истории |

Сейчас реализован только дивергентный (исследование). Каждый паттерн = другая стратегия в tick.

### Мета-режим: А → Б

Пользователь задаёт только точку Б. Система:
1. Находит точку А (последний активный узел, контекст, или корень)
2. Автоматически подбирает ритм по трём координатам
3. Ведёт к цели, адаптируя шаг за шагом
4. Останавливается когда уверенность в достижении цели > порога

Управление намерением, не координатами.

### Конкретные обновления графа per-режим

| Режим | Типы нод | Рёбра | α/β |
|-------|---------|-------|-----|
| Исследование | thought, hypothesis | similarity, directed | α+β рост на проверенных |
| Вектор | habit_node с confidence | temporal | β рост при пропусках |
| Создание | section, draft | structure | α+=2 при укреплении |
| Выбор | hypothesis per вариант | expected_utility | α/β per сценарий |
| Диагностика | hypothesis → evidence → fact | causal | β рост = ложная ветка |
| Обучение | skill_node с mastery_level | prerequisite | decay для забытого |
| Рефлексия | pattern (мета-узел) | occurs_in | confidence инсайта |
