# baddle

> Не спрашивай у ИИ ответ — смотри, как он его строит, и управляй процессом.

**[English version →](README_EN.md)**

Инструмент для экспериментов с генерацией текста нейросетью.
Позволяет заглянуть внутрь процесса — не просто получить ответ, а управлять им пошагово.

Большинство LLM-интерфейсов — чёрный ящик: отправил промпт, получил текст.
**baddle** открывает этот ящик: можно идти токен за токеном, смотреть распределение вероятностей,
редактировать текст прямо в окне генерации, сравнивать как один и тот же промпт
ведёт себя при разных настройках — и всё это в браузере.

С флагом `--server` baddle использует нативный `llama-server` из llama.cpp,
который умеет обрабатывать **два промпта одновременно** на GPU — оба генерируются
параллельно и занимают примерно столько же времени, сколько один.

## Режимы

### `step` — пошаговая генерация

![Step mode](images/step.jpg)

**Step mode** — главный исследовательский режим.
Модель генерирует **один токен за раз**. После каждого токена видно
распределение вероятностей (top-10), можно сменить температуру и top_k
и продолжить генерацию.

Текст в окне генерации — **редактируемый**. Кнопка `Edit` включает режим
редактирования: можно дописать текст (inject), обрезать лишнее, исправить
любой фрагмент — и нажать `Sync` чтобы модель подхватила изменения.

Токены подсвечиваются **heatmap уверенности** — зелёный (модель уверена),
жёлтый, красный (высокая энтропия, модель гадает).

---

### `parallel` — два промпта одновременно

![Parallel mode](images/parallel.jpg)

Два разных промпта генерируются параллельно. Результат — live split-screen,
оба потока обновляются в реальном времени.

Интересно давать противоположные фреймы — модель с одинаковой
уверенностью обосновывает обе позиции. Или два одинаковых промпта с temp>0 —
расходятся уже на 1–3 токене, первый случайный токен задаёт всю траекторию.

---

### `compare` — один промпт, два набора параметров

![Compare mode](images/compare.jpg)

Один промпт, но **два разных конфига** (температура, top_k). Оба потока стартуют
из идентичных токенов и расходятся как только параметры дают разные выборки.
Интерфейс показывает точный шаг расхождения.

На академическом тексте расхождение наступает поздно,
на художественном — сразу. Это токен, где распределение было достаточно широким
чтобы разные настройки дали разный выбор.

---

### `chat` — разговор с моделью

![Chat mode](images/chat.jpg)

Чат с моделью через chat template (ChatML / Jinja2). Настраиваемые роли
(system prompt), температура, лимит токенов. Кнопка **Continue** позволяет
догенерировать ответ если он обрезался по лимиту.

Ответы подсвечиваются heatmap уверенности — видно где модель уверена
в ответе, а где угадывает.

---

### `graph` — графовое мышление

Экспериментальный режим нелинейной генерации. Модель набрасывает короткие мысли
по теме, между ними строится граф связей (Jaccard similarity по токенам),
связанные мысли образуют кластеры.

Кластер можно **коллапсировать** — модель синтезирует связанные мысли в связный
абзац. Результат коллапса становится новым узлом в графе — и процесс можно
повторять, углубляясь на каждом уровне.

Цикл: генерация → фильтрация → кластеризация → коллапс → повтор.
Ни один чат-интерфейс этого не даёт — ты не разговариваешь с моделью,
а думаешь вместе с ней.

Работает только в in-process режиме (без `--server`).

---

### Гибридный режим: parallel/compare → step

После завершения parallel или compare генерации — кнопка **→ Step** у каждого
потока. Переключает в step mode с сохранением текста и KV cache выбранного
потока. Можно продолжить генерацию токен за токеном с полным контролем.
Работает только в in-process режиме (без `--server`).

---

### Общие возможности

- **Heatmap уверенности** — во всех режимах (step, parallel, compare, chat)
  токены окрашиваются по энтропии: зелёный → жёлтый → красный.
  Видно где модель знала ответ, а где гадала.
- **Роли** — пресеты из `roles.json`, общие для всех режимов. В step/parallel/compare
  используются как prefix, в chat — как system message.
- **Seed** — для воспроизводимости. С одинаковым seed и параметрами модель
  даёт идентичный результат. Seed `-1` — случайный. Доступен в parallel и compare.
- **Счётчик токенов** — показывает использованные / доступные токены контекста.
