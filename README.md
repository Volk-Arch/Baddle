> AHI Protocol — Augmented Human Intelligence.

**[English version](README_EN.md)** | **[Установка](SETUP.md)** | **[Видение](VISION.md)** | **[TODO](TODO.md)**

Граф мыслей с байесовской уверенностью, марковскими переходами и диалектическим синтезом. Задаёшь цель, нажимаешь Run — система автоматически строит путь к ней. Локально через llama.cpp или через API. Бесплатно, на квантизированной 8B модели.

## Концепция: алгебра мышления

Любая задача сводится к трём вопросам:

1. **Сколько целей?** — 0, 1 или несколько
2. **Какой оператор между ними?** — четыре примитива: `none / AND / OR / XOR`
3. **Какой тип цели?** — finite (есть финал), repeatable (ритм), open (бесконечное уточнение)

Четыре примитива + стратегии их применения + три типа целей дают **12 режимов мышления** — полный базис, без натяжек.

### Четыре примитива

| Примитив | Суть | Формально |
|----------|------|-----------|
| **none** | Нет целей | — |
| **AND** | Все цели обязательны | Все подзадачи должны завершиться |
| **OR** | Любая одна достаточна | Первая завершённая завершает задачу |
| **XOR** | Ровно одна из нескольких | OR + принудительное исключение остальных |

Всё остальное — это **стратегии** применения примитивов:

- **AND** может обходиться в любом порядке (Конструктор), строго по зависимостям (Конвейер = SEQ), по приоритету (Каскад = PRIORITY), или с пропорциональной аллокацией над нефинитными целями (Весы = BALANCE).
- **XOR** может разрешаться через сравнение независимых опций (Турнир) или через диалектический синтез противоречивых утверждений (Диспут).
- **OR** различается по типу цели — finite (Гонка) или open (Веер).
- **Один goal** различается по типу — finite (Вектор), repeatable (Ритм), open (Горизонт).

### 12 режимов

| Слой | Режим | Примитив | Стратегия / тип | Пример |
|------|-------|----------|----------------|--------|
| **0 целей** | Разведка (Scout) | none | — | Прогулка без маршрута, серфинг |
| **1 цель** | Вектор (Vector) | focus | finite | Сдать проект, вылечить зуб |
| | Ритм (Rhythm) | focus | repeatable | Завтрак, зарядка |
| | Горизонт (Horizon) | focus | open | Изучение языка |
| **AND** | Конструктор (Builder) | AND | неупорядоченный | Сборка мебели |
| | Конвейер (Pipeline) | AND | SEQ — по зависимостям | Рецепт, чек-лист |
| | Каскад (Cascade) | AND | PRIORITY — по важности | Тайм-менеджмент |
| | Весы (Scales) | AND | BALANCE — пропорции | Бюджет, компромисс |
| **OR** | Гонка (Race) | OR | finite — первая достаточна | Любое свободное такси |
| | Веер (Fan) | OR | open — набор идей | Мозговой штурм |
| **XOR** | Турнир (Tournament) | XOR | comparative — сравнение | Одна машина из трёх |
| | Диспут (Dispute) | XOR | dialectical — синтез | Научная дискуссия, суд |

> **Статус:** сейчас реализован только Горизонт / Веер (исследовательский цикл). Остальные режимы — в [TODO](TODO.md).

### Типы целей и стоп-условия

Стоп зависит от **типа цели**, не от режима. Режим определяет *как идти*, тип цели — *когда остановиться*.

| Тип | Суть | Стоп за итерацию | Состояние после |
|-----|------|------------------|----------------|
| **finite** | Есть финал | `confidence ≥ threshold` | RESOLVED → архив |
| **repeatable** | Ритм/привычка | `step_complete + trigger` | SCHEDULED → ждёт триггера |
| **open** | Бесконечное уточнение | `diminishing_returns ИЛИ budget` | PARKED → ждёт новых данных |

### Иерархия графов

- **1 граф = 1 задача** (1 режим)
- Связи между графами хранятся в **отдельном мета-графе**

> **Статус:** сейчас один граф. Мульти-граф и мета-граф — в TODO.

## Цикл мышления (Tick)

`thinking.py` — движок автономного мышления. Моделирует то, как думает человек: набросать идей, сгруппировать похожее, углубить уникальные, усомниться, повторить.

### Четыре инструмента, четыре фазы

```
GENERATE    мало идей?            -> Think (набрать массу)
MERGE       есть похожие?         -> Collapse (слить ДО углубления)
ELABORATE   есть голые?           -> Elaborate (добавить evidence)
DOUBT       есть непроверенные?   -> Smart DC (усомниться)
GENERATE+   всё проверено?        -> Think ("что упустил?", с контекстом)
SYNTHESIZE  ничего нового         -> Stable (финальный результат)
```

### Почему этот порядок

1. **GENERATE** первый — нужна масса разных идей прежде чем работать с ними
2. **MERGE** до elaborate — не тратим работу на дубликаты. "8B дешевле" и "8B снижает затраты" сливаются в одну ноду ДО того, как обе получают evidence
3. **ELABORATE** до doubt — голая фраза проверяется хуже чем углублённая
4. **DOUBT** последний — SmartDC работает качественнее на развитых идеях
5. **GENERATE+** (META) — после проверки всего ищем пробелы

### Сходимость цикла

Цикл сходится естественно через три механизма:
- **Novelty check** — embedding similarity > 0.92 → мысль отклоняется. Когда модель исчерпала тему → сплошные отклонения → стоп
- **Lineage tracking** — каждая collapse-нода хранит `collapsed_from`. Merge не сольёт ноду с её предком — не перемалывает одно и то же
- **Phase completion** — каждая фаза отрабатывает до конца перед следующей

## Операции с графом

### Think
Генерирует N коротких независимых мыслей по теме. Каждая мысль — отдельная попытка. LLM автоматически классифицирует тип и начальную уверенность.

### Elaborate
Углубление конкретной мысли. Дочерние узлы автоматически становятся **evidence** — LLM определяет, поддерживают они родителя (supports) или противоречат (contradicts). Bayes update меняет confidence родителя.

### Expand
Ветвление от мысли в новом направлении. Не углубление, а другой ракурс.

### Collapse (Merge)
Синтез кластера мыслей в один узел. Из 5 отдельных идей — один связный абзац. Два режима: merge (удаляет исходные) и no-merge (добавляет синтез как дочерний).

### Rephrase
Переформулировка мысли. Сохраняет суть, меняет форму.

### Ask
Генерирует probing-вопрос к выбранной ноде — challenge или hidden assumption. Добавляется как дочерний узел типа question.

### Verify (Smart DC)
Диалектическая проверка — ключевой механизм. Три параллельных генерации:
1. **Тезис** — сильнейший аргумент ЗА
2. **Антитезис** — сильнейший аргумент ПРОТИВ
3. **Нейтраль** — контекст и условия

Три текста → три вектора в embedding-пространстве → **центроид** (точка равновесия) → синтез, максимально близкий к центроиду. Confidence = cosine similarity синтеза к центроиду. Принудительный антитезис ломает эхо-камеры.

### Walk
Random Walk по графу. Использует transition probabilities на рёбрах. Показывает куда ведёт мысль.

## Математическая база

### Байесовская уверенность

Каждый узел имеет **confidence** (0-1):

```
P(H|E) = P(E|H) * P(H) / [P(E|H)*P(H) + P(E|~H)*(1-P(H))]
```

При добавлении evidence (supports/contradicts) confidence обновляется через Bayes. Alpha/Beta модель отслеживает баланс поддерживающих и противоречащих свидетельств.

### Марковские переходы

Каждое ребро имеет **transition_prob**, нормализованное по исходящим:
- Косинусное сходство эмбеддингов → базовый вес
- Directed edges получают bonus ×1.5
- Хебб: при навигации пользователя `tp += lr * (1 - tp)` — часто используемые пути усиливаются
- Детектор ловушек: узлы с высоким входящим и низким исходящим tp

### Кластеризация

Connected components на рёбрах выше threshold. Используется для группового collapse, визуализации hull, и определения изолированных узлов.

## Источники данных

Три источника для generate/elaborate/doubt — настраиваются per-режим или per-этап:

| Источник | Когда использовать |
|----------|-------------------|
| **LLM** (local 8B / API) | Генерация из весов модели — творческие задачи, синтез, рефлексия |
| **Интернет** (search/RAG) | Факты, статьи, данные — исследование, диагностика |
| **Гибрид** | LLM генерит гипотезу → поиск проверяет фактами |

Выбор модели per-этап: local 8B для generate (быстро), API для doubt (качественный антитезис), API для final essay (длинная связная генерация).

## Интерфейс

Светлая тема (Notion-style). Два режима: **Graph** (основной) и **Chat**.

### Graph mode
- **Topic + Think/Add** — ввод темы, генерация мыслей
- **Save/Load** — сохранение графов, автосохранение
- **Actions bar** — Select, Collapse, Link, Flow, Layout, Run, Undo, Save, Reset
- **SVG визуализация** — drag, zoom, pan, цветовая кодировка по типу и уверенности
- **Detail panel** — heatmap по токенам, confidence, тип, alpha/beta, Walk, Verify, Ask, Rephrase
- **Контекстное меню** — Expand, Elaborate, Rephrase, Verify, Ask, Walk, Evidence, Chat, Edit, Delete
- **Generation Studio** — пакетные варианты для rephrase/elaborate/expand/collapse/ask

### Chat mode
Обычный диалог с моделью. Context sidebar для связи граф ↔ чат (отправить узлы как контекст, сохранить ответ как мысль). Continue для догенерации обрезанных ответов.

### Настройки Run

| Параметр | Default | Описание |
|----------|---------|----------|
| **Start ideas** | 5 | Сколько идей сгенерить перед началом цикла (3-20) |
| **Depth** | 2 | Раунды META "что упустил?" (1-10) |
| **Stable** | 0.8 | Порог confidence для verified |
| **Collapse at** | 50 | Шаг принудительного collapse |
| **Essay tokens** | 6000 | Максимум токенов финала (500-30000) |
| **Infinite** | off | Без лимита шагов — цикл сам найдёт точку остановки |
| **Batched essay** | off | Пирамидальный синтез: секции → финальный текст |
| **Output** | essay | Формат финала: essay / brief / list / none |

### Live-метрики

Во время Run в углу графа отображается оверлей: Step / Phase / Hyp / Verified / Avg + sparkline кривой сходимости.

## Быстрый старт

```bash
git clone https://github.com/Volk-Arch/Baddle.git
cd Baddle
python setup.py        # pip install flask numpy
python ui.py           # откроет http://localhost:7860
```

Baddle использует OpenAI-совместимый API для генерации. Самый простой вариант — [LM Studio](https://lmstudio.ai): загрузи модель (Qwen3-8B), запусти локальный сервер, укажи URL в Settings. Подробнее: [SETUP.md](SETUP.md)
