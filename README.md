## Что это

Задаёшь тему → нажимаешь Run → система думает сама: генерирует гипотезы, сомневается в каждой, синтезирует похожие, ищет пробелы, повторяет. Результат — структурированный текст, проработанный с разных сторон, с контраргументами.

На любой LLM через OpenAI-совместимый API (LM Studio, Ollama, OpenAI). Бесплатно, локально, на квантизированной 8B модели.

Пример: тема "Почему открытые LLM вытесняют проприетарные" → 28 шагов autorun → эссе с 5 верифицированными аргументами, контраргументами и сбалансированным выводом. Из одной фразы — без участия человека.

## Как работает

Граф мыслей с автономным циклом:

```
GENERATE    мало идей?            → Brainstorm (набрать массу)
MERGE       есть похожие?         → Collapse (слить ДО углубления)
ELABORATE   есть голые?           → Elaborate (добавить аргументы)
DOUBT       есть непроверенные?   → Smart DC (тезис/антитезис/синтез)
META        всё проверено?        → "Что упустил?" (ещё раунд)
SYNTHESIZE  ничего нового         → Финальный текст
```

Каждая мысль — нода с числовой уверенностью (0–1). Связи — по смысловой близости. Цикл останавливается сам, когда модель начинает повторяться.

| В голове | В графе |
|---|---|
| Интуиция | `confidence: 0.73` |
| Ассоциации | Рёбра + `transition_prob` |
| Сомнения | Низкий `evidence_count` |
| "За и против" | Smart DC → синтез |
| История | Timestamps, lineage |

## Почему работает: [байесовский конус](docs/cone-design.md) (дизайн-метафора)

Есть глобальное **поле возможностей** — всё что LLM может сгенерировать. Без ограничений — шум.

Человек — **точка** в этом поле. Мысли — **конус из этой точки**: веер направлений куда можно двигаться. Baddle управляет конусом, не точкой.

```
Широкий конус (conf=0.3)    Узкий конус (conf=0.9)    Повёрнутый конус
     ╱    ╲                      │╲                         ╱│
    ╱      ╲                     │ ╲                       ╱ │
   ╱ много  ╲                    │одна╲                   ╱  │
  ╱ гипотез  ╲                   │верс.╲                другой│
 ╱            ╲                  │     ╲               аспект │
```

Все инструменты — операции с конусом:

- **Генеративные** (Brainstorm, Expand, Elaborate) — расширение конуса
- **Проверочные** (Smart DC, Ask) — сужение конуса
- **Структурные** (Collapse, Rephrase, META) — фиксация и уплотнение
- **Навигационные** (Walk, Pump) — поиск путей

| Категория | Инструмент | Операция | Что происходит |
|-----------|-----------|----------|---------------|
| **Генеративные** | Brainstorm | Расширение | Больше направлений из текущей точки |
| | Expand | Ветвление | Новый ракурс от той же точки |
| | Elaborate | Удлинение | Глубже вдоль одного направления |
| **Проверочные** | Smart DC | Сужение | Отсечение невозможного, рост confidence |
| | Ask | Просвечивание | "А точно ли дно конуса твёрдое?" |
| **Структурные** | Collapse | Фиксация | Из веера → одна точка, перемещение |
| | Merge | Уплотнение | Два похожих направления → одно сильное |
| | Rephrase | Перенаправление | Тот же вектор, другая траектория |
| **Навигационные** | Walk | Обход | Куда ведёт мысль по transition probabilities |
| | META | Переоценка | "Конус направлен туда?" → расширение если нет |
| **Поисковые** | Pump | Поворот | Новая ось, невидимая изнутри конуса |

### Prediction error — ключевой механизм

Мозг **всегда ожидает чуть больше** чем получает (predictive processing). Это несоответствие (`ε > 0`) — один из ключевых двигателей внимания, обучения, любопытства.

В Baddle: `target_surprise` в CognitiveHorizon:
- `surprise < target` → "предсказуемо" → конус расширяется → ищем удивление
- `surprise > target` → "хаотично" → конус сужается → стабилизируемся
- `surprise ≈ target` → зона потока → система движется вперёд

### Замкнутый контур

Каждый компонент усиливает остальные:

```
prediction error → precision → temperature + top_k + novelty_threshold →
→ разнообразие идей → диалектическая проверка (Smart DC) →
→ confidence → surprise → prediction error
```

Precision управляет **всем сразу**: temperature (креативность генерации), top_k (количество кандидатов), novelty_threshold (строгость фильтра повторов). Один параметр — три эффекта:

- EXPLORATION (precision ↓): высокая temp, большой top_k, мягкий novelty → широко ищем
- EXECUTION (precision ↑): низкая temp, малый top_k, строгий novelty → фокусируемся

Идеи проверяются диалектикой. Диалектика даёт confidence. Confidence даёт surprise. Surprise обновляет prediction error → precision корректируется → все параметры сдвигаются. Один замкнутый контур.

### Операции подробнее

*Генеративные (расширение конуса):*

**Brainstorm** — генерация N независимых коротких мыслей от выбранной ноды. Каждая — отдельная попытка, не продолжение предыдущей. LLM автоматически классифицирует тип (hypothesis, fact, question) и начальную уверенность. Novelty check отсекает повторы.

**Expand** — ветвление от мысли в новом направлении. Не углубление (как Elaborate), а другой ракурс. Создаёт sibling, не child.

**Elaborate** — углубление конкретной мысли. Раскрывает деталь, следствие или механизм. Дочерние узлы становятся **evidence** — LLM определяет, поддерживают они родителя (supports) или противоречат (contradicts). Bayes update меняет confidence.

*Проверочные (сужение конуса):*

**[Smart DC](docs/smartdc-design.md) (Dialectical Compass)** — диалектическая проверка. Три параллельных генерации: сильнейший аргумент ЗА, сильнейший ПРОТИВ, нейтральный контекст. Три текста → три вектора в embedding-пространстве → центроид (точка равновесия) → синтез ближайший к центроиду. Принудительный антитезис — модель не может просто согласиться. Per-pole analysis: каждый полюс получает свой confidence, **lean** показывает куда склоняется (thesis vs antithesis), **tension** — насколько глубокий спор.

**Ask** — генерирует probing-вопрос к ноде: "какое скрытое допущение?", "при каких условиях это ломается?". Как хороший интервьюер — находит то что автор не проговорил. Полезен как подготовка к SmartDC (вопрос подсказывает направление антитезиса).

*Структурные (фиксация и уплотнение):*

**Collapse (Merge)** — синтез группы мыслей в одну. Из 5 идей — один связный абзац. Lineage tracking (`collapsed_from`) не даёт перемалывать одно и то же.

**Rephrase** — переформулировка мысли, суть та же. Критически важен когда: (1) нейросеть застряла — начинает каждую мысль одинаково, rephrase ломает паттерн; (2) формулировка слабая — SmartDC даёт низкий confidence из-за кривых слов, не из-за плохой идеи.

**META** — "что я упустил?" с контекстом уже верифицированного. Ищет пробелы в покрытии темы.

*Навигационные и поисковые:*

**Walk** — Random Walk по графу через transition probabilities (tp) на рёбрах.

Каждое ребро имеет вес — вероятность перехода. Три источника веса:
- **Embedding similarity** — базовый: чем ближе по смыслу, тем сильнее связь
- **Directed edges** — бонус ×1.5: parent→child связи весят больше
- **Хеббовское обучение** — `tp += lr × (1 - tp)` при навигации пользователя: чем чаще ходишь по связи, тем она сильнее. Граф запоминает твоё внимание

Walk запускает случайное блуждание из выбранной ноды и показывает куда мысль **естественно ведёт** — какие узлы оказываются конечными чаще всего. Аналог attention в нейросетях: из одной точки → распределение весов по соседям → фокус. Но в отличие от attention, веса дообучаются в реальном времени от навигации пользователя (Хебб).

Влияние на цикл: transition probabilities определяют **какие связи сильные** в графе. Tick использует это при выборе следующей ноды для обработки (BFS к goal через взвешенные рёбра). Детектор ловушек (`_detect_traps`) находит ноды с высоким входящим tp но низким исходящим — "чёрные дыры внимания" куда мысли втекают но не вытекают.

**Pump (Накачка)** — поиск пересечений в семантическом пространстве. Берёт две несвязанные идеи и **накачивает область каждой**: ассоциации, аспекты, следствия. Два облака растут итеративно. LLM-абстрактор ищет **скрытые параметры** — оси, на которых обе идеи оказываются точками. Каждая итерация расширяет облака и ищет новые оси, не повторяя уже найденные. Мосты накапливаются через итерации и ранжируются по качеству. Каждый мост верифицируется через SmartDC с контекстом обеих идей. Метрики качества:
- **lean** — куда склоняется (thesis vs antithesis). Баланс = интересный мост
- **tension** — насколько далеки тезис и антитезис. Низкая = глубокий спор
- **quality** = balance × depth × DC confidence

При сохранении — выбор: краткая связь (bridge) или полный синтез SmartDC.

Пример: "зимой холодно" + "летом холодно" → *"ощущение неожиданности и контраста"*, *"эмоциональная связь с ожиданием перемен"*, *"климат как общий параметр нестабильности"*.

## Научные основы

Каждый компонент — реализация концепции из нейробиологии. Не математически строго — та же интуиция, рабочее решение.

| Концепция | Компонент | Что делает |
|-----------|-----------|-----------|
| **Active Inference** (Фристон) | CognitiveHorizon  | Precision → temperature. Surprise → расширить/сузить конус. Состояния: EXPLORATION → EXECUTION → RECOVERY → INTEGRATION |
| **Байесовский мозг** | Confidence + Bayes | Каждая нода = гипотеза 0–1. Evidence обновляет: `P(H\|E) = P(E\|H)×P(H)/P(E)`. Никогда не 1.0 |
| **Диалектика** (Гегель) | Smart DC | Три генерации (за/против/контекст) → embedding centroid → синтез. Принудительный антитезис |
| **Хебб** | Transition prob | `tp += lr × (1-tp)` при навигации. Часто используемые пути усиливаются |
| **Дивергенция/конвергенция** (Гилфорд) | Generate ↔ Collapse | Brainstorm = расширение, Merge = сужение. Цикл повторяется, каждый раз абстракция выше |
| **Рабочая память** (Миллер, 7±2) | Collapse | 50 нод → 5 синтезов. Не помнишь детали — помнишь выводы |
| **Хабитуация** | [Novelty check](docs/novelty-design.md) | Embedding sim + rephrase-before-reject + адаптивный threshold через precision. Модель повторяется → стоп |
| **Метапознание** | META + overlay | "Что упустил?" Sparkline показывает как думает система |
| **System 2** (Канеман) | Весь инструмент | LLM в чате = System 1. LLM в Baddle = System 2. Та же модель, другой процесс |
| **Семантические сети** (Куиллиан) | Граф + Walk + Pump | Ноды = понятия, рёбра = cosine similarity, Walk = activation, Pump = поиск скрытой оси |

### Конвергенция, не дизайн

Эта связность не была спроектирована. Конус — из онтологии. Horizon — из проблемы фиксированной температуры. 4 примитива — из избыточности 8 операторов. Pump — из вопроса "как найти связь". Каждый компонент — ответ на конкретную проблему, и потом оказалось что он описан в науке 30 лет назад.

## CognitiveHorizon — адаптивный контроллер

CognitiveHorizon (`src/horizon.py`) — контроллер между tick и LLM. Не генерирует контент — управляет **как** генерировать. Без него tick использовал бы фиксированные параметры на все фазы — как ехать на одной передаче. Три ручки:

**Precision (Π)** — насколько система уверена в текущей модели мира (0–1). Управляет шириной конуса:

```
precision 0.3 → temperature 0.7, top_k 73   (широко ищем, креативно)
precision 0.5 → temperature 0.5, top_k 55   (баланс)
precision 0.8 → temperature 0.2, top_k 28   (узкий фокус, точность)
```

**Policy weights (Λ)** — какую фазу выбрать на этом шаге. Не жёсткий if/else, а веса:
`{generate: 0.3, merge: 0.2, elaborate: 0.2, doubt: 0.3}`. Фазы которые дают результат → их вес растёт. Которые не помогают → падает.

**Context frame (Γ)** — активный промпт и правила. Переключается когда novelty высокая (модель выдала что-то непохожее на предыдущее → сменить рамку).

### Обратная связь

После каждого шага Horizon обновляется:

```
surprise = 1 - confidence_после_SmartDC

Если surprise > target → "слишком хаотично" → precision ↑ → конус сужается
Если surprise < target → "слишком предсказуемо" → precision ↓ → конус расширяется
```

`target_surprise` > 0 всегда. Система **хочет** чтобы реальность чуть не совпадала с ожиданием — иначе зачем продолжать думать.

### Четыре состояния

| Состояние | Precision | Когда | Что делает |
|-----------|-----------|-------|-----------|
| **EXPLORATION** | 0.3–0.5 | Мало идей, начало, META | Широко ищет, high temperature |
| **EXECUTION** | 0.7–0.9 | Проверка, финальный синтез | Узкий фокус, low temperature |
| **RECOVERY** | 0.4→0.6 | Surprise spike, что-то не так | Расширяет обратно, ищет другой путь |
| **INTEGRATION** | 0.5–0.6 | Merge, Elaborate, данные собраны | Сбалансированный синтез |

Переходы автоматические: мало идей → EXPLORATION, всё verified → EXECUTION, surprise spike → RECOVERY, данные есть → INTEGRATION.

### Связь с режимами

Каждый из 13 режимов = preset для Horizon:

| Режим | Начальная precision | Policy акцент | Target surprise |
|-------|-------------------|--------------|----------------|
| Мозговой штурм | 0.3 (широко) | generate 0.5 | 0.5 (много удивления) |
| Исследование | 0.4 | balanced | 0.3 |
| Фокус | 0.7 (узко) | doubt 0.5 | 0.15 (мало удивления) |
| Выбор | 0.7 | compare 0.5 | 0.15 |

Один движок, 13 настроек. Не 13 отдельных алгоритмов — один Horizon с разными пресетами.

## 13 режимов мышления

Любая задача = три вопроса:
1. **Сколько целей?** — 0 / 1 / несколько
2. **Как они связаны?** — все нужны (AND) / любая подойдёт (OR) / выбрать одну (XOR)
3. **Есть ли финал?** — да / повторяется / бесконечно уточняется

| Режим | Что делаешь | Когда использовать | Стоп |
|-------|-----------|-------------------|------|
| **Свободный** | Работаешь руками, без автопилота | Хочешь сам управлять каждым шагом | Сам решаешь |
| **Блуждание** | Идёшь куда глаза глядят | Нет цели, ищешь неожиданное | Остановишь сам |
| **Фокус** | Одна задача → довести до конца | "Сдать проект", "написать статью" | Цель достигнута |
| **Привычка** | Делать регулярно, отслеживать streak | "Зарядка каждый день", "3 приёма пищи" | Никогда (streak) |
| **Исследование** | Изучать тему вглубь и вширь | "Разобраться в экономике", "понять тему" | Модель исчерпалась |
| **Сборка** | Все части обязательны, порядок неважен | "Собрать мебель", "подготовить документы" | Всё готово |
| **По шагам** | Строго по порядку, шаг за шагом | "Рецепт", "чек-лист", "маршрут" | Последний шаг |
| **Приоритеты** | Всё нужно, но срочное первым | "Тайм-менеджмент", "очередь задач" | Всё выполнено |
| **Баланс** | Распределить ресурсы между целями | "Бюджет", "работа/семья/здоровье" | Никогда (snapshot) |
| **Любой вариант** | Нужен любой результат, неважно какой | "Найти такси", "любой свободный слот" | Первый найден |
| **Мозговой штурм** | Набросать максимум идей | "Придумать название", "варианты решения" | Идеи кончились |
| **Выбор** | Сравнить варианты, выбрать один | "Какую машину купить", "куда поступить" | Лидер определён |
| **Дебаты** | Столкнуть позиции, найти синтез | "Научный спор", "за и против" | Синтез найден |

### Как это устроено внутри

Под капотом — 4 примитива (`none / AND / OR / XOR`) × тип цели (`finite / repeatable / open`) + стратегии (SEQ, PRIORITY, BALANCE для AND; comparative, dialectical для XOR):

| Режим | Примитив | Стратегия | Конус |
|-------|----------|-----------|-------|
| Свободный | — | ручной | Управляется вручную |
| Блуждание | none | — | Сфера — все стороны равны |
| Фокус | focus | finite | Узкий → сужается до точки |
| Привычка | focus | repeatable | Пульсирующий — сужение ↔ расширение |
| Исследование | focus | open | Сужается медленно, никогда не до точки |
| Сборка | AND | unordered | Несколько параллельных, все сходятся |
| По шагам | AND | SEQ | Цепочка: следующий из предыдущего |
| Приоритеты | AND | PRIORITY | Разного размера, большие первыми |
| Баланс | AND | BALANCE | Два+ с общим ресурсом |
| Любой вариант | OR | finite | Несколько, первый схлопнувшийся |
| Мозговой штурм | OR | open | Один широкий, не сужается |
| Выбор | XOR | comparative | Все сужаются, выживает один |
| Дебаты | XOR | dialectical | Два сталкиваются → синтез |

> **Статус:** все 13 режимов сконфигурированы в `modes.py`. Tick + Horizon работают для любого. Алгебра (AND/OR/XOR диспетчер) — [TODO](TODO.md).

## Интерфейс

**Graph** (основной): mode selector → SVG граф → detail panel (Brainstorm, Expand, Elaborate, Ask, Rephrase, Verify, Walk) → Run overlay со sparkline сходимости → Generation Studio

**Chat** (вспомогательный): диалог с моделью, context sidebar связывает граф ↔ чат

---

## Принципы

**Человек — Root Admin.** ИИ предлагает, человек решает.

**Confidence, не истина.** 0.73 честнее чем "я уверен".

**Принудительный антитезис.** Модель обязана показать другую сторону.

**Сходимость через исчерпание.** Стоп когда модель повторяется, не по лимиту.

## AGI vs AHI

| AGI | AHI |
|---|---|
| Заменяет человека | Усиливает человека |
| Чёрный ящик | Прозрачный граф |
| "Решает за вас" | "Предлагает, решаете вы" |
| Alignment через RLHF | Alignment через архитектуру |
