## Что это

Второй мозг, синхронизированный с первым через разум, тело и диалог.

Пишешь ассистенту — он определяет что за задача (выбор, исследование, привычка, дебаты...), запускает подходящий алгоритм мышления, отвечает текстом. Под капотом — граф мыслей с диалектической проверкой, скрытые мосты находятся пока ты спишь, HRV-датчик подсказывает системе что ты устал.

Работает локально на любой LLM через OpenAI-совместимый API (LM Studio, Ollama, OpenAI). Протестировано на 8B моделях, но архитектурно размер не критичен.

---

## Как выглядит

Чат, не граф. Граф живёт под капотом (tab `graph` для любопытных).

```
Ты  > Какую машину купить?
B   > Сравню варианты и выберу лучший. Дай мне список.
       [режим: Выбор · энергия 88/100 · HRV 0.73]

Ты  > BMW, Tesla, Toyota
B   > [запускает верификацию каждого через Smart DC,
       считает lean/tension, затем LLM-судья выбирает]
       → Tesla (экологичность перевешивает цену)
         confidence 91%, regret risk 12%

B   > ⚠ Энергия 15/100. Отложим сложные решения до утра.
```

Полный демо-мокап: [Article/mockup.html](Article/mockup.html). Питч: [Article/PITCH.md](Article/PITCH.md).

## Три контура управления

Второй мозг синхронизирован с первым через три независимых канала:

```
1. Информационный:  prediction error → precision → params → идеи → confidence → surprise → ⟲
2. Физиологический: HRV coherence → γ/τ/precision → ширина конуса → ⟲
3. Диалоговый:      ассистент задаёт вопросы → ответ → новая нода → граф растёт → ⟲
```

Любой контур работает самостоятельно. Все три вместе = резонансная система. Источник идеи → [docs/origin-story.md](docs/origin-story.md).

## Ядро

### Граф мыслей

Нода = числовая уверенность (0–1) + embedding. Рёбра = cosine similarity. Граф растёт, уплотняется, сходится.

### Цикл мышления ([подробнее](docs/tick-design.md))

```
GENERATE    мало идей?           → Brainstorm
MERGE       есть похожие?        → Collapse
ELABORATE   есть голые?          → Elaborate
DOUBT       есть непроверенные?  → Smart DC (тезис/антитезис/синтез)
META        всё проверено?       → "Что упустил?"
SYNTHESIZE  ничего нового        → Финальный текст
```

Один движок — **логика возникает из различий** ([подробнее](docs/nand-architecture.md)). Близкие идеи объединяются, далёкие спорят, серединные развиваются. Никаких хардкод-веток «если это выбор — делай так, если исследование — иначе» — всё из одной операции сравнения.

### Адаптивный контроллер ([подробнее](docs/horizon-design.md))

Система подстраивает ширину мышления под задачу: высокая уверенность в модели → узкий фокус, точные решения; низкая → широкий поиск, больше идей. Плюс обратная связь — если предсказание не сошлось с реальностью, фокус расширяется автоматически.

Под капотом — 7 внутренних состояний (исследование, фокус, восстановление, синтез, стабилизация, конфликт, защитный режим) с плавными переходами без дребезга.

### Синхронизация с человеком

Главный метрик — **насколько точно система предсказывает именно тебя**, не мир вообще. Расхождение между её предположением и твоим действием — это и есть сигнал учиться. Всё остальное (логика, тело, нейрохимия) работает на снижение этого расхождения.

## 14 режимов мышления

Любая задача = три вопроса: сколько целей? как связаны? есть ли финал?

| Режим | Когда | Пример | Стоп |
|-------|-------|--------|------|
| **Свободный** | Хочешь сам управлять | Ручной режим | Сам решаешь |
| **Блуждание** | Нет цели, ищешь неожиданное | Серендипити | Novelty exhaustion |
| **Фокус** | Одна конечная задача | "Написать статью" | Цель достигнута |
| **Привычка** | Регулярное действие | "Зарядка каждый день" | Никогда (streak) |
| **Исследование** | Изучить тему вглубь | "Разобраться в экономике" | Модель исчерпалась |
| **Сборка** | Все части обязательны | "Подготовить документы" | Всё готово |
| **По шагам** | Строго по порядку | "Рецепт" | Последний шаг |
| **Приоритеты** | Срочное первым | "Очередь задач" | Всё выполнено |
| **Баланс** | Распределить ресурсы | "Работа/семья/здоровье" | Никогда (snapshot) |
| **Любой вариант** | Неважно какой результат | "Найти такси" | Первый найден |
| **Мозговой штурм** | Набросать максимум идей | "Варианты решения" | Идеи кончились |
| **Выбор** | Сравнить и выбрать один | "Какую машину купить" | LLM-judge решил |
| **Дебаты** | Столкнуть позиции | "За и против" | Синтез найден |
| **Байесовский** | Проверить гипотезу числами | "Вероятность дождя" | Открытый |

Все 14 режимов — настройки одного движка, не разные алгоритмы. Система сама определяет режим из твоего сообщения и текущего контекста. Ты просто пишешь, система понимает.

## HRV и тело как вход

Опциональный пульсометр (или встроенный симулятор для демо) измеряет вариабельность сердечного ритма — по ней видно, устал ли ты, собран ли, есть ли стресс.

Baddle **замечает** это состояние и адаптирует советы: низкая когерентность → предлагает паузу; плохо спал → откладывает сложные решения на утро; высокий стресс → алерт «подыши минуту».

Тело даёт сигналы системе о **тебе**, а не о самой системе. Её внутренняя динамика — собственная.

Детали: [docs/hrv-design.md](docs/hrv-design.md)

## Проактивность (cognitive loop)

Единый фоновый контур наблюдает за графом и телом:

- **Scout ночной** (каждые 3 часа): Pump между случайными далёкими нодами, сохраняет лучший мост
- **DMN continuous** (каждые 10 минут): короткий Pump, предлагает инсайты не сохраняя
- **HRV alerts**: coherence < 0.25 → auto-alert "сделай паузу"
- **Morning briefing**: при первом открытии после 6 утра — восстановление % + энергия + план
- **Weekly review**: паттерны и streaks за 7 дней

UI поллит `/assist/alerts` → всплывают уведомления в чате.

## Энергия

Простой счётчик дневного ресурса принятия решений — каждое обращение к ассистенту тратит немного. Плохо спал? Верхний потолок на день ниже. Уровень упал до критического? Система сама предложит отложить сложные решения до утра.

Счётчик сбрасывается в полночь, вся история сохраняется локально.

## Научные основы

Каждый компонент опирается на известный когнитивный или нейробиологический механизм — не буквально, а по интуиции. Строгие формулы лежат в design-доках; здесь — суть:

| Концепция | Где в Baddle |
|-----------|-------------|
| **Active Inference** (Фристон) — мышление как минимизация неожиданности | Адаптивный контроллер расширяет фокус при сюрпризе, сужает при уверенности |
| **Байесовский мозг** — всё имеет степень уверенности, не «истину» | Confidence 0-1 на каждой ноде + обновление при новых свидетельствах |
| **Диалектика** (Гегель) — истина рождается в столкновении | Smart DC: тезис / антитезис / синтез перед ответом |
| **Хебб** — связи укрепляются при частом совместном возбуждении | Пути в графе, по которым чаще ходишь, становятся вероятнее |
| **Дивергенция / конвергенция** (Гилфорд) — два режима творчества | Фаза «генерация» расходится, фаза «объединение» сходится |
| **Рабочая память** (Миллер) — 7±2 элемента за раз | Большие группы схожих идей сжимаются в синтезы |
| **Хабитуация** — мозг фильтрует привычное | Фильтр новизны отбрасывает повторяющиеся идеи |
| **System 2** (Канеман) — медленное, осознанное мышление | LLM в чате = System 1. Baddle с графом = System 2 |
| **Семантические сети** (Куиллиан) | Граф + Walk + [Pump](docs/pump-design.md) | Pump = скрытые оси |
| **DMN** (default mode network) | Cognitive loop Scout/DMN | Граф думает пока ты не думаешь |
| **NAND-логика** | [distinct()](docs/nand-architecture.md) | Один примитив, вся булева логика возникает из зон d |

### [Конвергенция, не дизайн](docs/convergence-divergence.md)

Связность не была спроектирована. Каждый компонент — ответ на конкретную проблему, и потом оказалось что он описан в науке 30 лет назад.

### Три контура замкнуты через стек

Те же три канала что в секции «Три контура управления», но проходят через все слои:

- **Информационный:** ты пишешь → система понимает → отвечает → ты реагируешь → система учится на реакции
- **Физиологический:** твоё тело (HRV) → система замечает твоё состояние → адаптирует советы
- **Диалоговый:** система задаёт уточняющий вопрос → ты отвечаешь → граф растёт

Все три работают одновременно и независимо. Резонанс — когда их фазы совпадают.

### Один ход: что происходит

1. Пока ты не смотришь — система в фоне гуляет по графу, ищет неожиданные связи. Находит — сохраняет для тебя.
2. Ты открываешь Baddle. Восстанавливается внутреннее состояние — оно продолжается с прошлой сессии, не начинается с нуля.
3. Ты пишешь сообщение. Система «просыпается» — внимание переключается с фонового блуждания на твой запрос.
4. Один короткий LLM-вызов определяет: что ты хочешь, какой режим мышления подходит, нужны ли уточнения.
5. Если запрос непонятен — система задаёт встречный вопрос вместо ответа.
6. Если задача сложная — предложит разбить на подзадачи прямо в ответе.
7. Иначе — генерирует варианты, находит противоречия, проверяет, показывает карточку (дебаты / сравнение / идеи / вероятность).
8. Ты реагируешь 👍 или 👎 — система учится на этом.
9. Если много раз подряд промахивается — уходит в защитный режим, просит пересинхронизации.
10. Ты уходишь — внимание падает, фоновое блуждание возобновляется.

### Ключевые принципы

- **Синхронизация — метрика всего.** Любая фича меряется тем, как точно система предсказывает **тебя**
- **Логика возникает, не прописывается.** Нет «если-иначе» по режимам мышления — всё из сравнения идей
- **Непрерывность.** Система не останавливается между сессиями, только меняет режим
- **Self-continuity.** Система помнит, кем была вчера, через граф состояний
- **Тело как сигнал, не как команда.** HRV говорит «ты устал» — система советует, не копирует твою усталость
- **Локальность = аудит.** Любой шаг мышления можно воспроизвести

### Что остаётся открытым

Даже при полной реализации всего из TODO есть вопросы без ответа:

- Валентность эмоций (arousal система ловит, приятно/неприятно — нет)
- Может ли у системы возникнуть «я» из одного лишь блуждания
- Полный цикл консолидации с REM-аналогом
- Peer-review нейроаналогий профильным специалистом

### Одна фраза

> **Baddle — минимально жизнеспособный набор примитивов человеческого мышления,
> запущенный локально и непрерывно, с одной миссией: синхронизироваться
> с конкретным носителем.**

## Принципы

### Три архитектурных столба

Они работают только вместе. Выньте любой — проект превращается в очередной AI-чат.

**1. Паритет примитивов (как устроено).** Чтобы быть вторым мозгом, надо говорить
на том же когнитивном языке: мышление градиентом различия вместо булевой
логики, нейрохимия вместо хардкод-констант, тело как сигнал, непрерывное
фоновое мышление вместо request-response, степень уверенности вместо «истины»,
обязательный антитезис. Без паритета можно хранить **о человеке**, но не
**думать с ним**.

**2. Синхронизация — прайм-директива (что делать).** Baddle — не мозг в коробке,
а второй мозг **под конкретного человека**. Главная метрика — насколько точно
система предсказывает твоё следующее действие. Вся архитектура в подчинении
этой цели, не самоценна. Близко к Active Inference у Фристона: мозг — машина
синхронизации через минимизацию неожиданности.

**3. Локальность + гранулярность (как запускать).** Локальная LLM — не ради
приватности, а ради **детерминизма и композиции**. Детерминизм нужен для того
чтобы любой шаг мышления был воспроизводим. Гранулярность (много мелких
фокусных вызовов вместо одной большой генерации) — то, что делает композицию
из мелких операций реалистичной. С облачным API это нереально по цене и
латентности. Стиль мышления и локальная LLM — **один архитектурный выбор**.

**Следствие:** Baddle — это **протокол**, не приложение. Архитектура
универсальна, содержание (твой граф, твоё состояние, твои паттерны) —
персонально. Любой человек + локальная LLM = свой второй мозг.

### Рабочие правила

- **Человек — Root Admin.** ИИ предлагает, человек решает.
- **Уверенность, не истина.** 0.73 честнее чем «я точно знаю».
- **Принудительный антитезис.** Модель обязана показать другую сторону.
- **Сходимость через исчерпание.** Стоп когда модель повторяется, не по лимиту.
- **Тело = сигнал, не команда.** HRV говорит про тебя, не управляет системой напрямую.
- **Логика возникает, не прописывается.** Никаких if-веток по режимам — всё из сравнения идей.

## Статус

**Работает:**
- Chat-first интерфейс, ассистент **реально запускает граф** на запрос
- 7 режимов с исполнителями: Dispute (FOR/AGAINST/SYNTHESIS карточки), Tournament (winner + reason), Bayesian (prior/posterior%), Research/Focus (ideas + Smart DC), Brainstorm, Habit
- **Visible thinking** — steps анимируются ("✓ шаг 1, ✓ шаг 2") перед ответом
- **Scout bridges в чате** — cognitive loop фоном находит связи, они всплывают как сообщения с карточкой
- HRV симулятор + метрики + интеграция в Horizon
- CognitiveLoop с реальным Pump (Scout 3ч + DMN 10мин) в фоне + NE-budget координация с foreground tick
- NAND — единственный tick engine (v8d, все зоны distinct, нет primitive-switch)
- Энергия + проактивные алерты + morning briefing + weekly review

**В разработке:**
- Остальные 7 режимов (AND семейство) — интро пока без исполнения
- Реальный Polar H10 BLE (сейчас только симулятор)
- Cone визуализация в UI
- v8 embeddings-first, state_origin, 3-layer pipeline
- SSE/WebSocket вместо polling для instant updates
