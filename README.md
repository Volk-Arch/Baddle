# baddle

Инструмент для ручных экспериментов с генерацией текста нейросетью.
Позволяет заглянуть внутрь процесса — не просто получить ответ, а управлять им пошагово.

Большинство LLM-интерфейсов — чёрный ящик: отправил промпт, получил текст.
**baddle** открывает этот ящик: можно идти токен за токеном, смотреть распределение вероятностей,
вбрасывать свои токены в середину генерации, сравнивать как один и тот же промпт
ведёт себя при разных настройках — и всё это в терминале или в браузере.

---

## Установка

```bash
git clone https://github.com/yourname/baddle
cd baddle

# Положи GGUF-модель в папку models/
# например: models/Qwen3-8B-Q4_K_M.gguf

python setup.py        # определит CUDA, установит llama-cpp-python + зависимости
pip install flask      # только для веб-UI
```

---

## Запуск

```bash
python main.py         # CLI-интерфейс с меню
python ui.py           # веб-интерфейс, откроет браузер на localhost:7860
```

Если в `models/` одна модель — загрузится автоматически. Если несколько — появится выбор стрелочками.

```bash
python main.py -m Qwen3-8B-Q4_K_M.gguf   # выбрать модель явно
python main.py --ctx 8192                 # увеличить контекст
python main.py --no-gpu                   # только CPU
python ui.py --port 8080
```

## Режимы

### `step` — пошаговая генерация

Модель генерирует **один токен за раз**. После каждого токена можно посмотреть состояние
распределения, вбросить произвольный текст, сменить температуру — и продолжить.

#### Команды

| Команда | Действие |
|---|---|
| `[Enter]` | следующий токен |
| `top N` | N самых вероятных следующих токенов с вероятностями и гистограммой |
| `inject <текст>` | вбросить текст вместо следующего токена |
| `auto N` | автоматически N токенов без остановки |
| `temp 0.8` | изменить температуру прямо в процессе |
| `show` | вывести весь накопленный текст |
| `save session.json` | сохранить сессию в файл |
| `load session.json` | воспроизвести сохранённую сессию |
| `reset` | откатиться к исходному промпту |
| `q` | выйти |

`↑↓` — история команд,  `Tab` — автодополнение

---

#### Эксперимент 1 — Что модель хочет сказать следующим?

**Промпт:** `The capital of France is`  **temp:** `0`

```
>> top 10
```

Ожидаем: `" Paris"` с вероятностью ~0.99, остальные девять — `" Lyon"`, `" the"`, `" a"`
с вероятностью < 0.01.

**Интерес:** на фактических вопросах модель почти детерминирована — распределение
сжато в один токен. На творческих текстах картина другая: 10–20 токенов
делят вероятность примерно поровну. `top` делает это видимым.

---

#### Эксперимент 2 — Вброс хаоса и восстановление

**Промпт:** `Once upon a time there was a kind princess who`  **temp:** `0`

```
>> auto 5               # модель пишет "loved her kingdom and"
>> inject " suddenly exploded"
>> auto 20              # смотрим как выходит из ситуации
```

Ожидаем: модель не «сломается» — она переинтерпретирует вброс и продолжит нарратив,
скорее всего что-то вроде *"...leaving the castle in ruins, but miraculously no one was hurt..."*.

**Интерес:** языковая модель держит весь контекст, а не только предыдущий токен.
Аномалия не рвёт генерацию — attention видит всю историю и строит правдоподобное продолжение
даже из нонсенса. Это и есть то, что делает inject интересным: можно форсировать
любой поворот и наблюдать как модель с ним справляется.

---

#### Эксперимент 3 — Точка ветвления

**Промпт:** `The scientist looked at the results and realized`  **temp:** `0`

```
>> top 15               # видим: "that"(0.31), "the"(0.18), "something"(0.12), "he"(0.09)
                        # модель не уверена — точка ветвления
>> inject " nothing"    # выбираем маловероятный путь
>> auto 30
>> reset
>> inject " everything changed"
>> auto 30
>> show
```

Ожидаем: два совершенно разных нарратива из одного промпта.

**Интерес:** точка ветвления — это место где распределение широкое.
Один вброс разворачивает историю в противоположную сторону. `top` перед inject
позволяет осознанно выбирать: идти по «хайвею» вероятностей или свернуть в хвост.

---

#### Эксперимент 4 — Температура меняет характер

**Промпт:** `The old man sat by the window and thought about`

```
>> temp 0
>> auto 30
>> show                 # предсказуемый, структурированный текст
>> reset
>> temp 1.4
>> auto 30
>> show                 # хаотичный, неожиданный
```

Ожидаем: при `temp=0` — «хайвей» самых вероятных токенов, гладкий нарратив.
При `temp=1.4` — модель прыгает по хвостам распределения, неожиданные слова и повороты.

**Интерес:** температура — не просто «ползунок креативности».
Команда `top` показывает механизм: при высокой температуре токен с `p=0.01`
становится почти равновероятен токену с `p=0.40`. Это математически наблюдаемо.

---

#### Эксперимент 5 — Вброс нонсенса, потом `top`

**Промпт:** `In quantum mechanics, the wave function`  **temp:** `0`

```
>> auto 3               # "describes the probability"
>> inject " banana"
>> top 10               # что модель хочет написать ПОСЛЕ банана?
```

Ожидаем: неожиданное распределение — либо модель попытается продолжить
про квантовую механику как ни в чём не бывало, либо попытается «объяснить» банан
в физическом контексте.

**Интерес:** показывает силу attention к последнему токену.
Иногда аномальный токен почти полностью захватывает следующее предсказание,
иногда контекст его «поглощает». Результат непредсказуем и каждый раз разный
в зависимости от модели.

---

#### Эксперимент 6 — Воспроизводимый эксперимент

```
>> auto 5
>> inject " but strangely"
>> temp 0.9
>> auto 20
>> save interesting_run.json
```

Сессия сохраняется как JSON. При `load interesting_run.json` — воспроизводится
шаг за шагом и оставляет модель в том же состоянии. Можно продолжать с этой точки
или поделиться файлом.

### `parallel` — два разных промпта, один forward pass

Два разных промпта генерируются **одновременно**. Технически: оба sequence ID
обрабатываются в одном вызове `llama_decode` — GPU работает с обоими параллельно через KV-cache.
Результат — live split-screen, оба потока обновляются в реальном времени.

```
┌─ Stream A  step 12 ────────────────┬─ Stream B  step 12 ──────────────┐
│ The future of open source AI is    │ The biggest risk of closed AI is  │
│ bright — communities around the    │ concentration of power in a few   │
│ world are building tools that...   │ corporations that can shape...    │
└────────────────────────────────────┴──────────────────────────────────┘
```

---

#### Эксперимент 1 — Противоположные фреймы

**A:** `The new technology will help humanity because`
**B:** `The new technology will harm humanity because`
`temp=0.7`

Ожидаем: два одинаково уверенных, логически противоположных текста
с похожей структурой аргументации.

**Интерес:** модель не «знает» правды — она предсказывает правдоподобные токены.
Оба фрейма одинаково правдоподобны. Side-by-side это видно особенно наглядно:
модель с одинаковой уверенностью обосновывает взаимоисключающие позиции.

---

#### Эксперимент 2 — Butterfly effect из одного промпта

**A:** `The detective opened the door and saw`
**B:** `The detective opened the door and saw`
`temp=0.9`

Два одинаковых промпта, случайная выборка.

Ожидаем: разные детективные истории, которые расходятся уже на 1–3 токене.

**Интерес:** первый «случайный» токен задаёт всю последующую траекторию.
Посмотри какой именно токен стал точкой расхождения — обычно это самое неожиданное слово.

---

#### Эксперимент 3 — Два языка, одна тема

**A:** `The meaning of life is`
**B:** `Смысл жизни — это`
`temp=0.7`

Ожидаем: разные стили и концепции, отражающие то как тема представлена
в разных языковых корпусах.

**Интерес:** модель не переводит — она работает в пространстве токенов конкретного языка.
Один и тот же вопрос активирует разные паттерны в зависимости от языка.

---

### `compare` — один промпт, два набора параметров

Один промпт, но **два разных конфига** (температура, top_k). Оба потока стартуют
из идентичных токенов и расходятся как только параметры дают разные выборки.
Интерфейс показывает точный шаг расхождения.

```
┌─ temp=0.0  top_k=1  step 8 ──────┬─ temp=1.1  top_k=40  step 8 [diverged @ step 3] ─┐
│ Scientists recently discovered   │ Scientists recently discovered                     │
│ that a new species of deep-sea   │ that music actually rewires the                    │
│ fish can survive in...           │ human brain in ways we never...                    │
└──────────────────────────────────┴────────────────────────────────────────────────────┘
```

---

#### Эксперимент 1 — Greedy vs sampling

**Промпт:** `Scientists recently discovered that`
**A:** `temp=0.0, top_k=1`  **B:** `temp=1.0, top_k=40`

Ожидаем: A идёт по единственному «правильному» пути, B уходит в одну из многих
равновероятных веток. Момент расхождения — это токен где у модели было несколько
конкурирующих вариантов.

**Интерес:** greedy — это то, что модель считает «наиболее правдоподобным».
Sampling — это то, что она считает «правдоподобным в принципе». Разница между
этими двумя понятиями — весь смысл compare режима.

---

#### Эксперимент 2 — Где температура важна больше всего?

**Промпт:** `In conclusion, the research suggests that`
**A:** `temp=0.1`  **B:** `temp=0.9`

Следи за номером шага расхождения.

Ожидаем: на академическом тексте расхождение может наступить поздно (шаг 5–10),
на художественном — уже на шаге 1–2.

**Интерес:** шаг расхождения показывает «ширину» распределения в конкретном контексте.
Академический текст — распределение узкое, модель «знает» следующее слово.
Открытый нарратив — широкое, каждый токен — развилка.

---

#### Эксперимент 3 — Эффект top_k

**Промпт:** `The recipe calls for`
**A:** `temp=0.8, top_k=3`  **B:** `temp=0.8, top_k=100`

Одна и та же температура, разное ограничение словаря.

Ожидаем: A будет выбирать только из трёх вариантов — текст более связный
но предсказуемый. B может уйти в любой из сотни токенов — больше разнообразия,
выше риск странных слов.

**Интерес:** top_k и temperature — разные оси. Temperature растягивает вероятности,
top_k режет хвосты. Их взаимодействие и создаёт «характер» генерации.

## Веб-UI

```bash
python ui.py
```

Открывает `http://localhost:7860`. Поддерживает `parallel` и `compare` режимы
с live-стримингом в браузере: два потока обновляются в реальном времени через SSE.
Удобно когда нужно показать кому-то или делать заметки рядом с открытым текстом.

## Сессии

В step-режиме все команды записываются. `save <файл>` сохраняет полный журнал:

```json
{
  "model": "Qwen3-8B-Q4_K_M.gguf",
  "prompt": "The scientist looked at the results and realized",
  "temp": 0.0,
  "events": [
    {"type": "step"},
    {"type": "step"},
    {"type": "inject", "text": " nothing"},
    {"type": "auto", "n": 30},
    {"type": "temp", "value": 0.9},
    {"type": "step"}
  ]
}
```

`load <файл>` воспроизводит сессию и оставляет модель в том же состоянии —
можно продолжать эксперимент с этой точки.
