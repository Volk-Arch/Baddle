# baddle

> AHI Protocol — Augmented Human Intelligence.
> Не спрашивай у ИИ ответ — управляй процессом мышления.

**[English version](README_EN.md)** | **[Установка](SETUP.md)** | **[Видение](VISION.md)** | **[TODO](TODO.md)**

Граф мыслей с байесовской уверенностью, марковскими переходами и диалектическим синтезом. Задаёшь цель, нажимаешь Run — система автоматически исследует тему, проверяет гипотезы, синтезирует результат. Локально через llama.cpp или через API. Бесплатно, на квантизированной 8B модели.

---

## Как это работает

1. Задаёшь **цель** — "Доказать что прокрастинация полезна"
2. Нажимаешь **Run**
3. Система автоматически:
   - Генерирует гипотезы
   - Собирает evidence для каждой
   - Проверяет диалектически (тезис/антитезис/синтез через embedding centroid)
   - Задаёт себе вопросы ("а что я упустил?")
   - Коллапсирует верифицированные кластеры
   - Повторяет цикл
4. Получаешь **структурированное эссе** с аргументацией

## Операции с графом

### Think
Генерирует N коротких независимых мыслей по теме. Каждая мысль — отдельная попытка, не продолжение предыдущей. LLM автоматически классифицирует тип (hypothesis, fact, question, goal, action) и начальную уверенность.

### Elaborate
Углубление конкретной мысли. Раскрывает деталь, следствие или механизм. Дочерние узлы автоматически становятся **evidence** — LLM определяет, поддерживают они родительскую гипотезу (supports) или противоречат (contradicts).

### Expand
Ветвление от мысли в новом направлении. Не углубление, а другой ракурс на ту же тему. Создаёт siblings, а не children.

### Collapse
Синтез кластера мыслей в один узел. Из 5 отдельных идей — один связный абзац. Два режима: merge (удаляет исходные) и no-merge (добавляет синтез как дочерний, оригиналы остаются).

### Rephrase
Переформулировка мысли. Сохраняет суть, меняет форму. Полезно когда гипотеза слабая из-за плохой формулировки, а не из-за содержания.

### Verify (Smart DC)
Диалектическая проверка — ключевой механизм. Три параллельных генерации:
1. **Тезис** — сильнейший аргумент ЗА
2. **Антитезис** — сильнейший аргумент ПРОТИВ
3. **Нейтраль** — контекст и условия

Три текста -> три вектора в embedding-пространстве -> **центроид** (точка равновесия) -> синтез, максимально близкий к центроиду. Confidence = cosine similarity синтеза к центроиду. Принудительный антитезис ломает эхо-камеры.

### Walk
Random Walk по графу из выбранного узла. Использует transition probabilities на рёбрах. Показывает куда ведёт мысль — какие узлы чаще всего оказываются конечными точками.

## Математическая база

### Байесовская уверенность

Каждый узел имеет **confidence** (0-1):

```
P(H|E) = P(E|H) * P(H) / [P(E|H)*P(H) + P(E|~H)*(1-P(H))]
```

При добавлении evidence (supports/contradicts) confidence обновляется через Bayes. Alpha/Beta модель отслеживает баланс поддерживающих и противоречащих свидетельств.

### Марковские переходы

Каждое ребро имеет **transition_prob**, нормализованное по исходящим:
- Косинусное сходство эмбеддингов -> базовый вес
- Directed edges получают bonus x1.5
- Хебб: при навигации пользователя `tp += lr * (1 - tp)` — часто используемые пути усиливаются
- Детектор ловушек: узлы с высоким входящим и низким исходящим tp

### Временные связи

Узлы, созданные в пределах 5 минут, получают temporal links. Вес зависит от временной близости (0.3-0.6).

### Кластеризация

Connected components на рёбрах выше threshold. Используется для группового collapse, визуализации hull, и определения изолированных узлов.

## Цикл Tick — автоматическое мышление

`thinking.py` — движок автономного мышления. На каждом шаге анализирует граф и решает какое действие выполнить следующим.

### Классификация

На входе: все узлы, рёбра, directed children. На выходе:
- **hypotheses** — непроверенные утверждения
- **weak** — confidence <= 0.5
- **unverified** — confidence < stable_threshold
- **verified** — confidence >= stable_threshold
- **no_evidence** — гипотезы без дочерних evidence
- **isolated** — узлы без рёбер

### Навигация к цели

`pick_toward_goal()` — BFS от каждого кандидата до goal. Выбирает ближайшего. Trap avoidance: пропускает ловушки. Exploration: если лучший уже пробован — берёт менее очевидного.

### Deep mode (по фазам)

```
EXPLORE    если гипотез < 5     -> Think (набрать массу)
DEEPEN     если есть голые      -> Elaborate (добавить evidence)
REPHRASE   если 2+ детей, слаб  -> Rephrase (переформулировать)
VERIFY     если есть unverified -> Smart DC (проверить)
META       если verified >= 5   -> Think ("что упустил?")
COLLAPSE   если verified >= 5   -> Collapse (синтезировать кластеры)
EXPAND     если есть isolated   -> Expand (подключить)
ASK        если вопросов < 3    -> Ask (задать вопрос)
SYNTHESIZE все проверено        -> Stable (готов к финальному эссе)
```

### Fast mode (по приоритетам)

Те же инструменты, но не по фазам, а по приоритету: сначала чинит самое слабое, сходится когда может.

### Force Collapse

После N шагов (настраивается) — принудительное сжатие порциями по 5. Hard stop на 2N.

**Output:** essay / brief / list / none.

## Интерфейс

Светлая тема (Notion-style). Четыре режима: **Graph** (основной), **Chat**, **Step**, **Parallel**.

### Graph mode
- **Topic + Think/Add** — ввод темы, автоматическая генерация мыслей
- **Save/Load** — сохранение графов на сервер, автосохранение после каждого действия
- **Actions bar** — Select, Collapse, Link, Flow, Time, Layout, Auto, Run, Undo, Save, Reset
- **SVG визуализация** — drag, zoom, pan, цветовая кодировка по типу и уверенности
- **Detail panel** — heatmap по токенам, confidence, тип, alpha/beta, Walk, Verify
- **Контекстное меню** — Expand, Elaborate, Rephrase, Verify, Walk, Evidence, Chat, Edit, Delete
- **Generation Studio** — пакетные варианты для rephrase/elaborate/expand/collapse

### Chat mode
- Разговор с моделью, context sidebar (граф -> чат, чат -> граф)

### Step mode
- Токен за токеном, top-10 кандидатов с вероятностями, heatmap

### Parallel mode
- Два промпта параллельно, compare mode (один промпт, разные параметры)

## Настройки

- **Local / API / Hybrid** — генерация и embeddings раздельно
- **Embedding model** — отдельная модель для similarity и centroid (nomic-embed-text)
- **Heatmap** — настраиваемая шкала энтропии

> Qwen3-8B для генерации + nomic-embed-text для embeddings. Через LM Studio обе параллельно.

```
  src/

```
