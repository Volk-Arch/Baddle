# Резонансный промпт-преset — Слой 5

Chat-фича: пользователь описывает своё состояние через шаблон
(«контекст волны»), Baddle/LLM зеркалит параметры в ответе.
Делает active inference юзера явным. Без backend-изменений —
только preset в chat UI.

Статус: **черновик**. Простая фича, можно сделать за 1-2ч.

---

## Идея

Сейчас юзер пишет сообщение → LLM отвечает в своём стандартном
стиле (essay или формат который выставлен в settings). Нет
механизма **явно просить** ответ в определённом темпе, глубине
или формате — для конкретного состояния.

Резонансный взгляд: общение — это **интерференция двух
резонаторов**. Чтобы резонанс случился, параметры резонаторов
должны совпадать. Если юзер в короткой волне (быстро, по делу) —
а LLM отвечает длинным эссе — интерференция деструктивная, ответ
не попадает.

## Шаблон преsetа

```
[Контекст волны] 🔵 / 🔴 / смешанный
[Состояние] образно, 1-2 фразы как чувствуется
[Запрос] insight / план / критика / поддержка / структура
[Параметры ответа]
  • Темп: медленно / быстро
  • Глубина: образы и аналогии / факты и шаги
  • Формат: список / эссе / диалог / схема
  • Тон: поддерживающий / вызывающий / нейтральный
[Якорь] ключевая фраза или метафора (опционально)
```

### Пример использования (длинная волна)

```
[Контекст волны] 🔵 длинная
[Состояние] чувствую, идея где-то рядом, но не могу схватить форму
[Запрос] помоги проявить структуру мысли
[Параметры ответа]
  • Темп: медленно, с паузами
  • Глубина: метафоры и аналогии
  • Формат: диалог, не монолог
  • Тон: исследовательский
[Якорь] начни с: «Давай представим, что эта идея — это...»
```

### Пример использования (короткая волна)

```
[Контекст волны] 🔴 короткая
[Состояние] задача горит, надо быстро решить
[Запрос] конкретный следующий шаг
[Параметры ответа]
  • Темп: быстро
  • Глубина: факты и шаги
  • Формат: пронумерованный список
  • Тон: прямой
```

## UI реализация

### Вариант A: Preset-кнопка над chat input

Над полем ввода — маленький dropdown `📡 Резонанс ▾`:
- 🔵 Длинная волна (исследовательская)
- 🔴 Короткая волна (действие)
- ⚪ Сброс к нулю (из стресса)
- ✏️ Настроить (открывает полный шаблон)

Клик по варианту → pre-fills input с шаблоном (заполненные
параметры для этого режима, placeholder'ы для состояния/запроса
для заполнения юзером).

**Преимущество:** быстро, не нужно каждый раз писать шаблон.
**Недостаток:** Добавляет ещё один UI элемент.

### Вариант B: Slash-команда

`/rez 🔵 быстрый план задачи X` — парсится backend'ом, превращается
в полный шаблон перед передачей LLM.

**Преимущество:** минимальная UI нагрузка, power-user friendly.
**Недостаток:** нужно помнить команду.

### Вариант C: Auto-detect из frequency_regime

При `frequency_regime=short_wave` → автоматически добавляется
hidden header к промпту: «[system hint] user is in short_wave,
respond terse». Юзер ничего не видит.

**Преимущество:** zero UI, honest use of sensor data.
**Недостаток:** юзер не знает что LLM адаптируется, меньше
control.

### Рекомендация

**A + C в комбинации.** Preset-кнопка для explicit control
(когда юзер хочет), auto-detect через frequency_regime как
fallback (когда юзер не думает об этом). Sla`sh`-команду можно
добавить позже как power-user shortcut.

## Реализация (Variant A)

### Frontend

**[templates/partials/chat_input.html](../templates/partials/chat_input.html) (или существующий
input-row в [templates/index.html](../templates/index.html)):**

Добавить dropdown перед input:
```html
<div class="resonance-preset">
  <button id="resonance-btn">📡 Резонанс</button>
  <div class="resonance-menu hidden">
    <a data-preset="long">🔵 Длинная волна</a>
    <a data-preset="short">🔴 Короткая волна</a>
    <a data-preset="reset">⚪ Сброс</a>
    <a data-preset="custom">✏️ Настроить</a>
  </div>
</div>
```

**[static/js/assistant.js](../static/js/assistant.js):**

```js
const PRESETS = {
  long: `[Контекст волны] 🔵 длинная
[Состояние] %STATE%
[Запрос] %REQUEST%
[Темп] медленно, с паузами
[Формат] диалог, образы`,
  short: `[Контекст волны] 🔴 короткая
[Состояние] %STATE%
[Запрос] %REQUEST%
[Темп] быстро
[Формат] нумерованный список`,
  reset: `[Контекст волны] ⚪ сброс
[Состояние] много переключений, много шума
[Запрос] помоги выбрать одно главное
[Темп] средний
[Формат] 3-5 bullet'ов`,
};

function applyPreset(key) {
  const input = document.querySelector('#chat-input');
  input.value = PRESETS[key].replace(/%STATE%/g, '[опиши как чувствуется]').replace(/%REQUEST%/g, '[что нужно]');
  input.focus();
}
```

### Backend

**Нулевые изменения.** Шаблон просто становится частью user
message, LLM его понимает (современные модели хорошо реагируют
на structured prompts).

### Опционально: auto-detect (Variant C)

[src/assistant.py](../src/assistant.py) в `/assist` — перед вызовом
LLM добавить system hint если `frequency_regime` not flat:

```python
if user_state.frequency_regime == "short_wave":
    system_hints.append(
        "User appears to be in a fast, focused state. "
        "Keep response terse and action-oriented."
    )
elif user_state.frequency_regime == "long_wave":
    system_hints.append(
        "User appears to be in a reflective, associative state. "
        "Use metaphors and longer explanations."
    )
```

## Ценность

- **Explicit control.** Юзер явно управляет формой ответа без
  лезения в settings. Per-message, не per-session.
- **Обучает пользователя.** Просто заполняя шаблон, юзер начинает
  замечать своё состояние — это само по себе ценно.
- **Ложится в резонансную рамку.** Не случайная фича —
  операционализация принципа «общение = совпадение метрик
  резонаторов».
- **Низкий риск.** Минимум кода, не меняет backend, не
  конфликтует ни с чем.

## Риск

- **Шум в UI** — ещё одна кнопка рядом с чат-input. Решение:
  collapse в маленькую иконку 📡, открывается по клику.
- **Юзер может игнорировать.** Это ок — preset опциональный.
- **LLM может не зеркалить идеально** — это вопрос prompt
  engineering, не архитектуры. Через месяц можно калибровать
  шаблоны если окажется что «[Темп: быстро]» не влияет.

## Оценка

~1-2ч. Минимальный риск.

## Критерий успеха

Через 1-2 нед use:
- **Частота использования:** сколько % сообщений через preset.
  < 5% = фича невостребована, можно удалить.
- **Изменение паттерна:** использует ли юзер 🔵 vs 🔴
  адекватно ситуации (по корреляции с frequency_regime).
- **Feedback Игоря:** субъективное ощущение что «ответы попадают
  лучше».
