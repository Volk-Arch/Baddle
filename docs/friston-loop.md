# Friston Loop — предиктивная модель и ошибка предсказания

> Единственное место где описан предиктивный контур Baddle целиком. Если в других документах встречаешь упоминание ошибки предсказания (PE) — детали здесь.

---

## Оптика

Активный вывод по Фристону: мозг не ждёт сигналов пассивно, он постоянно **предсказывает** что увидит / услышит / почувствует и реагирует на разницу между предсказанием и реальностью — **ошибку предсказания** (prediction error, PE). Минимизация долгосрочной PE = гомеостаз.

У Baddle симметричная петля: Baddle наблюдает пользователя → предсказывает его следующее поведение → реальное поведение даёт ожидание (expectation) vs реальность → разница есть удивление (surprise) → **ошибка предсказания** (PE = reality − expectation) → агрегируется в **давление дисбаланса** (imbalance_pressure) через EMA → питает **отображаемое выгорание** (display_burnout) → замедляет циклы исследования.

Плюс **self-петля:** Baddle предсказывает **собственную** нейрохимию. Когда граф уезжает в экзотику или LLM деградирует — это тоже surprise, отдельный канал.

**Главное:** PE не просто пишется как observation. Растущее давление дисбаланса замедляет все investigation-циклы (DMN / pump / scout / night) через регулятор простоя ([_throttled_idle](../src/cognitive_loop.py)). Петля замкнута от предсказания до поведения системы.

## Анатомия

### Два предиктора

Baddle предсказывает **пользователя** (пять базовых линий) и **себя** (один трёхмерный baseline). Все — медленный EMA: baseline живёт 50+ обновлений (дни, не минуты). Именно это делает PE осмысленным — иначе baseline повторяет текущее и surprise = 0 всегда.

| Уровень | Что предсказывается |
|---|---|
| Поведение пользователя | **уровень состояния** (state_level) глобально + state_level по времени суток + трёхмерный вектор дофамин/серотонин/норадреналин (DA, S, NE) |
| Тело пользователя | когерентность сердечного ритма (hrv_coherence) по времени суток — физический baseline |
| Сама Baddle | собственные DA/S/NE (зеркальная петля) |

### Пять каналов ошибки предсказания

| Канал | Семантика |
|---|---|
| **Поведение, 3D** | норма разности 3D-вектора и 3D-ожидания, делённая на √3 — сдвиг по осям DA/S/NE |
| **Скаляр по времени суток** | реальность минус ожидание для текущего времени суток — «привычно ли для этого времени» |
| **Разрыв агентности** (agency_gap) | 1 − агентность — план не равен выполненному |
| **Удивление от HRV** | модуль разности текущей когерентности и baseline для текущего времени суток — физический канал (с Polar) |
| **Self-дисбаланс** | норма разности Baddle-вектора нейрохимии и собственного ожидания, делённая на √3 — PE Baddle на самой себе |

Пять разных физических источников ответа на вопрос «не как обычно».

### Агрегация

Все пять нормализованы в диапазон [0, 1] и собираются через максимум в [cognitive_loop._advance_tick](../src/cognitive_loop.py): агрегат идёт в [ProtectiveFreeze.feed_tick](../src/horizon.py) вместе с рассогласованием (sync_error) → EMA с временной константой в сутки → **давление дисбаланса** (imbalance_pressure).

Давление дисбаланса — один из трёх feeder'ов отображаемого выгорания (display_burnout), вместе с **накопителем конфликтов** (conflict_accumulator) и **давлением тишины** (silence_pressure). Подробно про замедление циклов — [alerts-and-cycles.md § Adaptive idle](alerts-and-cycles.md).

## Почему так

### Четыре baseline по времени суток, а не один

Без разделения по времени суток утренняя апатия и вечерняя сливаются в усреднённый baseline → PE на обоих около нуля, surprise теряет специфичность.

С разделением: у Baddle четыре baseline — ждёт разного утром и вечером **от этого конкретного** пользователя. Если он обычно тих после 23:00, ночная тишина не даёт PE. Если обычно активен в 11:00, утреннее молчание — сильный surprise.

Прямая интерпретация Фристона: **приор должен быть контекстуальным**. Контекст у нас — время суток (может быть расширено на день недели, погоду — см. [planning/TODO.md § OQ #5](../planning/TODO.md)).

### Трёхмерный вектор, не скаляр

Скалярный surprise, где реальность = (DA + S) / 2, теряет информацию при разнонаправленных сдвигах: DA падает (интерес упал), S растёт (спокоен) → уровень состояния почти не меняется → surprise = 0. Но произошёл сдвиг **типа состояния** — это настоящий PE.

Трёхмерная L2-норма ловит это честно. Максимум √3 ≈ 1.732 (все оси на противоположных концах), реально 0.1–0.6.

### Максимум, не сумма

Сумма пяти каналов: если каждый даёт 0.2, sum = 1.0 выглядит кризисом. Но ни один не сигналит реальную проблему — просто baseline-шум.

Максимум: любой канал достаточен чтобы поднять давление; зашумление одного не усиливается другими. 0.2 в каждом → max = 0.2 → честный baseline.

### Self-prediction (зеркальный Friston)

Асимметричный loop странен: Baddle предсказывает юзера, но не себя. Если собственная нейрохимия уезжает далеко (граф ушёл в экзотику / LLM деградирует / ложный спайк feedback) — это её проблема, должна сама поймать. Пятый канал агрегации.

### Разрыв агентности отдельным каналом

Пользователь может быть в отличной нейрохимии (высокие DA и S), но не выполнять планы — learned helplessness. Скалярная PE этого не ловит, разрыв агентности (1 − agency) ловит прямо.

### Baseline по HRV, не сырая когерентность

Когерентность сердечного ритма уже кормит серотонин через EMA. Но это state-level контур, не PE. Удивление от HRV — отклонение **сейчас** от **привычного за это время суток** — чистый физический PE. С реальным Polar это самый информативный feeder (минимум субъективного шума). Без Polar → 0, не мешает.

## Связь с прайм-директивой

Рассогласование (sync_error) + декомпозиция каналов PE пишутся раз в час в `data/prime_directive.jsonl`. Endpoint `GET /assist/prime-directive?window_days=30` возвращает агрегат + вердикт тренда (improving / stable / worsening) + среднее по каждому каналу.

Через 2 мес use смотрим не только «падает ли медленная EMA sync_error», но и **какой канал двигал** давление дисбаланса. Если среднее по HRV-каналу (mean_pe_hrv) стабильно около нуля — Polar не подключен. Если среднее self-канала сильно больше среднего user-канала — Baddle больше боится себя чем пользователя (странно, разбираемся).

## Связь с resonance protocol

Friston-loop — **операциональное основание** пяти механик [resonance protocol](world-model.md):

| Механика | Роль Friston-loop |
|---|---|
| Active sync-seeking | Гейт по давлению тишины (silence_pressure); выбор тона обучается через action-memory на изменении дисбаланса |
| System-burnout | Давление дисбаланса — feeder отображаемого выгорания |
| Hebbian decay | Обращение к ноде = предсказание подтверждено |
| Adaptive idle | Множитель простоя от display_burnout с учётом PE |
| Action Memory | Изменение рассогласования (delta_sync_error) в outcome-ноде = post-action PE change |

Без Friston-loop механики работали бы на шуме (скалярный PE или счётчики тишины). С ним — честная минимизация surprise.

## User-side surprise

До этого — Baddle surprise **о** юзере. Но у юзера есть собственный surprise: он встретил неожиданное **в мире**. Когда это случилось — наш baseline (expectation) должен подстроиться быстрее (его модель мира поменялась, не надо держаться старой).

Детектор — три канала, объединённые через OR:

- **Спайк HRV** — падение RMSSD не меньше чем на 1.5 стандартных отклонения от rolling-baseline (5 минут), с фильтром по активности (игнорируется при беге).
- **Текстовые маркеры** — регулярки на русском и английском («вау», «не ожидал», «??», многоточие, капс); порог 0.35.
- **LLM-fallback** — при пограничном regex-скоре (ниже 0.45, текст длиннее 15 символов) лёгкий классификатор возвращает число 0…1, результат кешируется по SHA1 текста.

При detect:
1. **Ускоренный спад baseline** (apply_surprise_boost) на 3 тика — скалярный decay 0.98 → 0.85, векторный 0.97 → 0.80. Baseline быстро подстраивается к новой реальности пользователя.
2. **Event-нода** в графе (actor = user, action_kind = user_surprise). DMN и scout потом могут строить мосты между surprise-нодами и темами — «чему этот человек часто удивляется».

После часа работы:

curl http://localhost:7860/assist/prime-directive?window_days=1

Возвращает среднюю медленную EMA (mean_ema_slow), вердикт тренда и разбивку по каналам (mean_pe_user / mean_pe_self / mean_pe_agency / mean_pe_hrv). Нулевой канал — не ошибка, просто сигнал не пришёл (Polar выключен / agency по дефолту).
