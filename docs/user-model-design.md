# Предиктивная user-модель

Пользователь для Baddle — не тег, а **физическая система** со своей энергетикой, ожиданиями и предсказуемой траекторией состояний. UserState зеркалит Neurochem системы, но кормится другими сигналами: тело, ввод, обратная связь.

Этот документ описывает что **сверх** базовых (дофамин, серотонин, норадреналин) живёт в UserState:

1. Surprise / expectation → **[friston-loop.md](friston-loop.md)** (канонический PE-слой)
2. **Валентность** (valence) — ось «приятно ↔ неприятно»
3. **Зоны активности** — 4 зоны из HRV × движение
4. **Именованные состояния** (named_state) — 10 регионов в двумерном пространстве
5. **Нагрузка и способность** → **[capacity-design.md](capacity-design.md)** (три контура вместо одной шкалы + симулятор дня)

---

## Валентность — ось «приятно / неприятно»

Возбуждение (норадреналин, дофамин) и валентность — **разные оси**. Одинаково высокое возбуждение может быть любопытством (положительная валентность) или стрессом (отрицательная). В одной шкале теряется смысл.

Валентность (UserState.valence ∈ [−1, 1]) — отдельный EMA-скаляр. Источники только явные, объективные:

- 👍 feedback: +0.7
- 👎 feedback: −0.7
- Streak bias: если отказов больше чем принятий на 3+, валентность падает дополнительно. Одного отказа недостаточно, серии хватает чтобы сказать «что-то идёт не так».
- Sentiment чата через light LLM — высокочастотный feeder (каждое сообщение).

HRV сюда не кормит — тело про возбуждение, валентность про ощущение. Разные источники, не смешивать.

## Зоны активности — HRV × движение

HRV один не различает покой и движение: низкая когерентность может быть стрессом сидящего или нормальным состоянием при беге. Читаем акселерометр как независимый канал и комбинируем.

| Движется? | HRV-ок? | Зона | Смысл |
|---|---|---|---|
| нет | да | 🟢 recovery | Здоровое восстановление |
| нет | нет | 🟡 stress_rest | Беспокойство в покое |
| да | да | 🔵 healthy_load | Flow-like нагрузка |
| да | нет | 🔴 overload | Перегрузка, тормози |

Пороги: активно = величина активности ≥ 0.5, HRV-ок = когерентность ≥ 0.5.

Данные: акселерометр Polar H10 (величина без g) или симулятор-слайдер.

Алерты идут только на негативные зоны — recovery и healthy_load позитивные, тишины хватает.

Влияние на именованное состояние: **ось активации** A = 0.7 · cognitive_arousal + 0.3 · min(1, активность/2). Лежащий пользователь с высоким дофамином не попадает в «flow» — тело вето.

## Именованные состояния — 10 регионов в пространстве (тон, активация)

Вместо голых скаляров маппим две производные — **тон** (T, валентность) и **активация** (A) — на именованные регионы. Их 10, распределены как Voronoi-ячейки:

| Имя | Тон | Активация | Когда |
|---|---|---|---|
| вдохновение | 0.90 | 0.95 | творческий подъём |
| поток | 0.85 | 0.90 | оптимальная вовлечённость |
| любопытство | 0.48 | 0.82 | активное исследование |
| благодарность | 0.64 | 0.58 | реальность превзошла ожидания |
| нейтрально | 0.50 | 0.50 | баланс |
| медитация | 0.43 | 0.21 | спокойствие без реакции |
| апатия | 0.35 | 0.25 | низкая вовлечённость |
| стресс | 0.30 | 0.85 | напряжение |
| разочарование | 0.26 | 0.40 | ожидания не оправдались |
| выгорание | 0.10 | 0.40 | критическое истощение |

Маппинг: тон ≈ серотонин, активация ≈ 0.7 · mean(дофамин, норадреналин) + 0.3 · активность.

Именованное состояние — состояние самого пользователя. Не путать с режимом синхронизации (FLOW/REST/PROTECT/CONFESS) — тот про симбиоз, 4 лейбла. Здесь разрешение выше — 10 регионов.

## Нагрузка и способность

UserState отвечает за состояние пользователя; решение «можно ли сейчас взять тяжёлый режим» принимается отдельным слоем — **capacity** ([capacity-design.md](capacity-design.md)). Три параллельных контура (физиологический / эмоциональный / когнитивный) сходятся в capacity-зону (`green` / `yellow` / `red`), которая и работает decision gate'ом. Дневная нагрузка измеряется через шесть observable (задачи начатые/завершённые, переключения контекста, суммарная сложность, прогресс, интерес) — не одной шкалой, а отчётом.

Симулятор плана (`POST /assist/simulate-day`) прогоняет задуманный день через clone UserState: какие контуры перейдут в жёлтую зону к вечеру, какие задачи гейт пропустит, какие нет. Живое состояние не меняется — ответ содержит пошаговую раскладку и прогноз зон на конец дня.

---

## Персистентность

Всё хранится в `data/user_state.json` — скаляры, ожидание, валентность, долгий резерв, HRV-снимок. Загружается при старте через `UserState.from_dict`, сохраняется после каждого `_log_decision`. Continuity между сессиями — ожидания по времени суток и HRV-baseline'ы по времени суток переживают рестарт.
