# Pump (Накачка) — описание реализации

> Справочный материал, перенесён из TODO при завершении v1.6.

## Суть

Smart DC принимает одну ноду и проверяет через диалектику.
Накачка принимает две ноды и ищет **третий элемент C** — мост через двустороннюю экспансию.

## Физическая аналогия

Частицы соединяются если сумма спинов = 360. A=180, B=90, вместе=270.
Добавляем C=90 → 360 → связь замкнулась. C не похож на A или B — он дополняет.

## Отличие от BFS

BFS ищет существующие связи. Pump **генерирует** промежуточные узлы. Не поиск — создание моста.

## Алгоритм

```
Вход: нода A, нода B

Цикл (max_iterations):
  1. Расширить cloud_A (elaborate, expand, ассоциации)
  2. Расширить cloud_B
  3. LLM-as-abstractor:
     "Какой скрытый параметр связывает эти две идеи?"
     → текст C
  4. Confidence = min(cosine(C, centroid_A), cosine(C, centroid_B))
  5. confidence > 0.5 → мост найден
```

## Ключевой момент: LLM как абстрактор

C не содержится в облаках — это новая абстракция. Embedding только для оценки confidence.
"Иммунная система" + "банкротства" → "адаптивность vs жёсткость в ответ на угрозы"

## Место в архитектуре

Примитивная операция наравне со Smart DC:
- Smart DC: "Это правда?" → confidence
- Pump: "Что связывает?" → третий элемент

Подоперация для: Диспут (общая почва), Турнир (скрытые критерии), Разведка (серендипити)
