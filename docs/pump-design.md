# Pump (Накачка) — поиск скрытых мостов

## Идея

SmartDC проверяет одну мысль через диалектику. Pump берёт **две несвязанные идеи** и ищет скрытые оси, на которых обе — точки. Не поиск существующих связей (как BFS/Walk), а **генерация** нового абстрактного элемента.

Аналогия: "иммунная система" + "банкротства" → "адаптивность vs жёсткость", "каскадный отказ", "ложные сигналы". Три оси — три инсайта из одной пары.

## Алгоритм

### 1. Расширение облаков (итеративно)

Каждая идея "накачивается" контекстом — ассоциации, аспекты, следствия. N=5 ассоциаций за итерацию.

```
cloud_A: [идея A, ассоциация_1, ассоциация_2, ...]
cloud_B: [идея B, ассоциация_1, ассоциация_2, ...]
```

Облака растут с каждой итерацией — больше контекста → более глубокие мосты.

### 2. Поиск скрытых осей

LLM-as-abstractor получает оба облака:

> "Найди 3 РАЗНЫХ скрытых параметра, оси или измерения, которые связывают эти две идеи"

Промпт содержит уже найденные оси ("НЕ повторяй, ищи ДРУГИЕ") → каждая итерация даёт новые мосты.

### 3. Верификация каждого моста через SmartDC

Каждый мост проходит полный SmartDC с pump_context:

```
Statement: "Связь: A='идея A' и B='идея B'. Мост: текст_моста"
→ thesis (адвокат) → antithesis (критик) → neutral (аналитик)
→ synthesis (3-4 предложения)
```

### 4. Per-pole analysis — метрики качества

- **lean** = thesis_conf - antithesis_conf. Куда склоняется: >0.05 thesis, <-0.05 anti, иначе balanced
- **tension** = cosine(thesis, antithesis). <0.6 deep (нетривиальный спор), >0.8 trivial
- **quality** = balance × depth × DC confidence

### 5. Накопление и ранжирование

Мосты **накапливаются через все итерации** (не пересоздаются). Дедупликация по тексту. Стоп-условие:
- 3+ моста с quality > 0.4, или
- лучший мост quality > 0.5, или
- исчерпаны итерации

Результат отсортирован по quality.

## UI

Каждый мост отображается с полными метриками:
- **Заголовок** — текст моста (краткий)
- **Метрики** — quality%, lean, tension, DC confidence
- **Thesis / Antithesis** — раскрываемый блок (FOR / AGAINST / NEUTRAL)
- **Synthesis** — полный текст синтеза SmartDC (без обрезки)
- **Save bridge** — сохраняет краткую связь как ноду
- **Save synthesis** — сохраняет полный синтез SmartDC
- После Save кнопка → "Saved ✓", панель остаётся открытой

Сохранённая нода линкуется к обоим источникам (A и B).

## Место в архитектуре

Pump — **поворот** конуса. Не расширение (Brainstorm) и не сужение (SmartDC), а смена оси. Два конуса (от двух идей) → поиск общего пространства → новое направление, невидимое изнутри каждого конуса.

Составлен из существующих примитивов:
- Brainstorm → расширение облаков
- SmartDC → верификация мостов
- Embedding similarity → оценка покрытия

Планируется: Pump в autorun для Scout mode (автономный поиск мостов между далёкими нодами = DMN).
