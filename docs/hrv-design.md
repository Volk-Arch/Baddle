# HRV Integration — тело как вход в пространство возможностей

## Идея

CognitiveState управляет конусом через prediction error (чисто информационный сигнал). Но конус — метафора не только когнитивная. Тело — многоуровневый резонатор (Бентов), и его ритмы напрямую маппятся на параметры конуса.

> **Важно (после v8d Симбиоза):** HRV теперь **не модулирует** внутреннюю
> нейрохимию системы напрямую. HRV — сигнал тела **пользователя**; он
> обновляет `UserState.serotonin` (coherence) и `UserState.norepinephrine`
> (stress). SystemState (Neurochem) эволюционирует по собственной динамике
> графа. См. [symbiosis-design.md](symbiosis-design.md).
>
> **Activity как отдельный канал (из прототипа Polar+accelerometer):**
> `UserState.activity_magnitude` — L2-magnitude движения. В паре с
> `coherence` даёт 4-зонную классификацию (`activity_zone` derived):
> recovery / stress_rest / healthy_load / overload. См.
> [user-model-design.md](user-model-design.md) секция 1c.

## Источники данных — разные устройства, разная семантика

Текущий `hrv_manager.py` построен под **Polar-like модель**: непрерывный
RR-поток + accelerometer как single magnitude. Это не подходит «из
коробки» для других устройств. Вот реальность:

| Источник | RR | Частота | Accel | HR | Sleep |
|----------|-----|---------|-------|-----|-------|
| **Simulator** (сейчас) | эмулируется каждый beat | ~1/s | скалярный слайдер | derived из RR | нет |
| **Polar H10** | каждый beat | ~1/s непрерывно | 50 Hz X/Y/Z | derived | нет |
| **Apple Watch** | sparse samples | раз в минуты | отдельный поток 50 Hz | 5-секундный stream | через HealthKit |
| **Oura / Garmin** | raw RR часто недоступен | — | разные | continuous | yes (detailed) |
| **Phone IMU** | — | — | 50-100 Hz | — | — |

**Следствие:** один `hrv_manager` с `rr_buffer + activity_magnitude scalar`
хорошо моделирует Polar и симулятор. Для Apple Watch / Oura нужен
**адаптер** который конвертирует редкие RR-samples + непрерывный HR в
те же signals что ожидает `hrv_to_baddle_state`:
- `coherence` при разреженных RR → альтернативная формула (HR
  variability от HR timeseries, не RR-to-RR)
- `activity_magnitude` из Watch motion & workouts API
- `sleep quality` (новое) → feed in `UserState.long_reserve` overnight recovery

Это **отдельная задача** в TODO (экосистема/интеграции, deferred). Не
ломает текущую архитектуру — добавляется как ещё один `mode` в
`hrv_manager.start(mode="apple_watch")`.

## Push vs Pull синхронизация

С активити-зонами выяснилась важная тонкость: до периодического push'а
`UserState.hrv_*` обновлялся **только** при явном вызове `/hrv/metrics`
endpoint (pull). Если UI не поллит — UserState устаревает, zone-alerts
и sync_regime считаются на старом coherence.

**Сейчас:** `CognitiveLoop._check_hrv_push()` каждые 15с пушит свежий
baddle_state из hrv_manager в UserState. Это гарантирует что
`activity_zone`, `named_state`, `sync_regime` согласованы с реальным
телом ±15 секунд, независимо от UI-активности.

HRV (вариабельность сердечного ритма) — самый доступный физиологический сигнал, который отражает баланс симпатика/парасимпатика, когерентность дыхания и сердца, уровень восстановления. Polar H10 через BLE даёт RR-интервалы в реальном времени.

**Два режима работы:**
- **Автономный** (текущий) — Baddle управляет конусом через информационный контур (surprise → precision → params)
- **Резонансный** — тело управляет конусом через физиологический контур (HRV → θ/φ → params)

Переключение плавное, не бинарное.

## Параметры конуса

| Параметр | Физический аналог | Эффект |
|----------|------------------|--------|
| **θ (угол/ширина)** | HRV-coherence, альфа-тета активность | Расширение → креативность, неопределённость. Сужение → фокус, уверенность |
| **φ (ориентация)** | Вектор внимания, смена контекста | Поворот → доступ к другому подпространству без изменения глубины |
| **ω (высота)** | Горизонт предсказания | Удлинение → стратегическое мышление. Укорочение → реакция на "здесь и сейчас" |
| **Качество поверхности** | Когерентность ритмов | Гладкий → стабильная траектория. Шероховатый → когнитивное трение |

### Формулы адаптации

```
θ_{t+1} = θ_t - α·∇_θ(уверенность) + β·ε(энтропийный шум)
φ_{t+1} = φ_t + γ·∇_φ(новизна) + δ·∇_φ(контекст)
Постериор = BayesUpdate(Приор(φ_t, θ_t), Данные)
```

## Mapping: HRV → θ

```python
def calculate_theta(hrv_coherence, rmssd):
    # Базовый θ от когерентности
    if hrv_coherence > 0.7:
        theta_base = 0.8  # Широкий: система стабильна, можно исследовать
    elif hrv_coherence > 0.4:
        theta_base = 0.5  # Средний
    else:
        theta_base = 0.2  # Узкий: нестабильно, нужен фокус

    # Коррекция от RMSSD (восстановление)
    if rmssd < 20:
        theta_base *= 0.7  # Принудительное сужение

    return clamp(theta_base, 0.1, 0.9)
```

## Mapping: HRV → φ (разрешение поворота)

```python
def calculate_phi_adjustment(lf_hf_ratio, hr_trend, theta):
    if lf_hf_ratio > 3.0:       # Симпатика доминирует
        return {'allow_shift': False}
    if hr_trend > 5:             # Пульс растёт быстро
        return {'allow_shift': False}
    if theta > 0.6 and 0.5 < lf_hf_ratio < 2.0:
        return {'allow_shift': True, 'max_delta_phi': 0.3}
    return {'allow_shift': True, 'max_delta_phi': 0.1}
```

## Пять состояний (расширение четырёх из Horizon)

| Состояние | θ | φ | Байесовский маркер | Поведение |
|-----------|---|---|-------------------|-----------|
| **STABILIZE** | Сужается→0 | Фиксируется | Q-factor ↑, энтропия ↓ | Калибровка, сброс шума |
| **FOCUSED** | Узкий | Стабилен | P(H) > 0.8, низкая энтропия | Готовность к действию, блокировка ветвления |
| **EXPLORE** | Широкий | Свободен | 0.4 < P < 0.8, рост дисперсии | Активный скан, пометка мостов |
| **SHIFT** | Средний | Δφ > threshold | Смена контекста, пересчёт приоров | Перестройка связей |
| **CONFLICT** | Любой | Любой | Несовместимые приоры, merge_fail | Остановка, запрос разрешения |

### Триггеры с гистерезисом (без дребезга)

```
FOCUSED → EXPLORE:  confidence < 0.7
EXPLORE → FOCUSED:  confidence > 0.85  (разрыв 0.15)

Взвешенный скоринг:
score = w₁·Δconfidence + w₂·entropy_rate + w₃·user_input + w₄·timer_drift
Переход: score > threshold И удерживается N тиков

Debounce для φ: устойчивое смещение в окне 3-5 шагов
```

### HRV-условия переходов

| Состояние | Вход | Выход |
|-----------|------|-------|
| STABILIZE | coherence < 0.3 ИЛИ rmssd < 15 | coherence > 0.5 в течение 60 сек |
| FOCUSED | 0.4 < coherence < 0.7 И lf_hf < 2.5 | coherence > 0.7 ИЛИ coherence < 0.3 |
| EXPLORE | coherence > 0.7 И rmssd > 30 | coherence < 0.5 ИЛИ lf_hf > 3.0 |
| SHIFT | Ручной ИЛИ rsa_amplitude ↑ 50%/30с | phase_lock стабилен 90 сек |
| CONFLICT | coherence ↓ 40%/30с ИЛИ hr ↑ 15 bpm | coherence > 0.4 И hr стабилен 60 сек |

## Метрики с Polar H10

| Метрика | Формула | Окно | Значение для конуса |
|---------|---------|------|-------------------|
| HR | 60000 / mean(RR) | 10с | Общий уровень активации |
| RMSSD | sqrt(mean((RR[i+1]-RR[i])²)) | 60с | Парасимпатический тонус |
| SDNN | std(RR) | 60с | Общая вариабельность |
| HRV-coherence | peak/total power (0.04-0.26 Гц) | 60с | **Ключевая**: синхронизация дыхания↔сердца |
| LF/HF | power(0.04-0.15)/power(0.15-0.4) | 120с | Баланс симпатика/парасимпатика |
| RSA | Амплитуда модуляции HR на частоте дыхания | 60с | Качество связи дыхание↔сердце |

## Резонансные метки (по Бентову)

Дополнительные индикаторы помимо байесовских весов:

- **Q-factor** = сигнал/шум в текущем срезе графа. Высокий → узкий стабильный пик, низкий → широкий поиск
- **Phase lock** = синхронизация тезисов. Высокая → готовность к коммиту
- **Harmonic bleed** = расширение θ захватывает смежные домены. Система помечает как потенциальные мосты, не шум. Отличает "творческий резонанс" от "когнитивного расфокуса"

## Тело как многоуровневый резонатор

| Уровень | Частота | Влияние на конус |
|---------|---------|-----------------|
| Нейроны (бета/гамма) | 14-100 Гц | Сужение θ, фиксация φ (аналитический) |
| Сердце (HRV) | ~1 Гц (модуляция 0.1-0.4 Гц) | Стабилизация θ, плавность поворотов φ |
| Дыхание | 0.2-0.3 Гц | Расширение θ, замедление внутри конуса |
| Спинномозговая жидкость | 0.05-0.1 Гц | Глобальный поворот φ, доступ к "тихим" веткам |

Когда ритмы входят в фазовую синхронизацию → тело = высокодобротный осциллятор → повышенная чувствительность к тонким паттернам.

## Медитация как ручная настройка конуса

| Практика | Механизм | Эффект на конус |
|----------|----------|----------------|
| Наблюдение дыхания | Синхронизация дыхание→HRV | Плавное расширение θ |
| Сканирование тела | Рост когерентности микровибраций | Обнаружение искажений φ |
| Открытый мониторинг | Снижение Q, расширение полосы | Максимальное θ, креативность |
| Фокус на объекте | Повышение Q, одна частота | Минимальное θ, максимальная глубина ω |
| Пауза между мыслями | Условия для стоячей волны | Сброс φ, перепозиционирование |

## Два режима в одной системе

- **Автономный**: исполнение. Baddle проходит граф, считает апдейты, мерджит. Быстро, воспроизводимо
- **Резонансный**: настройка и экспансия. Когда граф зашёл в тупик или нужны инсайты — тело меняет θ/φ, система переключается со "решать" на "сканировать"

```yaml
mode:
  type: "autonomous" | "resonant" | "hybrid"
  transition_guards:
    auto_to_resonant:
      - hrv_coherence < 0.35
      - conflict_duration > 120s
    resonant_to_auto:
      - hrv_coherence > 0.75
      - phase_lock_stable > 90s
```

В резонансном режиме меняется стратегия:
- argmax(confidence) → sample(distribution, temperature=θ)
- shortest_path → resonant_walk (гармонические связи)
- hard_merge → standing_wave_buffer (удержание противоречий до синхронизации)

## Калибровка

10-минутная сессия для определения персональных порогов:
- Спокойное дыхание → baseline coherence, RMSSD, LF/HF
- Пороги = mean ± 0.5·std
- Сохраняется в конфиг, обновляется со временем

## Шаги реализации

1. **MVP**: Polar H10 → RR-интервалы → консоль (bleak BLE)
2. **Alpha**: Расчёт θ из HRV, ручное переключение состояний
3. **Beta**: Автоматические переходы + UI-визуализация конуса
4. **Production**: Адаптивная калибровка + интеграция с tick cycle
