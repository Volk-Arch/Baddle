# NAND-архитектура — один примитив вместо всей логики

## Идея

Классический подход к «умной» системе — прописать разные ветки для разных задач: «если это выбор, делай так; если исследование, иначе; если противоречие, третий путь». Это работает, но ручная логика — каждая ветка отдельный код, отдельная калибровка, отдельный баг.

Другой подход: **один оператор различения** — **мера различия двух идей** (d = distinct(A, B)), возвращающая число от 0 (идентичны) до 1 (максимально различаются). Вся булева логика выводится из зон этого числа — не прописывается отдельно, а **возникает** из значений.

Аналогия: NAND-гейт в электронике — один элемент, из которого собрана вся логика компьютера. Различение (distinct) — вероятностный NAND для мышления.

> Концепция из статьи [alphaxiv.org/abs/2603.21852v2](https://www.alphaxiv.org/abs/2603.21852v2).

## Два двигателя реальности

Система живёт не на 0 и 1. Она живёт на **рельефе** — пространстве состояний, по которому скатывается к устойчивому. 0 и 1 — флажки на карте, которые мы ставим когда нужно нажать кнопку. Сам рельеф задаётся двумя принципами:

| Начало | Правило | Что измеряет |
|---|---|---|
| **Структура** | различие между двумя объектами (A ≠ B) | Расстояние в пространстве состояний |
| **Действие** | различие объекта с самим собой во времени (A ≠ A) | Скорость изменения во времени |

**Различие — базовое, тождество — производное.**

### Единица помнит путь

В реальной системе единица, полученная из пары (0,0), и единица, полученная из пары (0,1), — **не одно и то же**:

- **Близость к переключению** — из (0,1) до нуля система дойдёт быстрее
- **Энергетическая стоимость** — удержание единицы при (0,1) требует постоянного расхода, при (0,0) — нет
- **Реакция на шум** — (0,0) → 1 устойчиво к флуктуациям, (0,1) → 1 уже «на взводе»

Гистерезис, память пути, энергетическая история. Baddle различает два подтипа единицы:

- **покой** (1_rest, от пары 0,0) — потенциал, низкое рассогласование с пользователем (sync_error), готов сканировать
- **напряжение** (1_held, от пары 0,1) — активность, высокая чувствительность, требует быстрой валидации

Сложность рождается не из ресурсов, а из композиции одного правила с контекстом и историей.

### Философский корень

- различие с самим собой (A ≠ A) — рождение наблюдения (субъект обнаружил разницу в самом себе)
- различие с другим (A ≠ B) — рождение объекта (есть я и другое)
- рекурсия различия (distinct(distinct(A,A), distinct(A,B))) — рефлексия второго порядка

## Что делает distinct

**Мера различия** (d = distinct(A, B)) лежит в диапазоне от нуля (идентичны) до единицы (максимально разные). Технически — косинусное расстояние между embedding'ами или расхождение Кульбака–Лейблера; в Baddle используется косинус.

Из различия (d) выводится **правдоподобие соответствия** (L = exp(−γ·d)): при d → 0 правдоподобие L → 1 (согласие), при d → 1 правдоподобие L → 0 (конфликт). Здесь **чувствительность байесовского обновления** (γ) — адаптивный параметр, см. ниже.

Вес ноды обновляется в **логарифмическом пространстве** (для численной устойчивости): новый логарифмический вес = старый − γ·d + логарифм приора. Затем все веса пропускаются через softmax с **температурой выбора** (T), получая распределение вероятностей.

### Три зоны

Значения различия (d) разбиваются тремя порогами на смысловые зоны:

- d ниже **порога согласия** (τ_in = 0.3 по умолчанию) → ЗОНА СОГЛАСИЯ, связь подтверждается, вес усиливается
- d между τ_in и **порогом конфликта** (τ_out = 0.7 по умолчанию) → ЗОНА ИССЛЕДОВАНИЯ, неопределённость
- d выше τ_out → ЗОНА КОНФЛИКТА, ветвление или архив

Defaults и реализация — [src/main.py::distinct_decision](../src/main.py), [src/horizon.py::CognitiveState](../src/horizon.py).

### Адаптивные параметры γ и T

**Чувствительность** (γ) выводится из нейрохимии: γ = 2.0 + 3.0 · норадреналин · (1 − серотонин). Высокая мобилизация (norepinephrine, NE) + отсутствие стабильности (низкий serotonin, S) → γ растёт → Bayes строже наказывает различие. Детали — [neurochem-design.md](neurochem-design.md).

**Температура выбора** (T) адаптируется по сдвигу убеждений между тиками через расхождение Кульбака–Лейблера. Большой сдвиг весов → T растёт (мягкий выбор, шанс альтернативам); стабильно → T падает (резкий аргмакс). Реализация — [src/horizon.py](../src/horizon.py).

Пороги (τ_in, τ_out) — статичные defaults с тюнингом per-mode через preset. HRV влияет на них косвенно через нейрохимию (norepinephrine / serotonin) и дальше через γ.

## Git-коммит — каждый шаг версионирован

Каждый байесовский шаг пишется как коммит с полной трассой. Hash-chain через ссылку на родителя позволяет восстановить порядок тиков. Структура и схема записи — [episodic-memory.md](episodic-memory.md), [ontology.md § state_graph](ontology.md).

Что это даёт:
- **Трассировка решений** — каждый шаг аудируем
- **Восстановление порядка** через цепочку родительских хешей
- **Отладка мысли** — хвост state_graph становится хронологией рассуждений

State_graph линейный. Зона конфликта (d выше порога конфликта) обрабатывается действием диалектической проверки (smartdc) или сравнения (compare) на том же уровне, без forking веток.

## Зоны как семантические ярлыки

Три зоны различия (d) — не абстрактные «аналоги булевой логики», а именованные ветки кода в [nand.py](../src/nand.py) § «emergent routing by zone density»:

| Зона | Что делает tick |
|---|---|
| d ниже τ_in | коллапс — сливает похожие ноды в один кластер |
| d между τ_in и τ_out | pump / elaborate — углубляет уровень между ними |
| d выше τ_out | smartdc / compare — запускает диалектическую проверку |

Ранняя редакция главы декларировала формулы AND/OR/XOR/NOR как «эмерджентные операторы» над различением. Пилот 2026-04-24 (см. [planning/TODO.md § бэклог](../planning/TODO.md)) проверил три формулы для ранжирования pump-мостов на 45 наблюдениях — все три дали корреляцию с качеством SmartDC в шуме (модуль ρ ≤ 0.17). Причина фундаментальная: embedding-похожесть и диалектическое качество — разные измерения, и никакая формула свёртки `(d_a, d_b) → scalar` это не чинит. Эксперимент закрыт. В коде зоны остались как обычные if/elif ветки — так читаемее.

**Конфликт ≠ ошибка.** Различие около 0.5 — не баг, а сигнал «оба варианта уместны в одном пространстве». Система запускает диалектическую проверку — конфликт как источник информации, не её отсутствие.

## Три слоя обработки

1. **Эмбеддинги (маршрутизация).** Различение (distinct) считается в пространстве векторов, без текста → решение: зона согласия / исследования / конфликта.
2. **Символическое ядро.** Байесовское обновление через произведение γ·d, маршрутизация по зонам в `nand.tick_emergent`, трекинг происхождения состояния (1_rest vs 1_held).
3. **Отложенный текст.** LLM только когда нужен текст пользователю. Система мыслит эмбеддингами, говорит текстом.

Эмбеддинг (порядка миллисекунд) первичен. Текст (секунды LLM-вызова) — производный, нужен только для UI. См. [thinking-operations § Embedding-first](thinking-operations.md#embedding-first-brainstorm).

## sync_error как связка

**Рассогласование с пользователем** (sync_error) = различие между предсказанием модели и действием пользователя (d(model_prediction, user_action)). Это не абстрактная ошибка предсказания, а мера расхождения с **конкретным** человеком:

- Высокий sync_error → система не понимает пользователя → расширить поиск, задать вопрос
- Низкий sync_error → точное предсказание → сузить, действовать уверенно

В связке с происхождением состояния (state_origin):
- покой (1_rest) при низком sync_error — медленные переходы, готов сканировать
- напряжение (1_held) при высоком sync_error — уже в работе, требует быстрой валидации

Полный предиктивный контур — [friston-loop.md](friston-loop.md).

## Два режима мышления

**Активный фокус** ([CognitiveState](../src/horizon.py), т.е. когнитивное состояние):
- d ниже τ_in → коллапс сливает похожие ноды
- d выше τ_out → диалектическая проверка (smartdc / compare)
- чувствительность (γ) адаптивна через нейрохимию (norepinephrine / serotonin); температура (T) — через KL между тиками; пороги (τ_in / τ_out) — статичные defaults плюс mode preset

**Фоновый поиск** (DMN, default mode network):
- ночная творческая фаза ищет пары близкие в embedding-пространстве, но далёкие в графе ([cognitive_loop._rem_creative](../src/cognitive_loop.py))
- pump-мосты между парами ([pump_logic.pump](../src/pump_logic.py)) в continuous-режиме
- детали — [dmn-scout-design.md](dmn-scout-design.md)

Переключение: высокая нагрузка → активный фокус доминирует, фоновый поиск в low-priority. Покой → фоновый поиск сканирует граф, готовит инсайты через hash-chain в state_graph.

## Итог

Единый примитив — мера различия (d = distinct(A, B)) — даёт базовую единицу для тридцати с лишним мест кода: novelty-фильтры, байесовское обновление через γ·d, маршрутизация зон в `nand.tick_emergent`, гейт релевантности для profile-injection. Зоны — именованные ветки, не магические операторы. Чувствительность (γ) адаптивна через нейрохимию, температура (T) — через KL между тиками. State_graph — линейный hash-chain каждого тика, не древовидные ветки.

Претензия на «эмерджентную булеву логику из различения» была попыткой — проверили, не подтвердилось (см. pilot). Примитив работает, но AND/OR/XOR как вычисления над парой различий не дают сигнала выше шума на реальных задачах. Обычные `if d < τ: ...` читаются и работают лучше.

Beta-распределения для приоров (вероятность + уверенность в ней), «режим сенсорной депривации» (различение без LLM-генерации нового), полная миграция текст → эмбеддинги (сейчас частичная — embedding-first brainstorm работает, но не везде).
