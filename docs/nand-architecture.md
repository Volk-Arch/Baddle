# NAND-архитектура — один примитив вместо всей логики

## Идея

Классический подход к «умной» системе — прописать разные ветки для
разных задач: «если это выбор, делай так; если исследование, иначе;
если противоречие, третий путь». Это работает, но ручная логика —
каждая ветка отдельный код, отдельная калибровка, отдельный баг.

Другой подход: **один оператор различения** `distinct(A, B)` возвращает
число от 0 (идентичны) до 1 (максимально различаются). Вся булева
логика выводится из зон этого числа — не прописывается отдельно,
а **возникает** из значений.

Аналогия: NAND-гейт в электронике. Один элемент, из которого собрана
вся логика компьютера. `distinct()` — вероятностный NAND для мышления.

> Концепция из статьи [alphaxiv.org/abs/2603.21852v2](https://www.alphaxiv.org/abs/2603.21852v2).

## Два двигателя реальности

Система живёт не на 0 и 1. Она живёт на **рельефе** — пространстве
состояний, по которому скатывается к устойчивому. 0 и 1 — флажки на
карте, которые мы ставим когда нужно нажать кнопку. Сам рельеф задаётся
двумя принципами:

| Начало | Правило | Что измеряет |
|---|---|---|
| **Структура** | `A ≠ B` | Расстояние в пространстве состояний |
| **Действие** | `A ≠ A` | Скорость изменения во времени |

**Различие — базовое, тождество — производное.**

### 1 ≠ 1 — состояние помнит путь

В реальной системе `1` от `(0,0)` и `1` от `(0,1)` — **не одно и то же**:

- **Близость к переключению** — из `(0,1)` до 0 система дойдёт быстрее
- **Энергетическая стоимость** — удержание 1 при `(0,1)` требует
  постоянного расхода, при `(0,0)` — нет
- **Реакция на шум** — `(0,0) → 1` устойчиво к флуктуациям,
  `(0,1) → 1` уже «на взводе»

Гистерезис, память пути, энергетическая история. Baddle различает два
подтипа единицы:

- `1_rest` (от 0,0) — потенциал, покой, низкий sync_error, готов сканировать
- `1_held` (от 0,1) — активность, напряжение, высокая чувствительность,
  требует быстрой валидации

Сложность рождается не из ресурсов, а из композиции одного правила
с контекстом и историей.

### Философский корень

- `A ≠ A` — рождение наблюдения (субъект обнаружил разницу в самом себе)
- `A ≠ B` — рождение объекта (есть я и другое)
- `distinct(distinct(A,A), distinct(A,B))` — рефлексия второго порядка

## Что делает distinct

`d = distinct(A, B)` ∈ [0, 1]. Технически — cosine distance между
embedding'ами или KL-дивергенция. В Baddle — cosine.

Из `d` выводится правдоподобие: `L = exp(−γ·d)` — при `d → 0` имеем
`L → 1` (согласие), при `d → 1` имеем `L → 0` (конфликт).

**Байес в лог-пространстве** (стабильно):

```
logW_new = logW_old − γ·d + log(prior)
W_new = softmax(logW_new / T)
```

`γ` — чувствительность. `T` — температура выбора. Оба **адаптивные**.

### Три зоны

```
d < τ_in        → CONFIRM   (связь подтверждена, усиление)
τ_in < d < τ_out → EXPLORE   (зона неопределённости)
d > τ_out       → CONFLICT  (ветвление или архив)
```

Пороги калибруются через HRV (см. ниже).

### γ и T

**γ** (`neuro.gamma`) выводится из нейрохимии:
`γ = 2.0 + 3.0 · norepinephrine · (1 − serotonin)`. Высокая NE (мобилизация) +
низкий S (нет стабильности) → γ растёт → Bayes строже наказывает distance.
Детали — [neurochem-design.md](neurochem-design.md).

**T** адаптируется по сдвигу убеждений между тиками:
`T ← T₀ · (1 + α · KL(W_t ‖ W_{t−1}))`. Большой сдвиг → T растёт (мягкий
выбор, шанс альтернативам); стабильно → T падает.

`τ_in = 0.3`, `τ_out = 0.7` — статичные defaults, тюнинг per-mode через
preset. Автокалибровка из HRV напрямую не реализована — HRV влияет через
`neuro.norepinephrine` / `neuro.serotonin` и дальше через γ.

## Git-коммит — каждый шаг версионирован

Каждый байесовский шаг пишется как коммит с полной трассой. Структура
state_graph — в [episodic-memory.md](episodic-memory.md). Типичная
запись:

```json
{
  "hash": "a3f8c2d1", "parent": "7b2e4f09",
  "distinct_trace": {"d": 0.342, "decision": "EXPLORE"},
  "weight_delta": {"7": 0.12, "3": -0.05},
  "gamma": 2.34, "temperature": 0.12,
  "thresholds": {"tau_in": 0.28, "tau_out": 0.71},
  "physio_state": {"hrv_coherence": 0.73, "stress": 0.31}
}
```

Что это даёт:
- **Трассировка решений** — каждый шаг аудируем
- **Hash-chain** через `parent` — восстановим порядок тиков
- **Отладка мысли** — хвост state_graph становится хронологией рассуждений

Ветвление на ветки при `d > τ_out` — архитектурная идея, не реализованная:
state_graph сейчас линейный, conflict-зона обрабатывается через действие
`smartdc` / `compare` на том же уровне, без forking.

## Зоны как семантические ярлыки

Три зоны `d` — не абстрактные «аналоги булевой логики», а именованные
ветки кода в [tick_nand.py](../src/tick_nand.py) § «emergent routing by
zone density»:

| Зона | Что делает tick |
|---|---|
| `d < τ_in` | `collapse` — сливает похожие ноды в один кластер |
| `τ_in < d < τ_out` | `pump` / `elaborate` — углубляет уровень между ними |
| `d > τ_out` | `smartdc` / `compare` — запускает диалектическую проверку |

Ранняя редакция главы декларировала формулы `AND/OR/XOR/NOR` как
«эмерджентные операторы» над `distinct()`. Пилот 2026-04-24 (см.
[planning/TODO.md 10](../planning/TODO.md))
проверил три формулы для ranking pump-bridges на 45 наблюдениях — все
три дали корреляцию с SmartDC quality в шуме (|ρ| ≤ 0.17). Причина
фундаментальная: embedding similarity и SmartDC quality — разные
измерения, и никакая формула свёртки `(d_a, d_b) → scalar` это не
чинит. Эксперимент закрыт. В коде зоны остались как **if/elif ветки**,
обычный Python — так читаемее.

**Конфликт ≠ ошибка.** `d ≈ 0.5` — не баг. Сигнал «оба варианта да
в одном пространстве». Система запускает smartdc для валидации —
конфликт источник информации, не её отсутствие.

## Pipeline — три слоя

```
1. Эмбеддинги (маршрутизация)
   distinct() считается в пространстве векторов, без текста
   → решение CONFIRM / CONFLICT / EXPLORE

2. Символическое ядро
   Bayes update через γ·d, zone-routing в tick_nand
   → отслеживается state_origin (1_rest vs 1_held)

3. Отложенный текст (LLM только при необходимости)
   текст для пользователя, не для системы
   система мыслит эмбеддингами, говорит текстом
```

Эмбеддинг (~1 мс) первичен. Текст (~2 сек LLM-call) — производный,
нужен только для UI. См. [thinking-operations § Embedding-first](thinking-operations.md#embedding-first-brainstorm).

## sync_error как связка

`sync_error = d(model_prediction, user_action)` — не абстрактная ошибка
предсказания, а **мера рассинхрона с конкретным пользователем**:

- Высокий → система не понимает юзера → расширить поиск, задать вопрос
- Низкий → точное предсказание → сузить, действовать уверенно

В связке с `state_origin`:
- `1_rest` (низкий sync_error) — медленные переходы, готов сканировать
- `1_held` (высокий sync_error) — уже в работе, требует быстрой валидации

Полный предиктивный контур — [friston-loop.md](friston-loop.md).

## Два режима мышления

Активный фокус (CognitiveState):
- d < τ_in → `collapse` сливает ноды
- d > τ_out → `smartdc` / `compare` запускает проверку
- γ адаптивна через `neuro.norepinephrine` / `neuro.serotonin`;
  T — через KL между тиками; τ_in / τ_out статичные defaults + mode preset

Фоновый поиск (DMN):
- REM creative ищет пары близкие в embedding-space но далёкие в графе
  (`cognitive_loop._rem_creative`)
- Pump-bridge между парами (`pump_logic.pump`) в DMN-continuous
- Детали — [dmn-scout-design.md](dmn-scout-design.md).

Переключение: высокая нагрузка → Horizon доминирует, DMN в low-priority.
Покой → DMN скан, готовит insights через state-graph hash-chain.

## Итог

Один примитив `d = distinct(A, B)` даёт базовую единицу для 33+ мест
кода: novelty-фильтры, Bayes update через `γ·d`, routing зон в tick_nand,
relevance-гейт для profile-injection. Зоны — именованные ветки, не
магические операторы. γ адаптивна через нейрохимию, T — через KL
между тиками. State_graph — линейный hash-chain каждого тика (не
древовидные ветки).

Претензия на «эмерджентную булеву логику из `distinct()`» была
попыткой — проверили, не подтвердилось (см. pilot). Primitive работает,
но AND/OR/XOR как вычисления над `d_a, d_b` → scalar не дают сигнала
выше шума на реальных задачах. Обычные `if d < τ: ...` читаются и
работают лучше.

**Открыто** ([planning/TODO.md](../planning/TODO.md)):
Beta-распределения для priors (вероятность + уверенность в ней),
«режим сенсорной депривации» (distinct без LLM-генерации нового),
полная миграция текст → эмбеддинги (сейчас частичная — embedding-first
brainstorm работает, но не везде).
