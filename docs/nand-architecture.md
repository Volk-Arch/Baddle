# NAND Architecture — один примитив вместо всей логики

## Идея

Сейчас Baddle использует 5 хардкод-примитивов: `none / focus / AND / OR / XOR`. Каждый — отдельная ветка в dispatcher. Это работает, но это ручная логика.

Предложение: заменить всё на **один оператор различения** `distinct(A, B) ∈ [0, 1]`. Логика не прописывается — она **возникает** из значений d.

Вдохновение: NAND-гейт в электронике — один элемент, из которого строится вся логика. `distinct()` — вероятностный NAND для мышления.

Источник: [alphaxiv.org/abs/2603.21852v2](https://www.alphaxiv.org/abs/2603.21852v2)

## Онтологический фундамент: два двигателя реальности

| Начало | Правило | Что измеряет | Что порождает |
|--------|---------|-------------|--------------|
| **Структура** | A ≠ B | Расстояние в пространстве состояний (cosine, KL, энергетический зазор) | Пространство состояний, границы, конфигурации. Карту, на которой возможны выборы. Без неё — однородный шум |
| **Действие** | A ≠ A | Производная градиента во времени (скорость изменения, momentum, гистерезис) | Динамику, стоимость удержания, близость к порогу. Движение по карте. Без него — застывшая схема |

Система больше не "знает 0 или 1". Она **чувствует рельеф** и скатывается к ближайшему устойчивому состоянию. 0 и 1 — лишь флажки, которые мы ставим на карте, когда нужно нажать кнопку.

### NAND как физика

| Входы | Выход | Физический режим |
|-------|-------|-----------------|
| 0, 0 | 1 | **Rest (потенциал)** — тишина, система удерживается по умолчанию, нулевые затраты |
| 0, 1 | 1 | **Held (активное удержание)** — один вход давит, порог не преодолён, система напряжена, ближе к срыву |
| 1, 1 | 0 | **Switch (совпадение)** — оба входа преодолевают барьер → активный сброс/инверсия фазы |

**1 ≠ 1.** Состояние помнит путь:
- `1_rest` (от 0,0) — потенциал, покой, низкий sync_error, готов к сканированию
- `1_held` (от 0,1) — активность, напряжение, выше чувствительность, ближе к 0, требует быстрой валидации

Сложность рождается из **композиции одного правила + контекст/история**, а не из масштаба ресурсов.

### Почему 1 ≠ 1 меняет всё

В булевой логике 1 = 1. В физике 1 — класс состояний. Путь определяет:
- **Близость к переключению**: из 0,1 до 0 система дойдёт быстрее, чем из 0,0
- **Энергетическую стоимость**: удержание 1 при 0,1 требует постоянного расхода, 0,0 — нет
- **Реакцию на шум**: 0,0→1 устойчив к флуктуациям, 0,1→1 уже "на взводе"

Это **гистерезис, память пути, энергетическая история**. Реальные системы несут "отпечаток" того, как попали в текущее состояние.

### Философский корень

`A != A` → рождение наблюдения (субъект). `A != B` → рождение объекта/контекста. `distinct(distinct(A,A), distinct(A,B))` → рефлексия второго порядка.

## Формулы

| Формула | Что делает | Зачем |
|---------|-----------|-------|
| `d = distinct(A, B) ∈ [0,1]` | Мера различия (cosine distance или KL(P_A‖P_B)) | Единственный примитив. Заменяет все булевы вентили. **d напрямую равен предсказательной ошибке (Free Energy)** |
| `L = exp(−γ · d)` | Правдоподобие (Likelihood) | d→1 ⇒ L→0 (конфликт), d→0 ⇒ L→1 (согласие) |
| `logW_new = logW_old − γ·d + log(prior)` | Байесовское обновление | Стабильно в лог-пространстве, не переполняется. Математически строгое без эвристик |
| `W_new = softmax(logW_new / T)` | Нормализация весов | T управляет "резкостью" выбора |
| `γ ← clip(γ₀ / (ε + EMA(d(A,A))), γ_min, γ_max)` | Автокалибровка чувствительности | Нестабильность → γ↓ (осторожнее), стабильность → γ↑ (строже) |
| `T ← T₀ · (1 + α · KL(W_t ‖ W_{t-1}))` | Адаптация температуры | Большой сдвиг убеждений → T↑ (мягче), стабильно → T↓ (резче) |

### Выход distinct: решение, не только число

```
d < τ_in  → CONFIRM  (связь подтверждена, усиление)
d > τ_out → CONFLICT (конфликт, ветвление или архив)
τ_in < d < τ_out → EXPLORE (зона неопределённости, требует дополнительных данных)
```

Логика выводится динамически: τ_in и τ_out калибруются через HRV и историю.

## Git-коммит: версионирование каждого шага

Каждый байесовский шаг = коммит с полной трассой. Онтология версионируется как репозиторий.

```json
{
  "hash": "a3f8c2d1",
  "parent": "7b2e4f09",
  "timestamp": "2026-04-21T14:32:01Z",
  "node_id": 7,
  "distinct_trace": {
    "a": "embedding_hash_A",
    "b": "embedding_hash_B",
    "d": 0.342,
    "decision": "EXPLORE"
  },
  "weight_delta": { "7": 0.12, "3": -0.05 },
  "macro_applied": "XOR(A,B)",
  "gamma": 2.34,
  "temperature": 0.12,
  "thresholds": { "tau_in": 0.28, "tau_out": 0.71 },
  "physio_state": {
    "hrv_coherence": 0.73,
    "rmssd": 42,
    "stress": 0.31
  }
}
```

Даёт:
- **Полная трассировка решений** — каждый шаг аудируем
- **Автоматические ветки** — d > τ_out → создаётся ветка для тестирования альтернативы
- **Revert/cherry-pick** — можно откатить любой шаг или перенести вывод из ветки
- **Отладка когнитивных траекторий** — `git log` = хронология мысли

## Маппинг HRV → γ/τ

Физиологическая калибровка в реальном времени:

```
Вход:  R-R интервалы (окно 2–5 мин)
       Stress ∝ 1/RMSSD (симпатика)
       Coherence ∝ LF/HF balance (парасимпатика)

Маппинг:
  τ_in  = τ₀ · (1 + k₁ · Stress)         # стресс → шире ловим сигналы
  τ_out = τ₀ · (1 − k₂ · Coherence)       # когерентность → уже горизонт
  γ     = γ₀ + η · (Stress − Coherence)    # стресс → жёстче проверка
  α     = α₀ · Coherence                   # когерентность → плавнее обучение

Выход: динамические параметры CognitiveState
```

Адаптация под состояние:
- **Стресс** → широкий поиск + жёсткая фильтрация (много входов, строгая проверка)
- **Фокус** → узкий горизонт + плавные обновления (мало входов, мягкая калибровка)
- Система не "ломается" от шума, а **меняет стратегию обработки**

## Полная цепочка пайплайна

```
HRV-сенсор
  → Stress, Coherence
    → задаёт γ, τ_in, τ_out, α
      → distinct(A, B) считает d
        → d < τ → CONFIRM | d > τ → CONFLICT | между → EXPLORE
          → Bayes: W_new = W_old · exp(−γ·d)
            → macro: AND/OR/XOR из зон d
              → Git-коммит(d, ΔW, HRV, macro)
                → Horizon сдвигается / ветвится / архивируется
```

## Эмерджентная логика

Булевы операторы не хардкодятся, а **возникают** из зон d:

```
d ≈ 0    → согласие, усиление     → аналог AND
d ≈ 0.5  → конфликт, альтернатива → аналог XOR
d ≈ 1    → отрицание, разрыв      → аналог NOR/NOT
```

Логические макросы через distinct():

```python
XOR(A, B) ≡ distinct(A, B)                              # различие = исключающее ИЛИ
NOT(A)    ≡ 1 − weight(A)                               # или distinct(A, context)
AND(A, B) ≡ exp(−γ · (distinct(E, A) + distinct(E, B))) # оба согласны с evidence
OR(A, B)  ≡ 1 − min(distinct(E, A), distinct(E, B))     # хотя бы один согласен
```

Система работает в вероятностной логике — точнее отражает реальные когнитивные процессы и избегает ступенчатых артефактов.

### Конфликт ≠ ошибка

Конфликт (d ≈ 0.5) — не баг. Это сигнал "оба да в одном пространстве". Триггер разрешения: система не паникует, а создаёт ветку для тестирования альтернативы. Конфликт — источник информации, не её отсутствие.

## Pipeline: три слоя обработки

```
1. Эмбеддинги (маршрутизация)
   → distinct() считается в пространстве векторов, без текста
   → решения о маршруте: CONFIRM / CONFLICT / EXPLORE

2. Символическое ядро (fuse-примитив + контекст)
   → макросы AND/OR/XOR собираются из зон d
   → Bayes update, Git-коммит, Horizon shift
   → state_origin отслеживается (1_rest vs 1_held)

3. Отложенный текст (LLM только при необходимости)
   → текст генерируется для пользователя, не для системы
   → система мыслит эмбеддингами, говорит текстом
```

## sync_error — главный метрик

```
sync_error = d(model_prediction, user_action)
```

Не prediction error в абстрактном смысле — конкретная мера рассинхрона между системой и пользователем:
- Высокий sync_error → система не понимает пользователя → расширить поиск, задать вопрос
- Низкий sync_error → точное предсказание → сузить, действовать уверенно
- sync_error как обратная связь → система учится предсказывать конкретного человека

В контексте state_origin:
- `1_rest` (низкий sync_error) → медленные переходы, готов к сканированию
- `1_held` (высокий sync_error) → уже в работе, выше чувствительность, требует быстрой валидации

## Как это меняет Baddle

### Текущая архитектура → NAND-архитектура

| Компонент | Сейчас | С distinct() |
|-----------|--------|-------------|
| **Граф** | Рёбра = cosine similarity, типы (supports/contradicts) | Рёбра = история вызовов distinct(). AND/OR/XOR — макросы |
| **Байес** | `_bayesian_update(prior, p_e_h, p_e_nh)` | `logW += −γ·d + log(prior)`. Одна формула |
| **Horizon** | precision → temp/top_k, 4 состояния | γ и T калибруются автоматически через d(A,A) и KL(W). Состояния = зоны d |
| **Tick dispatcher** | if primitive == "and" / "or" / "xor" | Не нужен. d определяет действие автоматически |
| **SmartDC** | 3 промпта → тезис/антитезис/нейтральный → centroid | distinct(thesis, antithesis) = мера конфликта. Synthesis = минимизация d |
| **Pump** | Поиск мостов через облака + LLM | distinct(cloud_A, cloud_B) → зона 0.3-0.7 = потенциальные мосты |
| **Novelty check** | Embedding similarity > threshold | distinct(new, existing) < threshold → reject |
| **Stop conditions** | check_stop() с 3 ветками | Все W стабилизировались (KL → 0) → stop |
| **HRV** | Планируется: coherence → θ/φ | γ = γ₀ + η·(Stress − Coherence). HRV напрямую модулирует чувствительность |
| **States** | 4 состояния (EXPLORATION/EXECUTION/RECOVERY/INTEGRATION) | 5 состояний (FOCUSED/EXPLORE/SHIFT/CONFLICT/STABILIZE) + state_origin (rest vs held) |
| **Feedback** | surprise = 1 - confidence | sync_error = d(prediction, user_action). Персональная калибровка |
| **Pipeline** | Текст → парсинг → нода → эмбеддинг | Эмбеддинг (маршрутизация) → символическое ядро → текст только для UI |

### Ключевое изменение: мышление эмбеддингами

Сейчас Baddle мыслит **текстом**: LLM генерирует строки → парсинг → нода. Текст — промежуточный формат.

С distinct() система мыслит **эмбеддингами**:
- Генерация → эмбеддинг (вектор)
- distinct() считается без LLM-вызова
- Текст генерируется **только для отображения** пользователю
- Скорость: cosine distance = 1мс, LLM-вызов = 2 секунды

Где хранятся эмбеддинги: уже в нодах (`_graph["embeddings"]`). Нужно сделать их **первичным** форматом, а текст — производным.

## Два режима мышления

### CognitiveState (активный фокус)
```
d < τ_in  → вход в Horizon (обработка)
d > τ_out → выход (архив)
γ, T, τ — динамические, калибруются через d(A,A) и HRV
```

### DMN-сканер (фоновый поиск)
```
Фоновый обход пар в зоне 0.3 < d < 0.7
Тестирование в изолированных ветках (git-коммит)
var(ΔW) < δ → мердж в основной граф
var(ΔW) ≥ δ → архив (не подтвердилось)
```

### Переключение
```
Высокая нагрузка → Horizon доминирует, DMN в low-priority
Покой → DMN массовый скан, готовит инсайты
Бюджет вычислений делится динамически
```

Это уже частично работает: Scout mode = DMN, autorun = Horizon, Pump = DMN-поиск.

## Prediction error как синхронизация

Новая интерпретация: prediction error = **система плохо синхронизирована с пользователем**.

```
d(model_prediction, user_action) → большое → система не понимает пользователя
                                 → маленькое → система точно предсказывает
```

Это расширяет текущий prediction error (surprise = 1 − confidence) до **персональной** калибровки: система учится предсказывать не только мир, но и конкретного пользователя.

## Априоры как распределения

Сейчас prior = одна цифра (0.5). Предложение: prior = **распределение** (Beta-распределение).

```python
# Сейчас
prior = 0.5

# С распределениями
prior = Beta(α=2, β=3)  # "скорее нет, но не уверен"
# α > β → скорее да
# α = β → не знаю
# маленькие α,β → очень не уверен
# большие α,β → уверен
```

Дает не только "вероятность", но и "уверенность в этой вероятности". 0.5 с Beta(1,1) = "понятия не имею" ≠ 0.5 с Beta(50,50) = "точно пополам".

## Режим камеры сенсорной депривации

Автоматический режим где система **изолируется** от внешних входов и работает только с внутренним графом:
- Отключает LLM-генерацию нового контента
- Только distinct() между существующими нодами
- Находит скрытые паттерны в том что уже есть
- Аналог медитации: не генерировать новое, а увидеть связи в имеющемся

## Пороги по умолчанию

```
γ₀ = 2.0     | γ ∈ [0.1, 10.0]
T₀ = 0.1     | T ∈ [0.05, 2.0]    | α = 0.5
ε = 1e-6     | λ_ema = 0.1
τ_in = 0.3   | τ_out = 0.7
DMN_budget = 0.2
W_min_log = -5.0
```

## Цепочка потока данных

```
HRV-сенсор → задаёт γ, τ
  → distinct(A,B) считает d
    → d идёт в Bayes (L = exp(−γ·d))
      → веса узлов пересчитываются
        → макросы собирают d в логические паттерны
          → Horizon сдвигается / архивируется / ветвится
            → commit в граф с трассой diff=distinct(...)
```

## Шаги реализации

1. **distinct()** — обёртка над cosine_similarity в main.py. `d = (1 - cos_sim) / 2`
2. **Байес через distinct** — заменить `_bayesian_update()` на `logW += −γ·d + log(prior)`
3. **Автокалибровка γ** — EMA(d(A,A)) → γ адаптируется
4. **Эмбеддинги как основной формат** — текст генерируется только для UI
5. **DMN с distinct** — фоновый обход пар, зона 0.3 < d < 0.7
6. **HRV → γ** — `γ = γ₀ + η·(Stress − Coherence)` из готового стека
7. **Beta-распределения для приоров** — вместо скаляра
8. **Камера сенсорной депривации** — режим без LLM, только distinct()

## Итог

Один примитив `d = distinct(A,B)` заменяет всю ручную логику, мгновенно превращаясь в байесовское правдоподобие `L = exp(−γ·d)`. Веса обновляются в лог-пространстве, γ и T калибруются автоматически. Система сама балансирует фокус/поиск, версионирует каждый шаг и устойчива к шуму без усложнения кода.

Не калькулятор, а самокалибрующийся когнитивный движок.
