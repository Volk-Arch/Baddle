# Life Assistant — Baddle как второй мозг для повседневной жизни

## Идея

Baddle — не только инструмент для мышления. Это платформа, на которую ложатся четыре ранее независимых проекта. Каждый решал свою задачу, но все используют один и тот же механизм: prediction error, конус возможностей, примитивы (AND/OR/XOR/Bayes).

## Четыре прототипа → один стек

### 1. MindBalance — энергетическая экономика сознания

Три параметра: ожидания (expectation), реальность (reality), ментальная энергия (energy).

Формулы:
```
ID = |expectation - reality| / 100    — индекс дисбаланса (= prediction error)
IP = (expectation + reality) / 200    — общее положение
T  = IP ± f(ID)                       — эмоциональный тонус
A  = energy × (1 + ID × 0.5)          — активация сознания
```

Состояния: поток, стресс, выгорание, апатия, нейтральное — карта в пространстве T×A.

Два уровня энергии:
- **Дневная** (100 единиц) — восстанавливается за ночь
- **Общий резерв** (2000) — долгоживущий, истощается при хроническом стрессе

Каждое решение расходует энергию. Стоимость зависит от сложности, неопределённости, важности. Долгосрочный прогноз с риском выгорания.

**Связь с Baddle:** ID = surprise, энергия = precision resource, состояния = расширение Horizon (4 → 7+ состояний).

### 2. Тамагочи — геймификация питания

Виртуальный питомец, которого кормишь реальными продуктами. 4 стадии роста, калорийность, срок годности. Режимы: спортивный, экономный, кулинарный.

Механики:
- Рекомендации блюд из имеющихся продуктов
- Автоматическое кормление (система выбирает)
- Образовательные факты о продуктах
- Соревнования и достижения

**Связь с Baddle:** repeatable mode (привычка питания), XOR comparative (выбор блюда), интерактивный ассистент ("что приготовить из того что есть?").

### 3. Time Player — планировщик задач

Task tracker (Telegram Mini App). Планирование дня, задачи с приоритетами.

**Связь с Baddle:** AND mode (приоритеты/по шагам), энергетическая стоимость задач из MindBalance.

### 4. HRV Reader — готовый стек Polar H10

**Уже написан и работает.** Polar H10 → BLE → RR интервалы → HRV метрики → реалтайм UI.

Компоненты:
- `polar_reader.py` — BLE scan, connect через bleak + bleakheart, RR callback с throttling, акселерометр
- `hrv_calculator.py` — RMSSD, SDNN, pNN50, LF/HF через FFT
- `app.py` — Flask + SocketIO сервер, история 24ч, API endpoints

Метрики уже реализованы:
```python
calculate_rmssd(rr_intervals)           # Парасимпатический тонус
calculate_sdnn(rr_intervals)            # Общая вариабельность
calculate_pnn50(rr_intervals)           # % интервалов с разницей >50мс
calculate_frequency_domain(rr_intervals) # LF, HF, LF/HF ratio через FFT
```

**Связь с Baddle:** Готовый MVP для HRV-интеграции (v5b). Не надо писать с нуля.

## Модель энергии через prediction error

Единый механизм для всего:

```
Решение = collapse prediction error (свести реальность к ожиданию)
  → каждый collapse стоит энергии
  → стоимость = f(сложность, неопределённость, важность)

Нерешённое = открытый prediction error
  → висит фоново, расходует ресурс
  → чем дольше → тем больше фоновый расход
  → хронически → стресс → выгорание

Выгорание = precision → 0
  → конус схлопнулся
  → нет энергии расширять
  → апатия
```

HRV coherence — прямой индикатор: высокая = энергия есть, низкая = ресурс истощается.

## Помощник в повседневности

Интерактивный ассистент (третий контур) применённый к жизни:

- **Утро:** "На основе HRV — ты отдохнул на 78%. План дня: 5 задач, энергетический бюджет 85 единиц. Первая задача стоит 20 — начнём?"
- **Обед:** "Coherence падает. Из холодильника можно: [3 варианта]. Рекомендую #2 — быстро и 450 ккал" (XOR mode)
- **Паралич выбора:** "Ты завис 3 минуты. Выбираю за тебя: вариант B. Причина: минимальная энергетическая стоимость"
- **Вечер:** "Дневная энергия: 15/100. Завтра утром резерв восстановится до ~90 при 7+ часах сна"

## Шаги реализации

1. **Import HRV** — подключить `hrv_calculator.py` к Baddle (уже Flask, совместимо)
2. **Energy overlay** — MindBalance формулы как слой поверх Horizon (daily/reserve energy meters)
3. **Persist energy** — сохранение энергии между сессиями (JSON)
4. **Daily routines** — repeatable mode + интерактивный ассистент для питания/задач
5. **Auto-decision** — XOR comparative с авто-выбором при параличе (timer → LLM judge)
