# Life Assistant — Baddle как второй мозг для повседневной жизни

## Идея

Baddle — не только инструмент для мышления. Платформа, на которую
ложатся четыре ранее независимых проекта. Каждый решал свою задачу, но
все используют один и тот же механизм: prediction error, конус
возможностей, примитивы (AND / OR / XOR / Bayes).

---

## Четыре прототипа → один стек

### MindBalance — энергетическая экономика сознания

Три параметра: expectation, reality, mental energy.

```
ID = |expectation − reality| / 100    # индекс дисбаланса (= prediction error)
IP = (expectation + reality) / 200    # общее положение
T  = IP ± f(ID)                       # эмоциональный тонус
A  = energy × (1 + ID × 0.5)          # активация сознания
```

Состояния (поток / стресс / выгорание / апатия / нейтральное) —
карта в пространстве T × A. Два уровня энергии: дневная (100 единиц,
восстанавливается за ночь) + общий резерв (2000, истощается при
хроническом стрессе).

**В Baddle:** ID = surprise; энергия = precision resource; состояния
= расширение Horizon (4 → 7+ состояний).

### Тамагочи — геймификация питания

Виртуальный питомец, которого кормишь реальными продуктами. 4 стадии
роста, калорийность, срок годности. Режимы: спортивный, экономный,
кулинарный. Рекомендации блюд из имеющихся продуктов, автоматическое
кормление, образовательные факты.

**В Baddle:** repeatable mode (привычка питания), XOR comparative
(выбор блюда), интерактивный ассистент («что приготовить из того что
есть?»).

### Time Player — планировщик задач

Task tracker (Telegram Mini App). Планирование дня, задачи с
приоритетами.

**В Baddle:** AND mode (приоритеты / по шагам), энергетическая
стоимость задач из MindBalance.

### HRV Reader — стек Polar H10

Чтение вариабельности сердечного ритма через Polar H10: BLE →
RR-интервалы → HRV-метрики → реалтайм-UI. Метрики: RMSSD
(парасимпатический тонус), SDNN (общая вариабельность), pNN50, LF/HF
(симпатика/парасимпатика через FFT), Coherence (синхронизация
дыхания и сердца).

**В Baddle:** `src/hrv_manager.py` + `src/hrv_metrics.py` уже
реализуют симулятор + интеграцию в UserState. Реальный Polar через
BLE — задача в TODO. Детали — [hrv-design.md](hrv-design.md).

## Модель энергии через prediction error

Единый механизм для всего:

- **Решение** = collapse prediction error (свести реальность к
  ожиданию). Каждый collapse стоит энергии. Стоимость = f(сложность,
  неопределённость, важность).
- **Нерешённое** = открытый prediction error. Висит фоново, расходует
  ресурс. Чем дольше — тем больше фоновый расход. Хронически — стресс
  — выгорание.
- **Выгорание** = precision → 0. Конус схлопнулся, нет энергии
  расширять, апатия.

HRV coherence — прямой индикатор: высокая = энергия есть, низкая =
ресурс истощается.

## Помощник в повседневности

Интерактивный ассистент (третий контур) применённый к жизни:

- **Утро:** «На основе HRV — ты отдохнул на 78%. План дня: 5 задач,
  энергетический бюджет 85 единиц. Первая задача стоит 20 — начнём?»
- **Обед:** «Coherence падает. Из холодильника можно: [3 варианта].
  Рекомендую #2 — быстро и 450 ккал» (XOR mode)
- **Паралич выбора:** «Ты завис 3 минуты. Выбираю за тебя: B. Причина:
  минимальная энергетическая стоимость»
- **Вечер:** «Дневная энергия: 15/100. Завтра утром резерв восстановится
  до ~90 при 7+ часах сна»
