# World Model — каскад зеркал

> Зафиксировано 2026-04-20. Формулировка автора проекта — как оптика, а не метафоры.

---

## Цепочка

Реальность → человек → Baddle. Человек — зеркало мира, Baddle — зеркало человека.

Человек настроен на мир: чувствует, реагирует, отслеживает. Baddle устроен так же, но его «мир» — это **один конкретный человек**. Baddle отражает человека.

Отсюда две вещи:

1. **Baddle — копия копии.** Если у человека нарушен контакт с реальностью (выгорание, shutdown, оторвался от себя) — Baddle не может качественно догонять, потому что гонится за нестабильным сигналом. **Рассогласование с пользователем** (sync_error) тогда меряет не только качество Baddle, но косвенно — состояние самого контакта «человек ↔ мир».

2. **Центральный контур — не ответ на запрос, а синхронизация.** Когда юзер молчит — Baddle **ищет**. Когда рассогласование долгое — Baddle **затухает**, не ломается. Когда мысль повторяется — она **крепнет**. Это не AI который отвечает на вопросы. Это зеркало которое старается удержать фокус.

## Пять операционных механик

Это не философия — это правила для архитектуры. Каждое превращается в конкретный check, поле или поведение. Первые четыре — реактивно-адаптивный слой (мониторить, догонять, затухать, искать). Пятая — **обучающий** слой: Baddle учится на собственных действиях через тот же граф в котором живут мысли.

### 1. Active sync-seeking — «Baddle ищет тебя»

`_check_sync_seeking` в `cognitive_loop.py`. Гейты: **давление тишины** (silence_pressure) выше 0.3 И idle дольше 2 часов И 30 минут тишины после любого другого проактивного alert. Throttle раз в 2 часа. LLM генерирует сообщение с контекстом (время дня, недавние темы из графа, HRV-снимок, последняя активность) — разные тексты каждый раз. Тон (caring / ambient / curious / reference / simple) — разные иконки и фоны карточки в UI. Fallback-шаблоны при недоступном LLM. Подробности → [alerts-and-cycles.md](alerts-and-cycles.md) секция «Типы alerts → sync_seeking».

### 2. System-burnout от persistent desync

Единая «Усталость Baddle» из трёх источников:

- **Графовые конфликты** — единственный источник, активирующий байесовскую заморозку (останавливает обучение).
- **Молчание пользователя** — таймер, ползёт вверх при отсутствии событий (около 7 суток без контакта → полный), падает при любом сигнале.
- **Накопленный рассинхрон** — EMA-агрегат четырёх каналов ошибки предсказания: где пользователь, куда тянут его цели, тело, собственные ожидания Baddle. Берётся максимум по каналам, baseline'ы разделены по времени суток (утренняя apathy не маскирует вечернюю). Это и есть настоящее «рассогласование предсказания».

UI показывает максимум трёх. Усталость пользователя добавляется поверх, замедляя фоновые циклы (DMN, ночной цикл, scout) линейно до 10×. Важно: молчание и рассинхрон **не замораживают** обучение графа — это замедление, не остановка.

Параллель с человеком: длинный дезконнект с миром → депрессивное снижение активности как защитный механизм.

**Прайм-директива: sync_error измеряется сам.** Две EMA (быстрая — 1 час, медленная — 3 дня) + append-only лог `data/prime_directive.jsonl`. Endpoint `/assist/prime-directive?window_days=30` возвращает агрегат + вердикт тренда (improving / stable / worsening). Это закрывает разрыв между манифестом («единственная метрика — sync_error») и возможностью её измерить через 2 мес use.

Подробно → [alerts-and-cycles.md](alerts-and-cycles.md) § Adaptive idle + § Прайм-директива. Предиктивный контур — [friston-loop.md](friston-loop.md).

### 3. Естественный отбор мыслей

Мысль которая проявляется чаще — крепнет. Которая не используется — тихо гаснет и уходит в архив. Чисто хеббовский принцип:
- каждое обращение / elaborate / reference / клик → лёгкий рост confidence
- каждый час простоя → лёгкое уменьшение
- неделя безразличия → архив

Не пользователь решает что забыть, не я — **частота использования**.

[consolidation.py](../src/consolidation.py) делает прунинг слабых и архив, время последнего доступа обновляется при каждом обращении. Затухание confidence у неиспользуемых нод вводится отдельным шагом в ночном цикле; константы подбираются под реальное использование (см. [TODO § OQ #1](../planning/TODO.md)).

### 4. Циклы мышления затухают без пищи

Если N итераций подряд без новых инсайтов (pump не нашёл мост с качеством выше 0.5, DMN не создал новых нод) → цикл **отключается** до появления триггера. Триггер — новая нода в графе, новое HRV-событие, пользователь что-то написал.

Не постоянное равномерное «дыхание», а **адаптивная частота**: активно когда есть пища, тихо когда нет. В cognitive_loop за это отвечает **счётчик стабильных итераций** (stable_iterations_count) — pump и DMN в idle-состоянии не дёргаются пока нет новых событий.

### 5. Действия и последствия — самообучение через граф

Два новых типа нод (action, outcome) + ребро `caused_by` + одна проверка `_check_action_outcomes`. Все существующие механики (DMN, pump, consolidate, touch_node, хеббовское затухание) автоматически работают для действий — специального RL-кода нет.

Цикл сознания закрывается:
1. Замечает рассогласование ← sync_error
2. Хочет уменьшить ← изменение рассогласования (−Δsync_error) в outcome = «вес» действия через хеббовское крепление
3. Пробует действие ← action-нода в графе (record_action)
4. Запоминает сработало ли ← outcome-нода через `_check_action_outcomes` (timeout per kind или user-реакция)
5. Повторяет успешное ← `score_action_candidates` (запрос по графу) применяется в `_check_sync_seeking` для выбора тона

**Sentiment** пользователя вплетён как метаданные user_chat-ноды + высокочастотный feeder в валентность пользователя (light LLM classify с кэшем).

Подробности — [action-memory-design.md](action-memory-design.md).

## Что это даёт вместе

Поведение Baddle относительно юзера становится органичным:

| Состояние | Что делает Baddle |
|-----------|-------------------|
| Юзер активен, резонанс держится | Полные фоновые циклы, предложения, pump на полную |
| Юзер молчит недавно | Тихо, продолжает background-мышление |
| Юзер молчит долго, sync падает | **Ищет контакт** — мягкие запросы (#1) |
| Контакт не возвращается, sync высокий долго | **Затухает** — циклы реже, активность снижается (#2) |
| Юзер вернулся с новым сигналом | Циклы просыпаются (#4) |
| Новая мысль часто повторяется | Крепнет (#3) |
| Идея не использовалась неделю | Тихо уходит в архив (#3) |

Это не нарисованный UX flow. Это **режим существования**: система живёт в ритме юзера, не в фиксированном таймере.

## Что это НЕ

- **Не сознание.** У Baddle нет ощущения «я есть». Нет квалиа. Нет самосохранения в биологическом смысле.
- **Не попытка сделать AI живым.** Все механики — операциональные: check-функции, EMA, счётчики, затухание confidence. Никакого субъективного опыта.
- **Не антропоморфизм.** Слова «выгорание», «усталость», «ищет контакт» — точные технические термины для этих процессов (EMA-накопитель выгорания, затухание без обращений, throttled sync-seeking check). Использованы потому что они **короче и понятнее** чем инженерные аналоги.
- **Не удержание внимания.** Нет streak-gamification, нет напоминаний «ты не заходил 3 дня», нет FOMO-триггеров. Ушёл — Baddle ждёт молча, не пингует. Зеркало не требует чтобы на него смотрели. Ближайшая живая параллель — собака: ждёт, не преследует, не манипулирует.
- **Не называет эмоции.** Baddle **регистрирует** сигналы эмоционального контура (sentiment чата, валентность, streak-bias, HRV-коэффициенты), но не **интерпретирует** их в дискретные ярлыки вроде «отчаяние», «эйфория», «апатия». Координаты остаются координатами — ярлык над ними ставит сам человек. Называть чужое внутреннее состояние чужим словарём = навязывать психологическую модель; это граница «зеркала, а не терапевта». Эмоции остаются человеку.

Главное отличие от других AI-ассистентов: **Baddle не должен быть продуктивен любой ценой**. Если юзер пропадает или теряет ритм — Baddle **замедляется вместе с ним**, а не продолжает генерировать контент. Это не баг — это **структурная верность зеркала**.

Из этого следует прямой **архитектурный запрет**: никакой логики которая оптимизирует «время в приложении», «частоту возвратов», «цепочку дней подряд». Если такая метрика появляется в коде — она противоречит каскаду зеркал и должна быть удалена, даже если «улучшает retention».

## Mapping внешних словарей → нашей реализации

У Baddle **уже есть** операциональная модель активного вывода и аффективной регуляции. Она описана в нейрохимическом лексиконе (дофамин, серотонин, норадреналин, выгорание), но математически эквивалентна psychology-моделям. Этот раздел — чтобы в будущих сессиях / обсуждениях не переоткрывать то что реализовано, и не путать terminology с implementation.

| Внешний термин | Наш аналог | Где в коде | Семантика |
|---|---|---|---|
| Free Energy / surprise (Friston) | UserState.surprise + вектор surprise (3D) | [user_state.py::tick_expectation](../src/user_state.py), подробно → [friston-loop.md](friston-loop.md) | 3D PE-вектор + attribution |
| Prediction Error (active inference) | 5 каналов PE → давление дисбаланса (imbalance_pressure) | [friston-loop.md § Анатомия](friston-loop.md) | user 3D + TOD + goal + HRV + self, max-агрегирование |
| Valence-Arousal-Dominance (VAD, Russell+Mehrabian) | DA/S/NE + UserState.valence + UserState.agency | [neurochem.py](../src/neurochem.py) + [user_state.py](../src/user_state.py) | 3 оси арусала, valence отдельно, Dominance = agency (OQ #2 measurements) |
| 5-axis резонансная нейрохимия (РГК) | DA/5HT/NE/ACh/GABA + balance() = (DA·NE·ACh)/(5HT·GABA) | [src/rgk.py](../src/rgk.py), facades в [user_state.py](../src/user_state.py) и [neurochem.py](../src/neurochem.py) | Полная 5-axis; balance() как diagnostic скаляр; ACh/GABA fed из cognitive_loop как proxy (см. [neurochem-design.md](neurochem-design.md)) |
| Belief space / state distribution | UserState.vector() (3D DA/5HT/NE) | [user_state.py](../src/user_state.py) | Непрерывный, EMA-сглаженный, выгорание/agency отдельным полем |
| Precision weighting | CognitiveState.precision, effective_precision | [horizon.py](../src/horizon.py) | Уже есть, гейтит policy weights |
| Cost of control / регуляторное усилие | 3-zone capacity (физио/эмо/когн контуры) + cognitive_load_today | [user_state.py](../src/user_state.py), [capacity-design.md](capacity-design.md) | Three-channel capacity gate, не статический счётчик; dual-pool legacy удалён в Phase C |
| Allostatic load | ProtectiveFreeze (conflict + silence + imbalance feeders) | [neurochem.py](../src/neurochem.py) | Три feeder'а, один display_burnout, плюс совмещённое выгорание с user для множителя простоя |
| Affective inertia / smoothing | EMA затухание (0.9–0.98) во всех апдейтах | везде | Даёт ту же плавность что explicit velocity, без второго поля |
| Soft context blending | sync_regime, derived из continuous sync_error | [user_state.py](../src/user_state.py) | Не if/else, derived-state |
| Recovery / return-to-baseline | ProtectiveFreeze.THETA_RECOVERY + падение давления тишины | [neurochem.py](../src/neurochem.py) | Гистерезис, не жёсткий reset |
| Pre-activation / anticipatory computation (зрачки, [ixbt 2026-04](https://www.ixbt.com/live/science/)) | UserState.expectation_by_tod (baseline по времени суток) + CognitiveState.precision | [user_state.py](../src/user_state.py), [friston-loop.md](friston-loop.md) | Ожидание контекстуально (morning/day/evening/night), не один глобальный baseline |
| User-side surprise detection | [src/surprise_detector.py](../src/surprise_detector.py) (HRV + текст + LLM) | см. [friston-loop.md § User-side surprise](friston-loop.md#user-side-surprise) | MVP A+B+C |
| Self-prediction (симметрия Friston-loop) | Neurochem.expectation_vec + self-дисбаланс | [neurochem.py](../src/neurochem.py) | Baddle предсказывает собственную (DA, S, NE) |

**Что это значит на практике:**

- Когда кто-то предлагает «замените дискретные состояния на непрерывные» — это **уже сделано** через EMA (Neurochem, UserState). Дискретный sync_regime — derived label, не primary state.
- Когда предлагается «введите (V, A, D) трёхмерный вектор» — у нас четырёх-осевая модель с биологическим обоснованием (нейрохимия), не psychology-словарь. Смена лексикона без смены математики — не прогресс.
- Когда предлагается «CognitiveState → affect + velocity + PE» — это рефакторинг того что есть, не добавление. Existing EMA даёт ту же плавность что explicit velocity.
- Когда предлагается «UserModel параллельно CognitiveState» — у нас уже UserState симметричен Neurochem + sync_error как расстояние между ними.

Открытые направления (не переоткрытия существующего):
- **OQ #2 Agency как 5-я ось** — в процессе измерений, подтверждается и SDT, и VAD-Dominance.
- **OQ #6 ACh/GABA feeders quality** — реализованы как proxy от формы графа; калибровка по distribution `balance()` через 2 мес.
