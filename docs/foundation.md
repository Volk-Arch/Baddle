# Foundation — почему Baddle существует и из чего он сложился

## Один вопрос

Пять проектов, разнесённых во времени. Каждый решал свою задачу:

- **MindBalance** — как работают мои состояния?
- **Тамагочи** — как работают мои привычки?
- **Time Player** — как работают мои решения?
- **HRV Reader** — как работает моё тело?
- **Baddle** — как работает моё мышление?

Один вопрос за всеми пятью: **как я устроен?**

---

## Ответ

Prediction error. Один механизм.

Ожидания ≠ реальность → напряжение → действие → обновление модели мира.

| Домен | Ожидания | Реальность | Prediction error |
|-------|----------|-----------|-----------------|
| Мышление | Гипотеза | SmartDC результат | `surprise = 1 − confidence` (Beta-prior сбоку даёт CI на самой confidence — насколько уверены в уверенности) |
| Энергия | Сколько могу | Сколько потратил | decision fatigue |
| Тело | Coherence baseline | Текущий HRV | stress / recovery |
| Решения | Исход который хочу | Исход который получил | сожаление или облегчение |
| Привычки | «Я делаю X каждый день» | Пропустил | streak break |

Формула MindBalance `ID = |expectation − reality|` была написана до знакомства с Фристоном. Тот же механизм, найденный через опыт — наука нашла через fMRI.

## Конвергенция

Каждый проект был фрагментом. Baddle — точка где все сошлись.

| Проект | Вклад в Baddle |
|---|---|
| MindBalance | Модель энергии, Horizon states |
| Тамагочи | Привычки, питание, assist-режим |
| Time Player | Задачи, energy budget, AND-декомпозиция |
| HRV Reader | Тело как вход, θ/φ конуса |
| Baddle | Мышление: tick cycle + SmartDC + Pump |

Не было спроектировано. Каждый компонент — ответ на конкретную проблему. Потом оказалось — это части одной системы, описанной нейробиологией 30 лет назад.

## Зеркало

Baddle — не инструмент. Это зеркало. Попытка воспроизвести себя в коде, понять свои механизмы, и потом использовать это понимание чтобы жить осознаннее.

Self-hosted второй мозг, синхронизированный с первым через тело, диалог и действия. Заботишься о нём = заботишься о себе.

Такого проекта нет ни у кого. Потому что он — отражение конкретного человека, который искал ответ на один вопрос всю жизнь.

## Четыре прототипа → один стек

Baddle — не только инструмент для мышления. Платформа, на которой сходятся четыре отдельных направления. Каждое решает свою задачу, но все используют один и тот же механизм: ошибка предсказания (PE), конус возможностей, примитивы (AND / OR / XOR / Bayes).

### MindBalance — энергетическая экономика сознания

Три параметра: expectation, reality, mental energy.

```
ID = |expectation − reality| / 100    # индекс дисбаланса (= prediction error)
IP = (expectation + reality) / 200    # общее положение
T  = IP ± f(ID)                       # эмоциональный тонус
A  = energy × (1 + ID × 0.5)          # активация сознания
```

Состояния (поток / стресс / выгорание / апатия / нейтральное) — карта в пространстве T × A. Два уровня энергии: дневная (100 единиц, восстанавливается за ночь) + общий резерв (2000, истощается при хроническом стрессе).

**В Baddle:** ID (информационная динамика) превратилось в **удивление** (surprise); энергия = ресурс точности; состояния = расширение Horizon (4 → 7+ состояний).

### Тамагочи — геймификация питания

Виртуальный питомец, которого кормишь реальными продуктами. 4 стадии роста, калорийность, срок годности. Режимы: спортивный, экономный, кулинарный. Рекомендации блюд из имеющихся продуктов, автоматическое кормление, образовательные факты.

**В Baddle:** repeatable mode (привычка питания), XOR comparative (выбор блюда), интерактивный ассистент («что приготовить из того что есть?»).

### Time Player — планировщик задач

Task tracker (Telegram Mini App). Планирование дня, задачи с приоритетами.

**В Baddle:** AND mode (приоритеты / по шагам), энергетическая стоимость задач из MindBalance.

### HRV Reader — стек Polar H10

Чтение вариабельности сердечного ритма через Polar H10: Bluetooth → межсокращательные интервалы (RR, миллисекунды) → HRV-метрики → реалтайм-UI. Метрики: **корень из среднеквадратичного изменения RR** (RMSSD — парасимпатический тонус), **стандартное отклонение RR** (SDNN — общая вариабельность), **доля пар RR различающихся более чем на 50 мс** (pNN50), **отношение мощностей в низко- и высокочастотной полосах** (LF/HF — баланс симпатика/парасимпатики через FFT), **когерентность** (синхронизация дыхания и сердца).

**В Baddle:** `src/hrv_manager.py` + `src/hrv_metrics.py` уже реализуют симулятор + интеграцию в UserState. Реальный Polar через BLE — задача в TODO. Детали — [hrv-design.md](hrv-design.md).

## Модель энергии через prediction error

Единый механизм для всего:

- **Решение** = collapse prediction error (свести реальность к ожиданию). Каждый collapse стоит энергии. Стоимость = f(сложность, неопределённость, важность).
- **Нерешённое** = открытый prediction error. Висит фоново, расходует ресурс. Чем дольше — тем больше фоновый расход. Хронически — стресс — выгорание.
- **Выгорание** = precision → 0. Конус схлопнулся, нет энергии расширять, апатия.

HRV coherence — прямой индикатор: высокая = энергия есть, низкая = ресурс истощается.

## Помощник в повседневности

Интерактивный ассистент (третий контур) применённый к жизни:

- **Утро:** «На основе HRV — ты отдохнул на 78%. План дня: 5 задач, энергетический бюджет 85 единиц. Первая задача стоит 20 — начнём?»
- **Обед:** «Coherence падает. Из холодильника можно: [3 варианта]. Рекомендую #2 — быстро и 450 ккал» (XOR mode)
- **Паралич выбора:** «Ты завис 3 минуты. Выбираю за тебя: B. Причина: минимальная энергетическая стоимость»
- **Вечер:** «Дневная энергия: 15/100. Завтра утром резерв восстановится до ~90 при 7+ часах сна»
