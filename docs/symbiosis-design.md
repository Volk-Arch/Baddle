# Симбиоз — два state-вектора

> Прайм-директива: **рассогласование с пользователем** (sync_error) = L2-норма разности векторов пользователя и системы в 3D пространстве нейрохимии (дофамин, серотонин, норадреналин). Не декларация, а вычисляемая метрика.
>
> Полный предиктивный контур (удивление, каналы ошибки предсказания, давление дисбаланса) — в **[friston-loop.md](friston-loop.md)**. Этот документ описывает симметрию двух state-векторов как таковую.

---

## Идея

Две симметричные структуры: **нейрохимия системы** (Neurochem) и **состояние пользователя** (UserState) одинаковой формы. Метрика синхронизации — L2-норма разности.

Оба вектора — тройка (дофамин, серотонин, норадреналин). Рассогласование = модуль разности, максимум около √3 ≈ 1.732.

**Выгорание** (burnout), **агентность** (agency), **валентность** (valence), **удивление** (surprise) остаются как **отдельные поля у обеих сторон**, но не входят в вектор рассогласования — разные физические явления, смешивание в одной L2-оси давало бы шум. Вектор трёхмерный.

---

## Откуда берутся сигналы

**Сторона системы** (Neurochem) питается **динамикой графа** — см. [neurochem-design.md](neurochem-design.md).

**Сторона пользователя** (UserState) питается **наблюдаемыми сигналами пользователя:**

| Канал | → | Скаляр UserState |
|---|---|---|
| Когерентность сердечного ритма (HRV coherence) | → | Серотонин |
| HRV-стресс / RMSSD | → | Норадреналин |
| Любое сообщение пользователя (engagement) | → | Дофамин (мягкий +0.007) |
| Sentiment чата (light LLM) | → | Валентность |
| 👍 feedback | → | Дофамин +0.9, валентность +0.7 |
| 👎 feedback | → | Дофамин +0.2, выгорание +0.05, валентность −0.7 |
| Количество решений × стоимость | → | Выгорание |
| Выполнено / запланировано | → | Агентность |

Все EMA — одна строка, как в Neurochem. Максимально простой контракт.

## Режимы симбиоза

Из тройки (рассогласование, уровень состояния пользователя, уровень состояния системы) выводится один из четырёх режимов:

| Режим | Условие | Поведение |
|---|---|---|
| **FLOW** (поток) | рассогласование < 0.3, у обоих среднее DA+S выше 0.55 | Полный объём, сложные задачи |
| **REST** (отдых) | рассогласование < 0.3, у обоих среднее ниже 0.35 | Предлагает паузу |
| **PROTECT** (защита) | рассогласование ≥ 0.3, пользователь low + система high | Берёт на себя: короче ответы |
| **CONFESS** (признаёт) | рассогласование ≥ 0.3, пользователь high + система low | «Дай мне время подумать» |
| *fallback* | любые промежуточные | FLOW |

Advice-слой ([/assist/alerts](../src/assistant.py)) выводит подсказки из режима, не из жёстких порогов на каждое поле. Это **третий контур замкнутости** (диалог) — адаптивное поведение на основе взаимного состояния.

## Адаптивный тон по химическому профилю

Поверх sync_regime (4 режима симбиоза с системой) живёт более тонкий слой — **prompt-routing** по `named_state` (8 регионов РГК-карты по chem profile, см. [user-model-design.md § Именованные состояния](user-model-design.md)). Sync_regime отвечает на вопрос «как соотносятся user и system», named_state — «в каком химическом режиме сам user». Эти оси независимы: можно быть в FLOW симбиозе и одновременно в Перегрузе по chem.

Для каждого региона есть выбор tone в `_check_sync_seeking` и system-prompt hint в `execute_deep`:

| Регион | Tone в sync_seeking | Hint в execute_deep |
|---|---|---|
| 🔵 Поток | ambient — не мешать, поддержать волну | развивай идеи без отступлений |
| 🟢 Устойчивость | simple — нейтрально | (без дополнительного hint, baseline) |
| 🟠 Фокус-Тревога | reference — по делу, без воды | короче, заземляющий тон |
| 🟡 Исследование | curious — поощрить exploration | предлагай аналогии, гибкие альтернативы |
| 🔴 Перегруз | caring — мягко, без давления | STOP-режим, минимум слов |
| ⚫ Застой | caring | разбей на микро-шаги ≤2 минуты |
| ⚪ Выгорание | caring | без давления, восстановление приоритетнее |
| ✨ Инсайт | reference | помоги зафиксировать аттрактор |

Маппинг приходит из РГК v1.0 spec §«Влияние на промпт-роутинг». Реализация — `_NAMED_TO_TONE` в `cognitive_loop._generate_sync_seeking_message` и `_NS_HINT` в `assistant_exec.execute_deep`. 5-axis chem (включая ACh+GABA) точнее ловит когнитивный режим юзера, чем 2-axis HRV-derived `frequency_regime` — поэтому named_state имеет приоритет в выборе tone.

## Три схемы классификации состояния

Состояние юзера в Baddle классифицируется по трём осям одновременно. Они **не конкурируют** — каждая отвечает на свой вопрос и работает параллельно с остальными.

| Схема | Что классифицирует | Сколько меток | Где живёт | Используется для |
|---|---|---|---|---|
| **sync_regime** | Симбиоз: как соотносятся USER и SYSTEM | 4 (FLOW / REST / PROTECT / CONFESS) | [symbiosis-design.md](symbiosis-design.md) | advice-слой, тон ответов |
| **named_state** | Само состояние пользователя по химическому профилю (5D) | 8 РГК-регионов (Поток / Устойчивость / Фокус-Тревога / Исследование / Перегруз / Застой / Выгорание / Инсайт) | [user-model-design.md § Именованные состояния](user-model-design.md) | UI-метка с emoji, суточный отчёт |
| **activity_zone** | Физиологический режим (HRV × движение) | 4 (recovery / stress_rest / healthy_load / overload) | [hrv-design.md](hrv-design.md) | alerts про тело, capacity-зона |

Конкретный момент у пользователя описывается тройкой сразу: например, «FLOW + поток + healthy_load» (идеальный рабочий момент) или «PROTECT + выгорание + overload» (нужна остановка). Метки разного разрешения — sync_regime груб, named_state тонок, activity_zone про тело — покрывают разные решения.

Есть ещё четвёртая ось — **capacity_zone** (green / yellow / red) из [capacity-design.md](capacity-design.md). Она агрегирует все три схемы выше в один сигнал decision gate: какие режимы мышления сейчас допустимы.

## Жизненный цикл сигнала

Пользователь пишет сообщение → обработчик `/assist` делает инъекцию NE в систему (+0.4), регистрирует вход (обновляет `last_input_ts`), обновляет выгорание пользователя по числу решений.

Далее `classify_intent_llm` получает подсказку о состоянии (текущую системную химию), `execute_via_zones` формирует карточку по режиму.

Пользователь жмёт 👍 / 👎 → `update_neurochem` кормит химию системы (различие 0.2 для 👍, 0.8 для 👎), `update_from_feedback` кормит химию пользователя.

UI-polling каждые 3 секунды: `sync_error = ‖user_vec − system_vec‖`, режим (sync_regime) — derived. Две симметричные панели + sync-индикатор.

## UI

Header таба — две симметричные панели вокруг sync-индикатора. Сверху строка «ТЫ»: Интерес / Стабильность / Напряжение / Усталость. В центре — пилюля режима и процент синхронизации. Снизу строка «BADDLE» с теми же метками.

Одинаковый цветовой код обеих сторон + симметричная вёрстка = визуально видно **где расходимся и куда тянет**. Цвет пилюли режима: FLOW — зелёный, REST — серый, PROTECT — оранжевый, CONFESS — малиновый.

## Где в коде
