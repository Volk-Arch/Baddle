# Байесовский конус — дизайн-метафора Baddle

## Происхождение

Метафора возникла из онтологии: человек материализован в точку, а мысли — это конус из этой точки. Не была спроектирована — была обнаружена: каждый компонент, созданный для решения конкретной проблемы, оказался операцией на конусе.

## Модель

Есть глобальное **поле возможностей** — всё что LLM может сгенерировать. Без ограничений — шум.

Человек — **точка** в этом поле. Его мысли — **конус из этой точки**: веер направлений, куда можно двигаться. Baddle управляет конусом, не точкой.

```
Широкий конус (conf=0.3)    Узкий конус (conf=0.9)    Повёрнутый конус
     ╱    ╲                      │╲                         ╱│
    ╱      ╲                     │ ╲                       ╱ │
   ╱ много  ╲                    │одна╲                   ╱  │
  ╱ гипотез  ╲                   │верс.╲                другой│
 ╱            ╲                  │     ╲               аспект │
```

**Ширина** = uncertainty (сколько направлений открыто).
**Длина** = depth (как далеко продвинулись вдоль направления).
**Ориентация** = focus (куда направлен конус).

## Пять операций на конусе

Каждый инструмент Baddle — одна из пяти операций:

### 1. Расширение (Generative)

**Brainstorm** — больше направлений из текущей точки. Конус шире.
**Expand** — новый ракурс от той же точки. Ещё одно направление.
**Elaborate** — глубже вдоль одного направления. Конус длиннее.

### 2. Сужение (Verification)

**Smart DC** — отсечение невозможного. Тезис/антитезис/синтез фильтруют слабые направления. Конус уже, но увереннее.
**Ask** — "а точно ли дно конуса твёрдое?" Проверка допущений.

### 3. Фиксация (Structural)

**Collapse** — из веера → одна точка. Конус схлопывается, появляется новая точка для нового конуса.
**Merge** — два похожих направления → одно сильное.
**Rephrase** — тот же вектор, другая траектория. Не меняет конус, меняет путь.

### 4. Навигация (Walk, META)

**Walk** — куда ведёт мысль по transition probabilities. Обход конуса.
**META** — "конус направлен туда?" Переоценка ориентации.

### 5. Поворот (Search)

**Pump** — новая ось, невидимая изнутри конуса. Два конуса (от двух идей) → поиск общего пространства → поворот на скрытый параметр.

## Prediction error — двигатель конуса

Мозг **всегда ожидает чуть больше** чем получает (predictive processing, Фристон). Несоответствие (`ε > 0`) — двигатель внимания, обучения, любопытства.

В Baddle: `target_surprise` в CognitiveState:

```
surprise = 1 - confidence_после_SmartDC

surprise < target → "предсказуемо" → precision ↓ → конус расширяется
surprise > target → "хаотично"     → precision ↑ → конус сужается
surprise ≈ target → зона потока    → система движется вперёд
```

`target_surprise > 0` **всегда**. Система хочет чтобы реальность чуть не совпадала с ожиданием — иначе зачем продолжать думать.

## Замкнутый контур

```
prediction error → precision → temperature + top_k + novelty →
→ разнообразие идей → SmartDC → confidence → surprise →
→ prediction error (цикл замыкается)
```

Precision управляет **всем сразу**: temperature (креативность), top_k (количество кандидатов), novelty_threshold (строгость фильтра). Один параметр — три эффекта:

- **EXPLORATION** (precision ↓): высокая temp, большой top_k, мягкий novelty → широко ищем
- **EXECUTION** (precision ↑): низкая temp, малый top_k, строгий novelty → фокусируемся

## Конвергенция, не дизайн

Эта связность не была спроектирована:
- Конус — из онтологии ("человек = точка, мысли = конус")
- Horizon — из проблемы фиксированной температуры
- 4 примитива — из избыточности 8 операторов
- Pump — из вопроса "как найти связь между далёкими идеями"

Каждый компонент — ответ на конкретную проблему. И потом оказалось что он описан в науке 30 лет назад (Active Inference, Хебб, диалектика, Гилфорд).

## Универсальный ритм: divergence / convergence

Расширение и сужение конуса — частный случай фундаментального паттерна,
который повторяется на каждом уровне реальности где есть эволюция во
времени. Вдох и выдох.

| Уровень | Дивергенция | Конвергенция |
|---|---|---|
| Космос | Большой взрыв, расширение | Гравитация, формирование галактик |
| Квантовый | Суперпозиция | Измерение (коллапс в одно) |
| Термодинамика | Излучение, рассеяние | Локальный порядок (звёзды, жизнь) |
| Биология | Мутации, видообразование | Отбор, конвергентная эволюция |
| Разум | Воображение, идеи | Обучение, решения, память |
| Git | branch, fork | merge, commit |
| Baddle | Brainstorm / Expand / Elaborate | SmartDC / Collapse / Merge |

Без вдоха нет структуры, без выдоха нет развития. Оба нужны. **Pump** —
особый случай: не расширение и не сужение, а **поворот** конуса
(см. раздел «5. Поворот» выше). Третья операция поперёк основной оси.

### Источник ритма — напряжение между состояниями

Ритм не данность. Возникает когда между двумя точками есть градиент:

| Уровень | Градиент | Что порождает |
|---|---|---|
| Физика | Температурный | Поток тепла |
| Космос | Гравитационный | Коллапс в звёзды |
| Жизнь | Голод, угроза | Поиск → потребление → рост |
| Разум | Вопрос | Поиск → понимание → новый вопрос |
| Baddle | Prediction error | generate → doubt → merge → новый generate |

`target_surprise > 0` всегда — системе нужно чтобы реальность чуть не
совпадала с ожиданием. Иначе нет топлива. И это не круг, а **спираль**:
каждый цикл создаёт новую абстракцию, новый уровень, новые вопросы.

### Отношение первичнее объекта

Обычная интуиция — сначала объекты, потом отношения. В физике и в
Baddle наоборот: смысл появляется из связей.

```python
a = 10
b = 10
a == b  # True — одинаковое значение
a is b  # False — разные сущности
```

В `is False` — источник напряжения. Два электрона (одинаковые свойства,
разная позиция), два коммита (одинаковый код, разная история), две
мысли в графе (похожий эмбеддинг, разный контекст). Merge — **синтез
нового из напряжения между похожими**. Если различие исчезает, ритм
останавливается. Различие — топливо.

В графе нода без рёбер бессмысленна — ценность определяется связями.
Поэтому `distinct()` ([nand-architecture.md](nand-architecture.md)) —
примитив: не свойство ноды, а отношение между двумя.

### Баланс двух крайностей

- **Слишком много конвергенции** — догма, узость, застревание в локальном
  оптимуме. Всё решается, ничего нового не появляется.
- **Слишком много дивергенции** — хаос, неспособность решить. Всё
  рассматривается, ничего не завершается.

Хорошее мышление — сначала разойтись, потом сойтись. Ни одна крайность
не достаточна. Нужен метроном — у нас это `CognitiveState.precision` и
`target_surprise`. Не оптимизируем конвергенцию («быстрее решение») —
оптимизируем **ритм**.

### Почему именно этот паттерн

Baddle не имитирует мышление, а воспроизводит оптимальный паттерн для
системы которая:

- обрабатывает информацию (ноды)
- ищет паттерны (рёбра)
- работает в условиях неопределённости (Bayes)
- имеет ограниченные ресурсы (LLM tokens)
- сохраняет историю (граф)

Одинаковые ограничения → одинаковые решения. Конвергентная эволюция идей.

## Связь с режимами

Каждый из 13 режимов = форма конуса:

| Режим | Форма конуса |
|-------|-------------|
| Свободный | Управляется вручную |
| Блуждание | Сфера — все стороны равны |
| Фокус | Узкий → сужается до точки |
| Привычка | Пульсирующий — сужение ↔ расширение |
| Исследование | Сужается медленно, никогда не до точки |
| Мозговой штурм | Один широкий, не сужается |
| Выбор | Все сужаются, выживает один |
| Дебаты | Два сталкиваются → синтез |
