# Байесовский конус — дизайн-метафора Baddle

## Происхождение

Метафора возникла из онтологии: человек материализован в точку, а мысли — это конус из этой точки. Не была спроектирована — была обнаружена: каждый компонент, созданный для решения конкретной проблемы, оказался операцией на конусе.

## Модель

Есть глобальное **поле возможностей** — всё что LLM может сгенерировать. Без ограничений — шум.

Человек — **точка** в этом поле. Его мысли — **конус из этой точки**: веер направлений, куда можно двигаться. Baddle управляет конусом, не точкой.

- **Ширина** = неопределённость (сколько направлений открыто).
- **Длина** = глубина (как далеко продвинулись вдоль направления).
- **Ориентация** = фокус (куда направлен конус).

Широкий конус (низкая уверенность около 0.3) — много гипотез веером. Узкий конус (высокая уверенность около 0.9) — одна основная версия. Повёрнутый конус — смена угла рассмотрения, другой аспект.

## Пять операций на конусе

Каждый инструмент Baddle — одна из пяти операций:

### 1. Расширение (Generative)

**Brainstorm** — больше направлений из текущей точки. Конус шире.
**Expand** — новый ракурс от той же точки. Ещё одно направление.
**Elaborate** — глубже вдоль одного направления. Конус длиннее.

### 2. Сужение (Verification)

**Smart DC** — отсечение невозможного. Тезис / антитезис / синтез фильтруют слабые направления. Конус уже, но увереннее.
**Ask** — «а точно ли дно конуса твёрдое?». Проверка допущений.

### 3. Фиксация (Structural)

**Collapse** — из веера в одну точку. Конус схлопывается, появляется новая точка для нового конуса.
**Merge** — два похожих направления в одно сильное.
**Rephrase** — тот же вектор, другая траектория. Не меняет конус, меняет путь.

### 4. Навигация (Walk, META)

**Walk** — куда ведёт мысль по вероятностям переходов. Обход конуса.
**META** — «конус направлен туда?». Переоценка ориентации.

### 5. Поворот (Search)

**Pump** — новая ось, невидимая изнутри конуса. Два конуса (от двух идей) → поиск общего пространства → поворот на скрытый параметр.

## Ошибка предсказания — двигатель конуса

Мозг **всегда ожидает чуть больше** чем получает (predictive processing, Фристон). Несоответствие (**ошибка предсказания** ε > 0) — двигатель внимания, обучения, любопытства.

В Baddle есть **целевое удивление** (target_surprise) в CognitiveState. **Текущее удивление** = 1 − confidence после SmartDC.

- удивление ниже целевого → «предсказуемо» → точность падает → конус расширяется
- удивление выше целевого → «хаотично» → точность растёт → конус сужается
- удивление около целевого → зона потока → система движется вперёд

Целевое удивление всегда больше нуля: система хочет чтобы реальность чуть не совпадала с ожиданием — иначе зачем продолжать думать.

## Замкнутый контур

Ошибка предсказания → точность → одновременно температура, top_k и порог новизны для LLM → разнообразие идей → SmartDC → confidence → удивление → ошибка предсказания (цикл замыкается).

Точность управляет **всем сразу**: температурой (креативность), top_k (количество кандидатов), порогом новизны (строгость фильтра). Один параметр — три эффекта:

- **EXPLORATION** (точность падает): высокая температура, большой top_k, мягкий фильтр новизны → широко ищем.
- **EXECUTION** (точность растёт): низкая температура, малый top_k, строгий фильтр новизны → фокусируемся.

## Конвергенция, не дизайн

Эта связность не была спроектирована:
- Конус — из онтологии («человек = точка, мысли = конус»)
- Horizon — из проблемы фиксированной температуры
- 4 примитива — из избыточности 8 операторов
- Pump — из вопроса «как найти связь между далёкими идеями»

Каждый компонент — ответ на конкретную проблему. И потом оказалось что он описан в науке 30 лет назад (Active Inference, Хебб, диалектика, Гилфорд).

## Универсальный ритм: divergence / convergence

Расширение и сужение конуса — частный случай фундаментального паттерна, который повторяется на каждом уровне реальности где есть эволюция во времени. Вдох и выдох.

| Уровень | Дивергенция | Конвергенция |
|---|---|---|
| Космос | Большой взрыв, расширение | Гравитация, формирование галактик |
| Квантовый | Суперпозиция | Измерение (коллапс в одно) |
| Термодинамика | Излучение, рассеяние | Локальный порядок (звёзды, жизнь) |
| Биология | Мутации, видообразование | Отбор, конвергентная эволюция |
| Разум | Воображение, идеи | Обучение, решения, память |
| Git | branch, fork | merge, commit |
| Baddle | Brainstorm / Expand / Elaborate | SmartDC / Collapse / Merge |

Без вдоха нет структуры, без выдоха нет развития. Оба нужны. **Pump** — особый случай: не расширение и не сужение, а **поворот** конуса (см. раздел «5. Поворот» выше). Третья операция поперёк основной оси.

### Источник ритма — напряжение между состояниями

Ритм не данность. Возникает когда между двумя точками есть градиент:

| Уровень | Градиент | Что порождает |
|---|---|---|
| Физика | Температурный | Поток тепла |
| Космос | Гравитационный | Коллапс в звёзды |
| Жизнь | Голод, угроза | Поиск → потребление → рост |
| Разум | Вопрос | Поиск → понимание → новый вопрос |
| Baddle | Ошибка предсказания | generate → doubt → merge → новый generate |

Целевое удивление всегда больше нуля — системе нужно чтобы реальность чуть не совпадала с ожиданием. Иначе нет топлива. И это не круг, а **спираль**: каждый цикл создаёт новую абстракцию, новый уровень, новые вопросы.

### Отношение первичнее объекта

Обычная интуиция — сначала объекты, потом отношения. В физике и в Baddle наоборот: смысл появляется из связей.

Два объекта с одинаковыми свойствами могут быть разными сущностями (в Python: a == b проверяет значение, a is b — идентичность). В различии (`is False`) — источник напряжения. Два электрона (одинаковые свойства, разная позиция), два коммита (одинаковый код, разная история), две мысли в графе (похожий эмбеддинг, разный контекст). Merge — **синтез нового из напряжения между похожими**. Если различие исчезает, ритм останавливается. Различие — топливо.

В графе нода без рёбер бессмысленна — ценность определяется связями. Поэтому различение (distinct) ([nand-architecture.md](nand-architecture.md)) — примитив: не свойство ноды, а отношение между двумя.

### Баланс двух крайностей

- **Слишком много конвергенции** — догма, узость, застревание в локальном оптимуме. Всё решается, ничего нового не появляется.
- **Слишком много дивергенции** — хаос, неспособность решить. Всё рассматривается, ничего не завершается.

Хорошее мышление — сначала разойтись, потом сойтись. Ни одна крайность не достаточна. Нужен метроном — у нас это точность (CognitiveState.precision) и целевое удивление. Не оптимизируем конвергенцию («быстрее решение») — оптимизируем **ритм**.

### Почему именно этот паттерн

Baddle не имитирует мышление, а воспроизводит оптимальный паттерн для системы которая:

- обрабатывает информацию (ноды)
- ищет паттерны (рёбра)
- работает в условиях неопределённости (Bayes)
- имеет ограниченные ресурсы (LLM tokens)
- сохраняет историю (граф)

Одинаковые ограничения → одинаковые решения. Конвергентная эволюция идей.

## Связь с режимами

Каждый из 13 режимов = форма конуса:

| Режим | Форма конуса |
|-------|-------------|
| Свободный | Управляется вручную |
| Блуждание | Сфера — все стороны равны |
| Фокус | Узкий → сужается до точки |
| Привычка | Пульсирующий — сужение ↔ расширение |
| Исследование | Сужается медленно, никогда не до точки |
| Мозговой штурм | Один широкий, не сужается |
| Выбор | Все сужаются, выживает один |
| Дебаты | Два сталкиваются → синтез |
