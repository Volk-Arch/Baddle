# Baddle: AHI Protocol

**Augmented Human Intelligence** — усиленный человеческий интеллект.

Не искусственный интеллект, который заменяет человека. Усиленный человеческий, который делает мышление прозрачным, проверяемым и направленным.

---

## Основная идея

Любая задача — это навигация из точки A в точку B. Человек знает где он и куда хочет. Но путь между ними — хаос из догадок, сомнений, забытых идей и повторных размышлений.

Baddle делает этот путь **явным графом**. Каждая мысль — узел, каждое сомнение — ребро, каждая проверка — confidence. Граф растёт, самоорганизуется, проверяет себя и в итоге приводит к цели.

```
A (где ты сейчас) ──▶ [граф мыслей] ──▶ B (куда хочешь)
                            │
                            ├── Байес: уверенность
                            ├── Марков: вероятность переходов
                            └── Smart DC: диалектика
```

---

## Алгебра мышления

Любая задача описывается тремя измерениями:

1. **Сколько целей?** — 0 / 1 / 2+
2. **Какой примитив объединяет их?** — `none / AND / OR / XOR` (всего четыре)
3. **Какой тип цели?** — finite / repeatable / open

Из этого получается **12 режимов мышления** — полный базис.

### Четыре примитива

| Примитив | Суть |
|----------|------|
| **none** | Нет целей вообще |
| **AND** | Все цели должны быть достигнуты |
| **OR** | Достаточно любой одной |
| **XOR** | Ровно одна выбирается, остальные отбрасываются (OR + исключение) |

Остальные операторы — не отдельные примитивы, а **стратегии применения**:

- **SEQ, PRIORITY, BALANCE** — это разные способы обходить AND (по зависимостям, по важности, пропорционально над нефинитными)
- **Comparative и Dialectical** — два способа разрешать XOR (сравнение независимых vs синтез противоречивых)

### 12 режимов

| Слой | Режим | Примитив | Стратегия / тип | Пример |
|------|-------|----------|----------------|--------|
| **0 целей** | Разведка (Scout) | none | — | Прогулка, серфинг |
| **1 цель** | Вектор (Vector) | focus | finite | Сдать проект |
| | Ритм (Rhythm) | focus | repeatable | Завтрак, зарядка |
| | Горизонт (Horizon) | focus | open | Изучение языка |
| **AND** | Конструктор (Builder) | AND | unordered | Сборка мебели |
| | Конвейер (Pipeline) | AND | SEQ (зависимости) | Рецепт, чек-лист |
| | Каскад (Cascade) | AND | PRIORITY (важность) | Тайм-менеджмент |
| | Весы (Scales) | AND | BALANCE (пропорции) | Бюджет |
| **OR** | Гонка (Race) | OR | finite | Любое такси |
| | Веер (Fan) | OR | open | Мозговой штурм |
| **XOR** | Турнир (Tournament) | XOR | comparative | Одна машина из трёх |
| | Диспут (Dispute) | XOR | dialectical | Научная дискуссия, суд |

### Типы целей

Стоп-условие зависит от типа цели, не от режима:

| Тип | Суть | Стоп | Состояние после |
|-----|------|------|----------------|
| **finite** | Есть финал | `confidence ≥ threshold` | RESOLVED → архив |
| **repeatable** | Ритм/привычка | `step_complete + trigger` | SCHEDULED → ждёт триггера |
| **open** | Бесконечное уточнение | `diminishing_returns ИЛИ budget` | PARKED → ждёт новых данных |

### Иерархия графов

- **1 граф = 1 задача** (1 режим)
- Связи между графами хранятся в **отдельном мета-графе**

Мета-граф соединяет задачи по смыслу, времени, зависимостям. Там живёт cross-graph ассоциативное воспоминание: "недосып коррелирует с пиком креативных идей".

## Три движка

| Движок | Что делает | Пример |
|---|---|---|
| **Байес** | Уверенность узла 0.0–1.0 | "Удалёнка эффективнее" → conf=0.5, после проверки → 0.82 |
| **Марков** | Вероятность перехода между узлами | A→B: 0.7, A→C: 0.3. "Куда ведёт мысль?" |
| **Smart DC** | Тезис + антитезис + нейтраль → синтез | Слабая гипотеза → три полюса → центроид → проверенный узел |

**Откуда числа:**
- Confidence: logprobs при генерации → расстояние синтеза до центроида → Bayes update от evidence
- Transition_prob: cosine similarity embeddings → directed edges → навигация пользователя (Хебб)

---

## Цикл мышления

```
Ввод мысли → confidence = 0.5 (гипотеза)
       ▼
Merge похожих (если есть)
       ▼
Elaborate (добавить evidence)
       ▼
Smart DC: тезис / антитезис / нейтраль → центроид → синтез (conf = 0.82)
       ▼
Рекурсия → META: "что упустил?" → новые идеи
       ▼
Novelty exhaustion → финальный синтез
       ▼
Человек: ✓ Merge | ✗ Reject | ✎ Edit | ↻ Retry
```

## Принципы

**Человек — Root Admin.** ИИ предлагает, человек решает. Никаких автоматических merge без возможности отката.

**Энтропия — топливо.** Система ищет слабые места (conf < threshold), не ждёт вопросов.

**Время — связь.** Мысли одной сессии связаны контекстуально, даже если семантически далеки.

**Принудительный антитезис.** SmartDC всегда генерирует контраргумент. Это ломает эхо-камеры и не даёт модели соглашаться сама с собой.

**Сходимость через исчерпание.** Цикл останавливается когда модель начинает повторяться (novelty check), а не по искусственному лимиту.

**Граф — явная внешняя память.**

| В голове | В графе |
|---|---|
| Интуиция | `confidence: 0.73` |
| Ассоциации | Рёбра + `transition_prob` |
| Сомнения | Низкий `evidence_count` |
| "За и против" | Smart DC → синтез |
| История | Git log, timestamps |

## Возможности

### Автономное блуждание
ИИ обходит граф в фоне: находит слабые узлы, предлагает синтезы, ищет мосты между кластерами. Каждый шаг — коммит. Merge — только за человеком.

### Цикл день/ночь
Днём — сбор мыслей (дивергенция). Ночью — автоколлапс по горячим точкам (конвергенция). Утром — структурированные выводы вместо хаоса.

### Ассоциативный доступ
Не поиск по ключевым словам, а распространение активации по графу:
- **Прямой** — по смыслу (embedding similarity)
- **Контекстный** — по состоянию (recent, related topics)
- **Непрямой** — по паттернам ("12 раз пропустил обед" — не вопрос, а инсайт)

### Cross-graph ассоциации
Несколько графов (Работа, Здоровье, Идеи) + связи между ними в мета-графе. Блуждание с перескоком: система видит корреляции между разными областями жизни.

### Timeline player
Граф как фильм. ⏮▶⏸⏭ по timestamps — видно как мысль развивалась, где сомнения появлялись, где разрешались.

## AGI vs AHI

| AGI | AHI |
|---|---|
| Заменяет человека | Усиливает человека |
| Чёрный ящик | Прозрачный граф |
| "Решает за вас" | "Предлагает, решаете вы" |
| Alignment через RLHF | Alignment через архитектуру |

**Безопасность по конструкции.** Граф виден. Человек мержит. Байес не даёт модели быть догматичной (любое утверждение — confidence, не истина). Принудительный антитезис ломает эхо-камеры. Каждое решение можно откатить.

## Интеграция с LLM

| | LLM без графа | LLM с графом |
|---|---|---|
| Режим | **Буст** — даём модели память | **Симбиоз** — два графа соединяются |
| Результат | Модель полезнее | Когнитивный партнёр |

Три слоя: LLM (смысл) + Байес (уверенность) + Марков (динамика).

## Экосистема

**Baddle Desktop** — локальный ИИ + персональный граф. 100% приватность, zero cloud.

**Graph Store** — маркетплейс графов знаний (МБ, не ГБ). Доменные графы: медицина, право, наука.

**Git Verify** — MR для знаний, review экспертами, рейтинги. Научный метод в архитектуре.

### Применение

- **Образование** — граф знаний с confidence вместо оценок. Экзамен = merge request
- **Медицина** — симптомы → паттерны → Smart DC по слабому диагнозу → врач решает
- **Наука** — публикация = граф. Рецензия = MR. Fork = альтернативная интерпретация
- **Работа** — мессенджер без поиска → всё важное в граф → ночной коллапс → утром выводы
- **Личные решения** — "стоит ли менять работу?" → Весы или Турнир → проработанное решение вместо паралича
