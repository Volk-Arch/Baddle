# Легенда к графу: онтология разворачивания (v3)

*Полное описание каждого узла в [ontology-v3.json](ontology-v3.json) с указанием связей, типа, уверенности и развёрнутого смысла. Включает ветвь H — реализация в Baddle.*

**Условные обозначения**

- **Тип узла** — `goal` (тема), `fact` (утверждение, в котором уверены), `hypothesis` (гипотеза для проверки), `evidence` (поддерживающее наблюдение), `question` (открытый вопрос).
- **Confidence** — субъективная уверенность в формулировке (0..1). Низкая ≠ «плохо», означает «требует доработки/проверки».
- **Глубина** — расстояние от корня. `-1` = тема, `0` = главная ветвь, `1+` = подузлы.
- **→** — направленная связь (родитель → ребёнок).
- **Realizes:** — для узлов ветви H, список ID онтологических узлов, которые этот код реализует.
- **Status** — для узлов ветви H, состояние реализации (`done` / `planning`).

## Узлы без реализации (точки расхождения с Baddle)

- `[4]` A ≠ A — нет явной реализации, философский фундамент
- `[9]` Полнота требует проявления — без реализации
- `[10]` Бытие = акт выбора — точка роста
- `[22]` Реальность многослойна — Baddle технически многослоен, но не выделено как принцип
- `[23-31, 33]` Эмерджентность и видимый мир — наблюдательная ветвь, не нуждается в реализации
- `[42]` Граница содержит больше — точка роста
- `[51, 52]` Горизонт + Остров — практические границы, без реализации
- `[53, 54]` Бесконечность + Форма мира — открытые вопросы

И обратно: некоторые Baddle-концепции **шире онтологии**:

- HRV / sensor stream — телесность, нет онтологического узла
- LLM как генератор — внешний резонатор-ассистент, нет узла
- Time-of-day / circadian — циклическое время, нет узла
- Action vs observation — у Baddle разделение, у онтологии слиты в «акт сужения»

Это **кандидаты на расширение онтологии в v4**.

## Что делать дальше

- **Использовать как карту**. При новой задаче в Baddle — посмотри, какой онтологический узел она реализует. Если **ни один** — либо концепт чужеродный, либо онтология недоописывает реальность.
- **Уточнять status**. Когда узел planning становится done — обновить status и поднять confidence (обычно 0.7 → 0.85+).
- **Добавлять связи `realizes`** при появлении новых концепций в Baddle.
- **Расширять онтологию**, когда в Baddle появляются устойчивые концепции без онтологического узла (HRV, LLM, time-of-day).
- **Сравнивать v2 → v3 → v4** через несколько недель — какие planning-узлы станут done, какие гипотезы подтвердятся, какие переедут.

Граф v3 — **самоописывающая система**. Он содержит и фундамент, и его реализацию, и связь между ними. Его можно использовать как навигационный инструмент при работе над Baddle, и как философский компас при осмыслении того, что строишь.
